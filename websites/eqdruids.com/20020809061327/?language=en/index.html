New Bazaar and UI information 8/9/02

From EQLive:

 

There are some new(ish) things to report about the interface, bazaar and other odd things. Some of this you may already know, but some of it you may not. I’m just going to give you a list of stuff.

---

We recently changed the way the UI looks for its files. If you load a skin that is missing pieces it will find and use those missing pieces in your default folder. What this means is that if you forget to copy a file into the folder for your new skin, the game will use the default file. It also means that if we update the interface and add new files, your existing skins should continue to work, since the UI will look for those files in the default directory.

---

/loadskin used without any parameters will give you a usage statement. But now it also gives you a list of all of the folder names it finds in your UI directory. That’s basically a list of your available skins.

---

We’re adding three new labels:

Current Alternate Advancement points available to spend = 71
Current experience percentage assigned to Alternate Advancement = 72
Character Last Name = 73

---

The issue that was causing folks to target and inspect people, open doors and all sorts of other things when using the mouse to move has been fixed.

---

I’ve had several emails asking for the ability to search for hand-to-hand weapons in the Bazaar. That ability already exists. Use “martial” on the dropdown list to find them.

---

Yes, we’re aware of the issue with tracking. Items leaving the list (like NPCs dying) causes the list to resort and the focus to jump to the top, making it very hard to use. This is fixed on Test.

---

/camp is ignoring optional parameters (like Desktop). We know about this, but it’s not a real priority item. We’ll get it when we can.


Alan

Cross Server chat now available 7/26/02

From EQLive:

EQ Technical Director, Scott Hartsman, answered questions from the community about an unannounced feature that was included with the July 24th patch.

Yes - Cross Server chat is something that was done intentionally - At no small amount of work for both our development and ops teams, for that matter.
 
The reason that it wasn't announced is because in order to allow this, we had to change around the chat systems quite a bit. We needed a few days of measuring what would happen with existing channel chat under the new arrangement as a proof of concept. 
 
If that blew up before anyone was chatting globally, we'd have known that we had more work to do. :)
 
The other reason that it wasn't advertised yet -- As it stands, people have the ability to spam (and in some cases, are spamming) cross-server without any of the standard in-game repercussions.  (i.e. If you're a jerk: I won't group with you, my guild won't help you, etc.)  That definitely wasn't part of the plan, and we'd hoped to have that addressed before it was released.
 
Over the next few days, we'll be adding more controls that allow people to keep their channels and private messages local to their servers, among other things.
 
Best,
 
- Scott

To use the new features you use the following format:

;tell <servername>.<playername> <message>

;join <servername>.<channelname>
