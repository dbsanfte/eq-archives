    PvP Character Transfer
    7/29/02, 11:36 PST | Kwill

    The EverQuest Character Transfer Service, a value added service, is now allowing players to transfer characters from PvP servers to non-PvP servers.

    Key points to this service change include:

  • Characters may be moved between one of your accounts to one of your other accounts on the following servers: Sullon Zek ,The Test Server and Firiona Vie. Movement to or from these servers will not be allowed.
  • Moves from a PVP Server (Rallos Zek, Tallon Zek and Vallon Zek but excluding Sullon Zek) to a non-PVP server are permitted, with the character's pvp switch set to non-pvp status.
  • Moves from a non-PVP Server to a PVP server (Rallos Zek, Tallon Zek and Vallon Zek but excluding Sullon Zek) are permitted, with the character's pvp switch being set PVP status.

    Cross Server Chat
    7/29/02, 11:35 PST | Kwill

    From EQ Live:
    Cross Server Chat - An Unannounced Feature

    EQ Technical Director, Scott Hartsman, answered questions from the community about an unannounced feature that was included with the July 24th patch.

    Yes - Cross Server chat is something that was done intentionally - At no small amount of work for both our development and ops teams, for that matter.

    The reason that it wasn't announced is because in order to allow this, we had to change around the chat systems quite a bit. We needed a few days of measuring what would happen with existing channel chat under the new arrangement as a proof of concept.

    If that blew up before anyone was chatting globally, we'd have known that we had more work to do. :)

    The other reason that it wasn't advertised yet -- As it stands, people have the ability to spam (and in some cases, are spamming) cross-server without any of the standard in-game repercussions. (i.e. If you're a jerk: I won't group with you, my guild won't help you, etc.) That definitely wasn't part of the plan, and we'd hoped to have that addressed before it was released.

    Over the next few days, we'll be adding more controls that allow people to keep their channels and private messages local to their servers, among other things.

    Instrument Mastery
    7/25/02, 21:05 PST | Kwill

    Absor had this to say to Bards, from the Developer's Corner:
    Ok, here’s what happened with Instrument Mastery. Though this seems obvious to me, I want to make it clear how this was supposed to work.

    Singing Mastery is designed to improve the bard’s ability to sing songs.

    Instrument Mastery is designed to improve the bard’s ability to play instruments.

    Now there is a bit of possible confusion when we start talking about songs that are instrument based when the bard is not using an instrument. At that point the bard is singing and instrument song, and it really becomes a matter of taste as to whether Singing Mastery or Instrument Mastery should apply. We’ve decided to base it on the skill used, not whether an instrument is in hand. So Singing Mastery applies to all Singing skill based songs and Instrument Mastery applies to all instrument based songs, whether or not the bard is using an instrument at the time.

    No, these two abilities were never meant to overlap. It was a bug when Instrument Mastery gave a bonus to songs that use the Singing skill.

    That bug was fixed this patch. Instrument Mastery no longer affects songs that use the Singing skill.

    What we’ve decided to do is increase the amount that Singing Mastery affects Singing skill songs by 50%. That should go in the next time we patch.

    I’m sorry for not catching this in the patch message. Basically because this was the way it was always intended to be, we were not aware that we’d fixed a bug.


    Spell Gem Colors
    7/25/02, 20:17 PST | Kwill

    Absor clarified some things about the new spell gem colors and the new interface on the Developer's Corner:
    With all of the new spell gems and with the change to the interface, it seems inevitable that we’d get a few spell icons or a few gem colors crossed.

    For example, I hear that Nektulos Portal has the Gate spell icon. We might want to change that to match the other group teleport spells.

    And Pillar of Frost seems to have an orange spell gem and it should probably be green.

    Here’s a list of the gem colors and what they should mean, for reference:

    Yellow: Personal - spells that you can only cast on yourself.
    Red: Single Target - spells that affect (and require) a single target.
    Blue: Point Blank Area of Effect - spell affects an area centered on the caster, but the caster is unaffected.
    Green: Area of Effect - spell affects targets in the area centered on the target.
    Orange: Area of Effect Rain - spell affects the area around the target, often in a continuing fashion.

    The distinction between Green and Orange is pretty narrow.
