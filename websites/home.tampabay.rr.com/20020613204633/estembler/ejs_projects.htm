Memory Status
The Memory Status program is a Internet Explorer applet, which displays the Physical Memory, Memory Usage, Page File Memory and Virtual Memory for the computer it is running on.

The program exhibits the following techniques and or methodologies internally:

  • The Object Oriented principle of Inheritance
  • The Object Oriented principle of Polymorphism.
  • The Factory design pattern.
  • The use of Windows APIs.
SIPAC
Acted as Senior Programmer Analyst and Lead Component Designer for Sitech, Inc. on their multi-million dollar, insurance property and casualty software, developed using C++Builder 3, Delphi 3 and 4.  Initially acted in the capacity of a consultant/advisor.   Devised a strategy for modularizing and distributing application as multi-tiered.   Redesigned entire user interface to coincide with "Explorer" style paradigm used by the operating system.  Designed and implemented, reusable component and object library, utilized system wide.  Employed various advanced Object Oriented techniques and or methodologies such as Design Patterns, Interfaces, Metaclasses, Threads, State Machines, Polymorphism and the Unified Modeling Language (UML) etc.   Responsible for solving numerous critical problems.  Resident Object Oriented expert and Microsoft Component Object Model (COM) guru.

Tax Trac'r 97
Business tax suite done for TRAC'R Software, Inc. during 1996-1997.  Developed from the ground up utilizing Delphi 3 Client/Server, Visual Basic 5.0 Enterprise, Access 7.0, Crystal Reports, ODBC, DAO and BDE 4.01.  The final product consisted of 23 Executables, 4 Dynamic Link Libraries, 1 ActiveX control, 51 OLE Automation objects, 31 Reports, 3 databases (52 Tables and 36 Stored Procedures), in addition to various third party distributed files. Developed many RAD utilities, used company wide, to expedite the development process.
