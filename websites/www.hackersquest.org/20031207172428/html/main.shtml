EthernalQuest updates :: 4.11.2003
Dont ask me how many chickens have been sacrificed to produce this code, but Zordon has produced a set of 3 updates over the past couple of days and added an amazing number of new features to EthernalQuest, the EverQuest Emulator!

0.7.05 Alpha [Source | Dev | DB]
* basic spell operations are in place
- scribing into spellbooks (and moving them around)
- memorizing spells
- casting spells (no effects yet)
* save/load support for languages, skills and factions
* initial skills should be now set correctly
* you no longer need to zone after #mob level
(you still need to zone if you lower you level)
* changing player level will set correct values of max available HP's and MANA on the server side
* hp & mana regeneration should work
* players should stop disappearing when they are not moving
* #summon should work
* #experience command was deleted
* #item find will show only first 50 items found
* /tell and /say should work
* corpses should no longer attack
* skill value in #skill command is validated
* item index in #item create command is validated
* changes to networking level
* no longer accepting client hp updates (all hp calculations are now server based)
* fixed issues with monster speed == 0
* #mob create should work better
* various problems fixed related to logining back when your character is still online
* #mob hp command will not be able to set HP over current mob's maxHP
(you still can use [#mob hp currHP, maxHP] to set both)
* HP regeneration will now work correclty if you manually set mob's hp to it's max value
or when MaxHP fall below mob's current HP value
* you can play frogloks now
* EqDb format was changed
* added soldungc zone

0.7.04 Alpha
* simple attack implementation (damage always = 1)
see EQ_MobLogic/EQ_AttackLogic/*.cpp if you do not like it
* mobs will follow you if you attack them
* you should die correctly (both server and client side death should be working fine)
* distance and direction calculation fixed to give almost correct values
* bug fix: problem with z being 10 times more then it suppose to be fixed
- as a result #goto and #zone should be working much better
- this required change in EqDb files but you can keep your player profiles
and just replace zone mob files
* most #mob commands should be working

0.7.03 Alpha
* updates to support current 'live' client
* NPC merchants are back to work
* money support
* bug fix: items will show correct stats
* EQ_DB database changes to support current 'live' client
- item format changes
- mob shop storage changes

Heads up to Zordon for sharing this big chunk of high quality code with us!

/winke

EthernalQuest 0.6.1 Alpha is out! :: 3.7.2002
Heres the changelog:

Version 0.6.1 Alpha

* bug fix: client was crashing when dealing with packs and books
* bug fix: client was crashing when mobs/players deleted from zone
* bug fix: client was disconnected after you stay in one zone for exteding period of time
* mobs will now follow you when attacked
* /consider should work properly
* new # commands
#spawn findskill [ skillPattern ] - list all skills that match pattern
#spawn findlanguage [ laguagePattern ] - list all languages that match pattern
#spawn skill [skillNameOrIndex [, skillValue]] - show or change skill values (player only)
#spawn language [languageNameOrIndex [, languageValue]] - show or change language abilities (player only)
* guildmasters will function properly
* you will be able to hit for more then 1 hp. Formula is a bit more real then one posted in forum:
STR / 5 + level + [ 0 .. level ] + 2. You will hit for about 80 to 140 at level 60.
* you can now scribe / memorize and cast spells. Do not expect any server side effects.
All client side effects should work fine.
* it should be easier to attack something
* Spell viewer/editor is included as part of EMU
* EqDb version 0.3.0003 included

Get the EthernalQuest / Src packages from the filetable above!
