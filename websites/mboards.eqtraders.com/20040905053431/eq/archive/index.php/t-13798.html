Heh. I do wonder what they use as the output from the RNG. Most RNGs I've seen have output in a fixed range, which is then squished into the desired range (usually along the lines of ( ( (randNum - minRand) / (maxRand-minRand) ) * (maxDesired-minDesired) + minDesired)). Thus creating the potential for output that looks really, really random to look extremely streaky when the range of possible outputs is small.

Random Number Generation is a MATURE TECHNOLOGY. There are some small improvements here and there, and that's well and good. But basicly even the most TRIVIAL pRNG (pseudo-Random Number Generator, the real name for a computer driven random) is much better than can be detected by human beings.

Please... unless you've written a new and innovative piece of code for random number generation... accept that SOE uses a decent RNG and has no POSSIBLE reason for NOT using the best one it can implement.
