Taarne, I have a 768 megs of RAM, a GeForce 2 card with 64 megs of RAM on it, and a 1ghz AMD processor (can't remember if it's an Athalon or the other one) in the computer I currently use to play EQ. Using EQW, and having only a single copy of EQ installed on my computer, I can run not one, not two, but as many as three accounts at once. Normally I only play one or two at a time: my main account with my 58 shaman, and sometimes I bring in my retired friend's 61 druid for extra exp. However, when I die I then have to bring in my friend's 65 cleric to rez the other two.

At any rate, the only problems I experience when using just two are as follows:

1) Whichever one is not active runs slower. The reason for this is that whichever EQ is in the background runs slower, and so can't process keeping track of where the character is properly. If someone else is watching the background character run (I live in a house with lots of computers in the same room, so I can see this kind of thing), he'll appear to teleport backwards along the path he was running about once a tick (6 secs). So he'll run forward two steps, teleport back one, and that's why the background character is slower. This is annoying because it can be hard to keep the two characters together, so if I need to navigate through difficult areas I have to run one character through at a time.

2) If I run other things in addition to EQ (winamp, internet explorer, word, outlook, etc.) I get random freezes, which happen more frequently depending on how many things I have going and how long its been since I last reset the computer. Basically, at random intervals, one of the EQ windows will lock-up, and I won't be able to see anything except the last frame (like when you zone) or control anything. Usually its prompted by some sort of lag-generating activity, such as casting a spell, using something that would generate a particle effect, etc. When this happens the screen will remain frozen for anywhere from 5 seconds to one minute. I can switch to the other character while its going on, but can't get back to controlling the first one until the screen unfreezes. As I said, this just takes time, though certain things (alt-tabbing, minimizing windows, etc.) can help speed it up. Its kinda stressful when it happens mid-fight, but since I usually keep my characters grouped, and they can see each other's health in the group bar, its not too stressful.

When I try running three characters, I get the same problems, only #2 happens more frequently. Other than this however, everything works fine, and if I really want to I can eliminate 99% of the #2 problems by just restarting and only running the 2 EQ windows and nothing else.

Now to address your situation specifically. I don't know what your computer's stats are, but if you want to run two or more instances of EQ on it you will need lots of memory and processing power. To get memory, throw as much raw RAM in as you can fit, using DDR ram if your comp supports it, and get a video card with lots of RAM. For processing power, just get a processor with a bigger number of mhz (and which chip it is matters, but that's sort of a religious debate so I'll stay out of it), and a more powerful graphics card.

As for your crashing problem, I'd give 100:1 odds that your problem is with your color depth, as Jaradis already suggested. You can either fix this yourself, by changing the color properties in control panels<display<settings<colors or by changing the line "BitsPerPixel=" in the file eqclient.ini in your everquest directory. However, Jaradis was incorrect in saying they had to be 16 bit. You can have 16 or 32 bit color depth, but the important thing is that you have to have both windows and everquest both set to the same thing.

Or if you are lazy like me, you can just let eqw do it for you, by checking the option "Fix eqclient.ini" when you run EQW. This option was only somewhat recently added, so if you haven't tried EQW in awhile it might not have been there.

As Tenjhai already said, you can get EQW from www.eqwindows.com, or if that doesn't work http://eqw.eqhackers.com

Hope this helps.
