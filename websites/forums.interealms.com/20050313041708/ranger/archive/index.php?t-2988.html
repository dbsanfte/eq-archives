First off, no, the computers do not have to be using the same OS, in fact you can have as many OS's as you want, because the computers are going to be talking to the internet gateway, and not directly to each other. Of course, if you want them to talk to each other, they need to at least be on the same stucture (i.e. in your case Windows based, which you already have).

Ok, on to the fun stuff. There are a couple ways you can set this up.

================================================

One option is to have the DSL go into one computer-1 using a NiC card (like you have now), and then computer-1 has a second NIC card that is used to connect it to computer-2 (directly NIC-to-NIC or put a hub between the 2 computers, if you have more then 2 computers then the hud is required). Each computer needs to have Microsoft's Internet Connection Sharring software installed for this to work (ICS come with ME and has an option to make a diskette to enable other computers to talk with it, even if the other computers are not on ME). With ICS, 1 computer is set to be the 'internet service provider' (this computer must have ME or Win98-2nd edition) and the other computers are set to be clients that make requests. In this scenario, computer-1 becomes the traffic cop for the other computers, and computer-1 MUST be turned on and running for anyone to talk to the internet. This puts an extra load on computer-1, because it is now processing and routing the internet requests for all of the other computers, but this is not usually a problem as long as computer-1 is fairly up-to-date computer. Also, if computer-1 burps or crashes or reboots or locks up, then ALL other computers loose their connection.

It would look like this:

DSL line-----DLS modem-----NIC1.....NIC2----------NIC
.....................................................Computer-1................Computer-2

================================================

Another option is a hub-central design, where each computer and the DSL all connect to 1 hub and they all share the DSL gateway for talking to the internet. Each computer makes requests directly to the DSL and the computers themselves are not effected by the status of the other cmputers. In this scenario, you don't have to mess with any Internet Connection Sharing, you just set windows to access the Internet via LAN instead of Dial-Up, and it automatically figures out the rest.

It would look like this:

DSL line-----DLS modem-----HUB----------NIC of Computer-1
.........................................................^------------NIC of Computer-2

With this method, the problem comes if your DSL provider has a restriction to how many computers you can connect to their DSL, or charges extra for each additional computer. If this is the case, you can buy a Router that goes between the DSL modem and the hub. The Router will watch the hub activity for any computer that makes an Internet request. The Router will then forward that request to the DSL and route the answer back to the correct computer when the reply comes back. What the router is doing is acting like a liason between the computers and the DSL. The DSL (and your provider) think it is just 1 computer because all of the requests are coming from the 1 Router ID, but the Router is actually funnalling multiple requests from all of the computers on the network.

It would look like this:

DSL line-----DLS modem-----Router-----HUB----------NIC of Computer-1
............................................................................^------------NIC of Computer-2

Many Routers actually have a hub built into them, so you don't need your own hub, the one box has both the router and hub all in one case.

So it would look like this:

DSL line-----DLS modem-----Router/HUB----------NIC of Computer-1
.......................................................................^------------NIC of Computer-2

On to the network. Personally I have my DSL line coming into a DSL modem (have had both external and internal. matters not). My P4/XP comp is connected to the modem through USB. Ran that, restarted, it found the connection, boom.

Some suggestions that I've learned. The computer with the newest OS generally has to be the computer to be the server. Older versions of an OS are unable to set up the networking correctly to work in such a set-up.

READ BEFORE POSTING: What you need to include in your post.
To begin to help users who are experiencing problems we will need some information.

All posts requesting general help should contain:

Description of the Problem: Be as exact as possible. Include things like when and where the problem/issue occurs, any error messages that are given and anything else that will explain the problem better. Be as informative as possible, you can never be too descriptive.

System Information: For many issues we will want to get more system details.(IMPORTANT!!!)(Just making sure you post info from both. The MSINFO and DXDIAG files can provide a wealth of information.

To create your DirectX Diagnostic information:

1) Click on the START menu, select RUN and type in DXDIAG and select OK.
2) When the DirectX Diagnostic tool opens, click on the SAVE ALL INFORMATION button on the lower right, next to the EXIT button.
3) Save the DxDiag.txt to your desktop or other easily found location.
4) Copy and paste the contents to your thread.

To create your MSINFO information:

1) Click on the START menu, select RUN and type MSINFO32 and select OK.
2) When the Microsoft System Information program opens, select FILE and select EXPORT.
3) Save the information as MSINFO.txt to your desktop or other easily found location.
4) This is a large file so lets copy only important parts.
1. Copy and paste from [System Information] to [DMA].
2. Copy and paste from [IRQs] to [Memory].
3. Copy and paste from [Running Tasks] to [System Hooks].

For Windows 2000

1) Click on the START menu, select RUN and type MSINFO32 and select OK.
2) When the Microsoft System Information program opens, select SYSTEM INFORMATION to highlight it and select SAVE TEXT REPORT.
3) Save the information as MSINFO.txt to your desktop or other easily found location.
4) This is a large file so let’s copy only the important parts.
1. Copy and paste from [System Information] to [DMA].
2. Copy and paste from [IRQs] to [Memory].
3. Copy and paste from [Running Tasks] to [System Hooks].

Providing the above information in the initial post will speed up the help that you will receive.

Sometimes we may ask for additional information, in the form of the DBG.TXT file, the EQCLIENT.INI file and the EVERQUEST.LOG file. All three files can be found in your EverQuest directory. EverQuest is typically in C:\Program File\EverQuest, C:\Program Files\Sony\EverQuest or C:\Program Files\Sony\EverQuest Trilogy. If the information is request from these files, simply locate them in the EverQuest directory and open them with notepad/wordpad and copy the contents to the appropriate thread.
