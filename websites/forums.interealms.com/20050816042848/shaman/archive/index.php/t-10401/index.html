I think what he means is that both the client and the server are in the know when you go below 0 hp. So for example, if you are lagged and the client thinks you went under 0, it will kill you. But the server also keeps that info, and it prefers to tell the client that you are now under 0. So you can't hack it to make your health stay up, because the server will keep updating it until you are indeed dead, but the client check is a "safety check", which also means that it may kill you, then the server will send a message saying you're at 20% health, and the client will say "sorry, I already killed him".
