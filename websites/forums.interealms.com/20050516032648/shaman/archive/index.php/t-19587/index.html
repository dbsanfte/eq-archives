Was just musing to myself a bit about using shamans as healers in raids. We have just about zero usefullness right now after the minimum number of shamans have been reached. On the other hand, upwards of 20% of your raid force MUST be clerics. Neither situation is particularly useful. Most guilds are desperately seeking more clerics, just to have enough on hand. This causes a bit of feast or famine. There's either not enough(raid night) or too many(every other night).
So, it might sound a bit out there, but would it really be that overpowering to let shaman heals over time stack? You could have a greatly reduced requirement of clerics. Got four 'extra' shaman but are short of clerics, just strap all 4 into a hot chain. The resulting healing would allow you to increase time between ch. Maybe not in very high end content, but would give a useful role to the majority of shaman out there.
Obviously this would require editing the spells and moving them to the 'bard box', but doesn't seem like that big of a challenge. It might need some kind of cap on quantity as well, though its hard to see it ever being as powerful as a single ch.
Thoughs, observations, reservations or just plain arguments, lets hear em.

Not that it matters, but a possible solution to the buff slot problem is to fix it so similar spells that have identical buff slot icons stack but occupy the same buff slot. Put a multiplier on the buff slot icon for game mechanics, so the program knows to multiply the base damage/heal by that stack multiplier. Change certain HoT/DoTs so they stack, so, for example, 4 shamans cast Q and it occupies 1 buff slot, though the game will multiply that HoT buff slot value by 4, the value of the multiplier. If one Q expires before the others, the multiplier drops to 3.

They'd have to treat and keep track of each buff separately as far as duration goes, so the game code would have to be tweaked a bit to take that into account and have it work properly.

Obviously, things like BA focus effects that change the base hp values of HoTs/DoTs would mess with this. One option would be to simply rule that if two buffs/DoTs land on the mob and the game code classifies them as "stackable", focus effect modifiers are negated. That would be one option. The other option is that they will not be stackable unless they have similar focus effect modifiers applied to them; i.e., two shamans with BA4 casting BoS. Focus effects that alter duration (SCRM. etc) wouldn't have a problem with this, for they only change duration, and the game program would have to keep track of each duration seperately anyway.

Just my idea. Comment as you will.
