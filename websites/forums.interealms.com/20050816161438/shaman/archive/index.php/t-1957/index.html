Hmm, what revision is your board? According to the web site the top end cpu's for this board (500 and 550) are only reliable with the final 2 boards.
--------
K6-2 500MHz 100x5.0 2.0 with "EO036" sticker, 2.1
VR: 3-4
CLK: CLK1=1-2 CLK2=1-2 CLK3=2-3 CLK4=1-2
SDRAM: SDRAM=2-3 NBCLK1=2-3 NBCLK2=1-2
FREQ: FREQ1=1-2 FREQ2=2-3 FREQ3=2-3
--------

Also, make sure you system is properly cooled (these things get hot), and get the latest bios revision ( http://www.fic.com.tw/techsupport/drivers/searchbios2.asp?selectedmodelname=PA-2013 ). If you are uncertain about flashing the bios BE CAREFUL, don't hose your motherboard, take it slow and read up on how to do it correctly. Are you running McAfee, by any chance? For a while there one of the virus updates was causing blue screens. Couple other things to try after you check ram and cooling: Update all drivers (including getting newer VIA busmastering drivers), kill everything running in the taskbar, turn off power management in the BIOS. Something else I came across while looking around:
------
If you are running with an AMD chip, you should also note that there have been quite a number of reports of a faulty L1 cache (the cache on the chip). Go into your computer's BIOS (see your handbook for instructions), and disable the Level 1 cache. If this resolves your problems, consider buying a new CPU.
AMD & Temperature
We've received information that running the (older) AMD K6-2 chips over 60ºC (140ºF) will produce a FATAL EXCEPTION OE ...0028:C0005338 IN VXD VMM(01)+00004338 (or similar) errors. So if you are running a K6-2 chip, it's another area to check!
----------
