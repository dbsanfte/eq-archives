I just got off a chat with Tech Support. I have an on-board card for my Celeron -- an Intel 82810 E.

My problem has been the "EQ must quit" error after the server select screen. Now I *DIDN'T* get this error two nights recently when I reloaded the ENTIRE game. But then the next time I tried it crashed again.

So the tech support guy said I need to delete the eqclient file from my EQ / EQ Trilogy directory. He said "since that video card isn't supported" when the game detects that eqclient file it causes the system to crash. He seemed to think this was a workaround that would work, he said the reason my system operated those two times recently was because there hadn't been an eqclient file after reloading.

Anyway, hope this is helpful. I'm at work so I can't test it now, I'll have to wait 'til tonight...

>>This time I tried Vinny2003s´suggestions in >>another thread:

"1. open your eqclient.ini file and look under video for windowedmode=true (or false) make sure it says false.
2. Next look for videomodebits= make sure it equals 16 or less.
3. Make sure your computer is a peak virtual performance. Its in the settings window.
start-settings-control panel-system-advanced-performance options: Set maximize performance for applications.
4. oh and make sure your eqclient.ini has the line
EQGFX_DX8.DLL in it at the top."

SUCCESS!!!!!!! These changes worked for me last night. I notice the screen resolution and "look" is a little less lush than before, because I reduced the graphics-enhancing settings a bit, but it works! I'm going to tweak the look of things tonight.
