** Bug Fixes **

- Fixed a bug that allowed items to be placed in the wrong equipment slots
- Fixed a problem that some folks had with polygons and clipping using the Matrox Parhelia card


** Spells **

- Changed Luclin port spells to be usable from anywhere
- Added new Druid and Wizard transportation spells for locations in Luclin


** Items **

- Removed the cap for items that granted bonus mana
- Bricks of Black Acrylia are now stackable
- Plains Pebbles are now stackable
- Casserole Dish Sketches are now stackable
- Yttrium Sleeves and Leggings now have a plate armor appearance. They are also no drop


** New User Interface **

- New Interface Window: The Compass! Press Alt-D to activate it. The higher your Sense Heading skill is, the more responsive and accurate the compass is

- Fixed the fatigue percentage Label to show 100% when you are not fatigued at all
- Clicking to add new items to a stack in your inventory will now stack those items into the inventory rather than onto the cursor
- Fixed a bug that was causing some people to lose items when looting too quickly
- Tracking window no longer shifts to the top every time an NPC dies
- Training window no longer only shows the first digit of the amount needed to pay for training
- Fixed the bug that was causing left clicking during mouselook to target
- The 'hits mode' menu and filtering for it is done, including the options for normal, abbreviated and number/miss only for the six main melee filters

- UI Skin Creators: We added the following EQType labels:
71 = Current AA pts available to spend
72 = Current exp % being assigned to AA
73 = Last Name of the player

- Alternate Skin Users: You may need to return to the default interface until the skin that you are using has been updated to work with the latest client. The addition of the new Compass window as well as some other changes may cause problems with skins for older patches of EverQuest. If you experience odd interface behavior or crashes, please try /LOADSKIN DEFAULT to return to the default skin.


** Bazaar **

- The Bazaar Search Window is resizable (added in an earlier patch)
- PCs can't beg or pickpocket from NPC's and PC's in the bazaar (added in an earlier patch)
- The Trader drop down box is now sorted alphabetically


** Alternate Advancement **

- Priests and intelligence casters now have the Combat Stability and Natural Durability Alternate Advancement abilities in their Archetype


** Miscellaneous **

- Reduced the faction hit for the final turn in on the Wizard Epic quest
- Implemented key rings. Certain keys will be placed on your key ring when you use them (like clicking on the appropriate doors or locks). The keyring is a part of the character, not a space in inventory. To find out what keys you have just type /key and the game will give you a list
- Moved the safe point in Marus Seru to a somewhat safer spot

Straight from Celt's great AA thread on Rangers Glade

Natural Durability
(2/4/6) This ability increases maximum hitpoints by 2, 5, and 10 percent, based off your stamina buffs and base hits, NOT hitpoint buffs. This of course includes working off of Riotous Health, Marvelous Visions, etc. Maxed, this skill will give you an additional 10 percent of your unbuffed hitpoints, as well as increase your hp per stamina point to 4.6, 4.74 with Physical Enhancement in Class, instead of 4.2. Well worth the point expenditure if you need the hits, a normal ish 2500 hp unbuffed ranger still gets 250 hp from this skill maxed..a noticable, sizable MEASURABLE improvement. Max this fairly early in the grand scheme for all builds Hitpoints are absolute, this skill alone, maxed, gives EVERY level 60 ranger 158 HP at the very least, with 0 gear/buffs at all, and every +100 hp item will actually be +110 hp, +112 hp with Physical Enhancement in Class. Also of note, if you plan on getting Physical Enhancement, you MUST have the first level of Natural Durability to get any effect on your hitpoints from Physical Enhancement.


Combat Stability
(2/4/6) This skill increases our chance to mitigate damage when we are hit, by 2, 5, and 10 percent respectively. This doesn't mean damage percentage, it means the chane that we will meitigate the damage down to something smaller, like a fungus shroom hitting us for 35s a lot instead of 130's. I have this abiltiy maxed, and can speak from experience that it DOES increase our tanking potential by quite a bit, ESPECIALLY if you subscribe to the mass hitpoints/medium ac school, for whatever reason. My ac gear isnt great, so I decided if I am gonna get hit, I want it to deal less and have my 38/55 point self buffed damage shield eat them up. The change in tanking is *very* noticeable at 1250 ac/4k ish selfbuffed hitpoints maxed out, and does show noticable improves from the second and third tiers especially. This skill works very very well in evening the gap between our tanking and that of a well equipped Paladin/SK without any AA, and helps ease the pain of watching them outdamage our epic wielding brothers when they have a vulak axe or similar large devastating 2hander. It is necessary to have this skill at the first tier if you wish to see any improvements in our mitigation from Physical Enhancement, and also works hand in hand with Combat Agility in improving our overall tankability.

Absor posted this advice somewhere:

Remember, the key here is that if the interface can’t find a file in the custom folder, it will look for it in the default folder. But if it can find the file, even if it’s an old version of the file, it will try to use it. We do this because the newest file might not be the file you want to use (the older file might be a customized one). This could cause trouble, however, if we update a file in the default folder that also exists in your custom folder. In a case like that, you’ll continue to use the old file - and that might cause your game to crash.

So here’s what we suggest.

If you are using a customized interface, make sure that only the files you have modified are in the folder for that skin. Delete all other files. That way the game can use new versions of files that you have not modified out of the default folder. This should help you avoid many of the issues that come up when we update the UI files.

We also wanted to let you know that we understand how updates to the interface can frustrate those of you that have customized it. We will endeavor to make it clear what files we’ve changed for the interface in our patch messages in the future.

For example, this morning a few files changed in for the interface, making it so that some people (and in some cases a specific character) could not log in. The files we changed were:

- EQUI.xml

- EQUI_Animations.xml

- EQUI_BazaarSearchWnd.xml

- EQUI_BazaarWnd.xml

- EQUI_OptionsWindow.xml

- window_pieces04.tga

If you have those files in your custom UI folder and you haven’t edited them yourself, deleting them from that folder should fix things.
