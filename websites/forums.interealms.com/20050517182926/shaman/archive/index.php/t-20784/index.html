Zorash
Greetings,

Personally... I would love a situation where the entire encounter retuned itself to fit you.

I'm with you Caanus, in the abstract. It would be GREAT to be able to have events tune themselves to your group, allowing you to bring any 6 people you wanted, and still have fun.

My opposition to this suggestion wasn't for that reason. I think that part of it would be wonderful. The problem I have with it is that it doesn't actually FIX the problem. It merely works around it through some inventive coding.

Now, for things such as trials, or any instanced zone for that matter, if done right, this fix could work. Mind you, to make it work, the devs would basically have to code all the mobs for each zone many, many times. If you want all the possible combinations, you're talking about having NINETY ((6 members per group) X (15 possible classes) = 90) different versions of each zone. This means 90 different versions of all the mobs in the zone, each tuned slightly differently to allow for the different group combinations. (pally, sk, warrior, rogue, ranger, cleric? how about pally, pally, pally, rogue, enchanter, druid?...you get the picture..)

So, as a "fix", this would require the devs to code up (and yes, I realize it would probably not be nearly this many versions) 90 versions of each of the mobs in the instanced zones of the current, and future expansions, all in an attempt to make sure that the classes can all contribute well in their current state.

That, in and of itself, is a big enough problem. But here's an even larger one: What happens in the non-instanced zones?

The answer? Nothing. If you base the "fix" for classes being unbalanced around the content, meaning the content is actively tuned when each group zones into an instanced zone, based on the makeup of the group, then you have done nothing to better that class in and of itself. This means that the classes that were having troubles before the mob-tuning "fix" are going to have the EXACT SAME problems AFTER the mob-tuning "fix", in any zone that isn't A.) Instanced and B.) Coded in multiple versions to allow for tuning based on group makeup.

So is every new zone in every expansion from now until the end of the game going to be instanced? And are those instanced zones ALL going to have up to 90 versions created each, to allow for this tuning to properly work based on group compilation? I highly doubt it. If these stipulations are not both true, then the "fix" will fall by the wayside, and the classes are left unchanged, and still just as unbalanced as before.

Like I said, I like the idea, in the abstract. It would be my dream as well to be able to take any 5 friends and jump into a zone and have some fun. The fact is, it just doesn't solve the problems classes are having. It skirts the issues, which, in the long run, doesn't do anybody any good. Not only does it not fix the actual problem, but it would probably cause about 10 times more coding work for the devs than an actual fix would, while still requiring them to take an in depth look at class strengths/weaknesses to allow them to properly tune the mobs of the different zones to the classes makeup of each group.

Anyway, I do appreciate the link, and the thoughts. It's good to think about these things and see what different types of fixes we can come up with. I just don't think, personally, that this is what we need. We (and the other classes) need someone to actually look at, and fix class balancing, allowing all classes to at least contribute a worthwhile amount to current and future content.

If they want to change the pace of the game to blindingly fast, 30 second encounters, with mobs with 3000dps...fine, just equip us to help against those mobs, that's all I ask. :)

Be well,
