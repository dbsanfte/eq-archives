There's a nifty real-time log parser I came across, called EQ Watcher. It features sound effects... such as, reading any channel or tell to you, or giving any selected sound whenever there's action in a chat.

It requires some customization, but it's pretty handy. I have it set up to read tells only, and give sounds for only one channel. It's also set up to handle all chat channels as one - the joinable chat, has seperate stuff for ooc, auc, shout, say, & anything else I've missed because it's 5:30 am. :p But you can set up a trigger to recognize individual channels, and do something then.

For example, I have mine set up to beep at me whenever anyone says anything in the server shaman channel (shaman:shaman, for anyone on CT) - but it's a small crowd, so not usually a lot of action. I have the rest turned off by default, with only tells being read to me.

Along with seperate chat colors, and spreading chats around to seperate windows, it helps a lot.
