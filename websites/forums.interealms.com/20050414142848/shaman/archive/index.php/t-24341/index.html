Honestly getting EQ to run under linux is WAY easier than it was a few years ago..

If you go with Cedega (recomended as perfomance in vanilla wine is ..lackluster) just save yourself the hassles and let the patcher download all of it. I have found that even though you load the expansion fom disk the patcher still goes and downloads so much that its hardly worth spending the time poping the cds in and out (providing you have a decent connection that is)


If you use the Gui frontend for Cedega, Point2Play, and want to get around the patcher just put the following in your ~/.point2play/EQ/games.ini file. It will give you 2 icons one runs the patcher and the other does not.

____________

[EverQuest]
path=C:/Program Files/Sony/EverQuest/EverQuest.exe
icon=/home/syowr/.point2play/EQ//icons/EverQuest.xpm
workdir=/home/syowr/.point2play/EQ//c_drive/Program Files/Sony/EverQuest
config=/home/syowr/.point2play/EQ/EverQuest.config

[EQ No-Patch]
path=C:/Program Files/Sony/EverQuest/"eqgame.exe patchme"
icon=/home/syowr/.point2play/EQ//icons/EverQuest.xpm
workdir=/home/syowr/.point2play/EQ//c_drive/Program Files/Sony/EverQuest
config=/home/syowr/.point2play/EQ/EQ No-Patch.config

____________

Angryor
I'm going to assume you are using lilo as a boot loader. If you want to fix your problem you can always do the following.

su -
/sbin/fdisk -l /dev/hda (That's lower case L for list)

Find the drive with the ntfs/win95, FAT32 partition
(note that partition)

vi /etc/lilo.conf

add the following entry to the end of the lilo.conf file.

other=/dev/hdaX (where x = the drive you found doing fdisk -l)
optional
label=Windows

save your file, then run lilo.

This is going from Mandrake 9.x though so may vary, I'm pretty sure it's the same.

If you are using grub or even menu.lst (I think menu.lst is Suse only though /shrug) post here and I can try to help.
