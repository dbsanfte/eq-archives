Gilador
I think you have misunderstood the 'max damage' field in the weapon procs list. I don't think I have documented it too clearly except in some EQ Companion ezboard threads when people have asked about it.

All you need to do is make sure it is set to a value that is greater than the initial DD of the DoT will ever do and you should be set.

My shaman is only 63 so I am using Bane of Nife myself all the time. The initial damage is 110 hit points. If I had a +0% - 30% focus item, I would alter the max damage to be (110*1.30) = 143.

The reason EQ Companion needs to know the max damage is because it is possible to have multiple non-melee messages right next to the weapon proc/spell landing message. This happens particularly often if you have multiple damage shields on. The non-melee message and the weapon proc/spell landing message are not necessarily next to each other either. The damage shield messages can get in between.

EQ Companion keeps an internal cache of the last 3 seconds (current second and 2 past seconds) non-melee messages and when it sees the phrase you have entered in the weapon procs list, it searches through that cache. First it looks for an exact match of the damage amount. If it cannot find one, it will look for the closest to, but not greater than the damage amount you entered in the max damage field.

This was originally for partial resists but these days it works great with all the new focus effects, etc. that can increase the damage amount.

So just as long as your max damage is greater than the largest amount of damage the spell can ever make, it should work OK.

The reason this is not working for you is because of version 0.79 which cannot handle the new format non-melee messages.

And even more if you want to include the negative damages for when you have Saryrn's nice effect on you.
