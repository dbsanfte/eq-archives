Macnair
Stacking Shaman healing.

If there are three shaman on the raid, what do they do?

If three clerics work at healing the tank, casting their Complete Heal spell with a 4 second interval, then they each contribute massive healing toward the raid's success.

A shaman's best healing is done with Heal over Time spells. If one shaman casts a HoT to supplement the healing on the tank, he can make a small contribution toward the raid's success. But the other two shaman cannot help. Their HoT spell does not take hold.

A possible solution would be to allow Heal over Time spells to stack. Each would take up a buff slot, so it would be necessary for the tank to leave some buff slots open. This leads to tactical decision making: what buffs to live without in order to have HoT buff slots available.

This would take some coding, but this should not pose a great problem. Multiple copies of the same DoT can run on the mob. The same coding would allow multiple copies of the same HoT to run on the tank.

Three shaman casting Quiescence on a tank would provide 1050 healing per tick. This would hardly be overpowering, with slow-immune and slow-mitigating mobs that can triple or quad for 1000 per swing. But 1050 per tick is supplemental healing that can make a difference. It would mean that the second and third shaman can contribute genuine healing toward the raid's success.

That takes care of making the spell better. Shaman would be able to land their slow faster, and it would reliably land without debuffing.The other problem is mitigation.

The core problem of mitigation is that it brings slows closer together. By making mitigation multiplicative, the gap between the various classes has narrowed considerably. With a partially effective slow (assumed 50% mitigation), the narrow 5% margin we have over enchanters narrows to 2.5%, and the advantage over beastlords is only 5%.

With "partially effective" slow mitigation:

Shaman: 75% x 0.5 = 37.5%
Enchanter: 70% x 0.5 = 35%
Beastlord: 65% x 0.5 = 32.5%
Bard: 52% x 0.5 = 26%

In my mind, the best way to make shaman slows preferable in a meaningful fashion is to allow them to bypass one level of mitigation on everything but bossmobs. This is for shaman slows ONLY. Please note that single group named do NOT count as bossmobs. Example: Chavoor, a Vxed named that currectly makes slows "slightly effective" for all classes would get a "partially effective" slow from a shaman.

Summary:

1) Increase resist mod on Balance of the Nihil to -55, and reduce recast time to 3 sec.

2) Allow shaman slows, and ONLY shaman slows, to bypass one level of mitigation on everything but bossmobs

SOE needs to learn something here: The game is scaling in a way no one saw coming. The newsflash, though, is that this always happens in role-playing type games. Anyone who's played D&D knows this game. The 20th level party annhialates everything in their path. A good GM resorts to some trickery - DS on the bad guy, maybe a deathguard/contingency/teleport, etc. A bad GM just scales the numbers, making the bad guy 40th level, with a -40 AC, 800 HP, innate CH, dual wielding Rod of Flailing with oil of bluntness + perma-hasted.
