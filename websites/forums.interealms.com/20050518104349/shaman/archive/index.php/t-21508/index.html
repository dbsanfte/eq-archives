I know there is a tongue in cheek post going on about the changes to login, but I wanted to start a thread on something that has been bothering me for a while, to get feedback, or a solution from anyone here if possible.

When I log in i have to sit through the file checking routine for a few minutes. It is slow, and totally unnecessary. We shouldn't have to go through it at all.... honestly....

Instead of checking a whole plethora of files to see if anything has been changed, can't SoE just have a single file that they use for patching. Anytime they need to force an upload they can change the size of the file. Then, when we log in the patcher can just compare the size, and the date/timestamp of that 1 file. If they differ then upload away.

I don't like having to sit through the checker for 5 minutes after going LD due to coding errors in a zone that hasn't been fixed yet, just to see it resolve to the play button....... yes, strangely enough I don't need to upload anything since the last time I ran the patcher (20 minutes prior)...

This would speed things up drastically. I don't run eqwindows because my system can't hack it, and I don't want to have to run some third party code to get round the patcher, or make unofficial changes to it.

I would just like to see it coded efficiently by SoE. Anyone else have any thoughts?
