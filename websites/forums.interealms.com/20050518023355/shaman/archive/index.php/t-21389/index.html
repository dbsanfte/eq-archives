Shaman slowed = 0.75 * 0.5 = 0.375 overall
Enchanter slowed = 0.70 * 0.5 = 0.35 overall
I agree that this is marginal.

A new system, aimed at having roughly the same slowability... Mitigation 0.40
Shaman slowed = 0.75 - 0.40 = 0.35
Enchanter slowed = 0.70 - 0.40 = 0.30

I agree 100%. I believe this is the way that slow mitigation should have been implemented all along. Using multiplication to calculate slow mitigation only multiplies the problem (how novel of a concept is that). Subtracting the mitigation from the original slow percentage maintains the class balance pertaining to slows. Coders could then design the mobs around what the maximum percentage they wanted the mob slowed to be. If you feel a mob should only be able to be 25% slowed at most, then make them mitigate 50 (through subtraction). At that point Enchanters would only slow by 20% and Beastlords by 15%. This maintains the 5% gap.
