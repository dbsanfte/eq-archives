On the topic of an open source OS, I would be inclined to agree that the most secure OS will be an open source one. Open source != Linux, of course, but in principle an open source OS at least has the potential to be properly secure - be it Linux, a BSD unix variant, or something thoroughly unforeseen :)

Of course, there are other factors to be considered. For one, just because an OS is open source doesn't make it a secure OS. Also, the same principle that applies to, say, a cryptographic algorithm doesn't necessarily apply to an operating system; the security of a crypto algorithm comes from withstanding years, even decades of concerted efforts to find weaknesses in it by the brightest minds in the academia and in the military. Applying the same principle to an OS would mean that in order for it to be secure we'd be using stale, ten-year-old technology.

Sure, it'd be secure but you could forget about MPEG-4 movies and MP3 players..
