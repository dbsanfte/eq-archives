This was an interesting perspective that someone posted in another discussion, long ago, but it's true any time someone is talking about changes to a class.

There are two ways to argue for/against an ability for a class - the "roleplaying way" and the "gameplay way". The Roleplaying argument is what Iceborn is making. "Wouldn't it make sense if Shaman could do xxx based on what the class is supposed to be?"

The Gameplay argument is "Shaman need xxx because of this gameplay element." Gameplay elements would be things like being able to be interdependent with other classes, how they solo, how they group, how they perform on raids, etc.

If you're trying to get something added to the class, you will NEVER WIN AN ARGUMENT with SOE using the roleplaying argument. NEVER.

Their primary task is to balance gameplay issues. You can argue back and forth how good/bad a job they are doing, but they are not going to give a major new ability to a class just because the class "ought" to be able to do it based on its description.

The roleplay argument could be used to argue for just about every cleric, druid, and necro skill. Come up with a solid GAMEPLAY reason for Shaman to have something and you might have better luck.
