Another possible problem:

I don't think it's right to use character entity references in links. For example, the "edit" link on a post looks something like this:
<pre>
&lt;a href="editpost.php?do=editpost<b>&amp;amp;</b>postid=128689"&gt;
</pre>
I *think* that's wrong, and more to the point it confuses my browser :) If the <b>&amp;amp;</b> is changed to a plain <b>&amp;</b> then all works well.

My reading of the HTML 4 spec is that the "link" part of an anchor tag is of type URI, whereas you're only allowed to use character entities in CDATA types. I could be wrong here, I'm no HTML expert.
