If you are running multiple simultaneous instances of EQ on one machine, it runs much better if each instance is running from a seperate directory.

I tried that for that reason, Difference is unnoticeable (at least on my machine) the slow-down is from sony's implementation of the windowed mode. If you really want to see an improvement that validates this, use eqwindows. THAT is an improvement.

Putting the files in different directories only really helps when EQ does WRITING to the files, reading doesnt matter much. When writing, the files are locked so that no other writing can take place at the exact same time (that would be bad), however 2 programs can read the same file (DLL files are used by many programs at once) the OS queues the reads and will deliver the requested data as fast as the hard-drive can spit it out. So when reading data, you are limited by hard drive speed, regardless of whether the files are the same or 2 different files in different places the hard drive can only read one at a time period (because obviously the read head on the drive can only be in one place at any one time).

The faster your hard-drive RPM's are the faster it can handle data output, and that is your limit.. Placing files in different directories still limits you to your hard-drive output. With the only exeption being file WRITES which must be locked and then unlocked before another write can occur. EQ does very little writing.

If you want better performance, save your hard drive space, have only one eq directory therefore removing the problem this thread is about, and install eqwindows which uses built in OS-Hooks to control window mode rather than software controlled method that Sony uses.

Hope this wasnt too confusing...
