For the past few weeks I've been experiencing seemingly random crashes to the desktop while using the new UI within EQ. Switching back to the older UI seemed to solve this problem for a while, but then the crashes started up again.

In order to resolve this issue I upgraded from:

AMD 1900+ processor
Soyo Dragon Motherboard
SB Audigy sound card
WindowsXP
512mb DDR Ram

to:

Intel 2Ghz processor
ASUS Motherboard
onboard audio
WindowsXP
1024mb DD Ram

I expected this to resolve my problem, but the crashes started up again even with a fresh installation. There are no third party programs running in the background, or installed.

Logic would dictate the crashing problem is graphics card related. The problem is either: graphics card settings in the system bios, or the card itself. Since I am using reference drivers from nVidia, I find it unlikely the drivers are at fault.

Being a bit of a graphics card addict, I try tweaking for the best performance. I believe the setting that is causing the problem on both systems is the AGP Graphics Aperture setting. I had it set to 256mb when installing the board, and have since scaled it back to 64mb with no crashes as of yet.

Needless to say this is an expensive lesson, but I hope this helps those of you out there with similar issues.
