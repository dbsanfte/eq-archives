Having spent way too many hours in BoT along the way ...

... here's how I think it works.

When a given mob is spawned, a number of variables are assigned. Some mobs are always the same level; others have a range, and so a level will be assigned to that particular mob. It will have a specific number of hitpoints assigned, based on the range for its level. It will have STR and DEX values assigned, within the range defined for this mob. And it will have resist values assigned as well, within the range defined for this mob.

Which is a long way of saying: all mobs are not created equal.

By random chance, then, you can end up with a mob that has very high resists in some or all areas. Which can make it very hard to slow that particular mob.

After you kill it, the next one that comes along will again be randomly generated, with a new set of stats. So it could be just as tough; or it could be considerably easier.

I've had strings where slow would always land on BoT names in the first or second cast. And other instances -- including with your buddy Gaukr there -- where I alternately cast both slows for the entire fight, and never landed it.

In the end, we concluded that to take on a named reliably, we needed to have sufficient healing available to beat it when it was unslowed. If we got it slowed, great; but it was unrealistic to count on it. So we had to plan on having enough healing there to keep the tank alive with the mob hitting at full speed.
