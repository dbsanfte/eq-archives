I've seen a lot of comments in 2box threads concerning how effective each character is when being run 2box (or 2window). Since the opinions are so vastly different about how efficient 2boxing can be, I wanted to throw some numbers at it. It may be meaningless babble, but it was fun to write :)

Baseline Assumptions:
I use 1 computer, so I'll go with my own setup. Others can comment if theirs would provide different results. I use 1 computer, but run with WinEQ in tile mode so I can see all of every window. I use a custom 640x480 UI so that I can see all the information I want without severe tunnel vision. I have a hot key specific to each window.

General 2boxing performance:
When playing, few classes are doing something ALL the time. Lets consider a shaman chain casting CannIV. The casting time is 2.5 with a recast time of 2.5. Even if the casting time is sped up 50% this is still 3.75 seconds from beginning of cast till next possible cast (assuming no fizzle, and no emergency that requires changing actions).

3 seconds is easily enough time to switch windows, perform a single simple task such as adjusting the meele positioning, cast a spell, etc and return in time to keep the shaman busy. Under these circumstances, an attentive 2boxer can approach 100% efficiency.

Situations that degrade performance:
The key assumption for the optimum performance is that actions can be staggered and the other character is run while waiting on the others action to complete. There are key situations where this just isn't true such as a sprint to the zone line, or if you have multiple mobs attacking different characters. There are also some classes that tend to do things while casting, such as bards and sometimes enchanters (can target the next mob in a crowd while casting). As a result, these classes tend to be much more difficult to 2box. The same holds true for trying to 2box two meele type classes. Since both may need to adjust to mob push at the same time, efficiency can take a hit.

2box advantages:
There are situations where a 2boxed set of characters will have advantages over non 2boxed characters. These situations usually involve an advantage based on communication, timing or coordination of the activities between the two characters. When I 2box my chanter/cleric, it is quite easy to time a mez to land on the soon-to-be-ex-pet just after the cleric's ITU lands on the chanter. I would not attempt to get the timing as precise if I had to use another player to cast the invis spell. The 2box advantage is especially nice if there is some overlap in the capabilities of the two characters. When I would 2box a paladin/shaman combo, I could patch heal with either character depending upon the needs of the situation. When I 2box a chanter/cleric, I had the option to pull with either character depending on the situation. In raids, I sometimes would request that my two characters be in separate groups so I could see all 12 life bars for patch healing.

3box efficiency:
Since I've gotten my 3rd account, I've noticed that 3boxing seemed to be much harder than 2boxing. Part of this may be me pushing the limits on my computer, but the numbers also show that effective 3boxing is twice as hard as 2boxing. Using the shaman example, as soon as I hit the cannIV spell button, I have a little over 3 seconds to take care of the other character(s). If there is one character then all 3 seconds can be spent on that character. With 3boxing, the time is split between the two characters... Perhaps with practice I will improve, but for now I don't find myself able to provide consistent performance on a character with 1.5 second bursts of control.

Conclusion:
I don't think these numbers are enough to quantify a specific efficiency number, but if people really want it I could copy the old shaman healing charts, relabel the axis and declare that clerics are broken because they are so easy to 2box due to the 10 second casting time of CH :)

HK for Malx and then Slow on 1 button. I have trained myself to watch for fizzles and resists on this. I usually use this on mobs I have had very few resists from in the past.

Double up the casts to help counter fizzles:

/cast malos
/pause XXX, /cast malos
/cast slow
/cast slow
/cast slow

If the spell doesn't fizzle, you'll just get a "you can't cast this spell yet" type message. If it does fizzle, you get a second or third shot at it.

Resists you are stuffed on though and have to do manually. Still, if you have group spell messages on you can look for them on another char. With practice I imagine you can get the timing down fairly well to realise when a slow message should have appeared but didn't due to resists.

Another possibly useful tool is the log file to speach conversion software. I don't recall the name, its really useful for people with bad eyesight but I can see for multi-boxing it might be handy too. I believe you can setup quite complex rules for it with different triggers to make it say different things.
