I've had problems in the past going link dead, often in the Bazaar, and found this little trick that seems to prevent it. At times, you can do a /who command to save yourself from going link dead, but this trick seems to take care of all that for you.

After you enter the game, go into windowed mode and open a command prompt window. Basically you are going to Ping your own internal address, which seems to keep your network connection alive in certain of these link death situations.

Your internal address is 127.0.0.1, and the ping command has a -t switch to continually run the ping. You can also use the -L switch to set the size of the packet, to keep it small.

So, the command I use at the command prompt is:

ping -t -L 8 127.0.0.1

While running behind your EQ, the window looks like this. I hope this helps some of you out there, it seems like it's been working pretty well for me.

There are several game bugs running around that become really significant due to the number of people in the bazaar, and the ability of certain spells to knock people offline.
When setting up your trader for the night, do the following:
Minimize clip plane, set opacity to 0%, /showspelleffects off
face a wall or look at the floor
turn off all chat - ooc, shout, auction, guild, all chat channels
set all spells to yourself or group
make sure badword filter is off (/filter badword)

These settings get my trader through the night more often.

I find it REALLY hard to believe that pinging the local loopback address (127.0.0.1) will have any effect whatsoever on the game. The loopback address is optimized in the network stack so that no data ever goes out on the wire.

I can believe, maybe, that pinging some external address might have an effect, but it's far-fetched since plenty of network traffic is taking place just on your Everquest connection.

You can get more sophisticated ping utilities that allow you to select the time between pings, btw, which would help your bandwidth a lot more than reducing the payload size. I use PingPlotter, it's very nice shareware and graphically shows the ping delay to ALL the intermediate steps in the route to your destination. http://www.pingplotter.com .
