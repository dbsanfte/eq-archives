Welcome to the home page for EverLogs (previously known as EQChar Info). Take a look at What's New in our web.

EverLogs is a character analyser and statistics program for Everquest.   Information is collected from the log file generated by Everquest.  I am designing this program to be as complete as possible and include all possible data that you would like to see about your time in Everquest.

Go to Features to check out a few of the things that EverLogs can do for you.

This program is still evolving and I appreciate any and all feedback from the Everquest community.  If you have any comments or suggestions, please send them to me at blood@gralin.com

Febuary 4, 2000

Here is the big news item.  EQCharInfo has been combined with the program set EverTools and replaced the existing EverLogs program which they were starting to develop.   I will continue to develop EverLogs (previously known as EQCharInfo) and will keep this site updated with info about the log analyzer program.
I have sent the latest version of the program (EverLogs 2.0) off to hydracomp to be included with the next release of EverTools.  He has an autoupdated built into it, so you can just get the latest version of EverTools, and as soon as he has the distrobuition sites set up with the new log program, it should automatically be copied to your system.
There have been many changes and additions to version 2.0, some of which are:
Fixed reload bug so that when you reload the old log files, it will not destroy your backups in the process.
I now scan the log file when loading it for your character info, meaning that if there if you did a /who which includes your character, I will pull the race and class info from that and not have to ask you for the info.
I am now tracking the level for different sessions, where possible.
There are several other improvents and additions.
I have started adding a list of people you interact with and being able to display chat logs (tell, ooc, say, guildsay, etc.)

January 24, 2000

I had some problems pointed out with my previous method of fixing the problem of looting your own corpse, so I am now trying another method which I think should work pretty well.   Upon dying, I will exclude the next loot message that I see that doesn't follow an experience message.  This means that your money from your own corpse will not appear included in the loot total as long as you do not loot anything you didn't kill and get experience for (i.e. green mobs).
I have had a few ask, so for those were wondering, I play a wizard named Shadowwalker on the Karana server.

January 6, 2000

I will have a new version out soon.
One note on the subject of loot reported from looting your own corpse.  I have had many people point out this problem with the program.  The problem is that the text in the log file is exactly the same as when you loot a mob, so there is no way to distiguish your own corpse from a mob when you loot.  I am going to try something in the next version to see how it works out.  I will have it make the assumption that people generally don't kill and loot anything until after they have retrieved their corpse, so I will exclude the next loot after they die from the loot total.  It is the only real way to fix the "bug" because of the ambiguity of the log file.

October 28, 1999

Version 1.2 has been released.  Besides some nice new features included in this version, I also fixed the bug where it would not reload data from previous sessions.   A side effect of this fix is it will now only ask race and class for a character the first time it sees that character rather than every time you run the program.
Changes and Additions to Version 1.2:
Added bash damage
Further merchant crossreferencing is done, which will help determine what zone a merchant is in, rather than sometimes showing "Unknown zone"
Individual monster info was added, so you can see each mob that you engaged and what the maximum and average damage that monster does to you, as well as total damage.
Changed percentages to include a fractional amount.  This was especially needed in critial hits, as they are usually less than 2%, so you can now see something like 1.35%
I have added a horizontal scroll bar to the lists, so you can now see items that were too long to be seen before.
I have started adding bard song support.   At this point, it will show you the number of fizzles.
Steal success rate info has been added.   You can now see how often steal works as well as what items you have stolen.
I have updated the Schedule page showing some of the feature ideas I have received so far that I am working on adding.

October 12, 1999

Version 1.1 has been released.  I have not implemented a lot of the features that I want to yet, but have included some in this version.  The main change for this version is to fix the problem where attack damage was not showing of for monks and rogues.
Changes and Additions to Version 1.1:
Added monk attacks (Punching and Eagle Strike) and rogue attacks (Backstab)
Moved fight defense info to the Weapons tab (Now Fighting tab) and added Block, Parry, and Riposte.
Added info to the Monsters tab.
Added skill details for Steal (pick pocket), Disarm, Sense Traps, and Intimidation (Instill Fear), and Trade skill trivial attempts.
Fixed bugs relating to initial load  of the log file and restoring from backups.
Thanks for all the input I have received so far.  Keep it up.  I still need race and class information to fully support all class information that people would like too see.  If there is something you want to see about your class that I have not included yet, please send me a sample log file or information about what you want  to see.

October 10, 1999

Request for Information:  I would like to be able to fully support all races and classes in this program.  It is not possible for me to full play all races and classes though to the point where they can use all their possible skills.  So I would like to request that people send me sample log files for their characters.  With the log file I will need a little info so I can put things together.
1) I need to know what skill, attack, or other info that is in your log file that you used.
2) Tell me basically what EQ tells you when you succeed or fail at something.  For example, with the rogues pickpocket, a success says "You have stolen 1 platinum." and a failure says "Your attempt at stealing was unsuccessful."

October 8, 1999

It has been brought to my attention that I have left out the monk's hand-to-hand attacks and the rogue's backstab from the weapons section.  I have finished adding these, and I am working on a few other things as well.  I will release a update sometime next week so the monks and rogues can correctly view their attack stats.
