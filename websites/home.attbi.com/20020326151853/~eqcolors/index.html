Monday, February 4th, 2002

I'd like to take this time to announce a new feature named SpeedEQ, what this feature does, is analyze EverQuest in the background, it monitors RAM and Hardrive usage, login and zone load times, it even enumerates threads and their pull (processor usage), then generates a very thorough report with performance charts, it then suggests settings to make EverQuest run optimally for your particular computer, this includes changes to operating system settings, bios, and various other things on all Windows 98se/2000/XP based systems, this feature is a "must-have" for any serious EverQuest player.

It's official, I've made my decisions on several issues regarding the program:

1.) EQcolors will now be known as TweakEQ; the functionality of this program has gone far beyond a simple eqclient.ini editor, it has many new advanced system-level features, that you would only expect to find in high-quality commercial configuration tools.

2.) The program will be initially privatized, meaning that I will not release it to the public when it's finished, "which is very soon pending the Bazaar UI addition in EverQuest", I simply haven't decided what I want to do with the program yet, I know it's being used by a rather large population of EverQuest players, but I get very little feedback from them.

Wednesday, January 2nd, 2002

Happy New Years everybody! my attention has been very divided with multiple side-projects lately, but the more I play with EQcolors 3.0, the more I envy it's slick interface and functionality, you can safely assume by what I've said here, that the program is in functional condition right now, I'm at a stage where I'm playing more with the program than working on it, I really need to stop that and start focusing on finishing the program!

If you want to see one of the "other" side-projects that's been eating up my sparetime, visit The Brell Serilis Nexus, also I need to finish the design for this site as well, what you see here was thrown together with one of my renderings, I was going to save that for the launch of EQcolors 3.0, so what do you think, should I just release it as a Beta? or go through it with a fine tooth comb to make sure it's absolutely stable like I did with version 2.0? ;)

Wednesday, December 19th, 2001

I'm currently working on the 'Optimizations' and 'Advanced' features of EQcolors 3.0, it's coming along very well with no bugs that have surfaced, however, testing the memory overhead of EQ with different settings is quite time consuming.

I have several projects going on at once, but EQcolors 3.0 is on the top of my priority list, it's by no means a simple upgrade, it is a major new release that will make version 2.0 obsolete, and I strongly believe that people will use nothing else once it is available.

Would you like to leave feedback? or have any suggestions for features you would like to see in version 3.0? visit the EQcolors General Discussions Forum here.

Thursday, November 29th, 2001

EQcolors version 3.0 will be released somtime after the expansion EverQuest: Shadows of Luclin is available, and not after Windows XP as previously stated. This new version has many exciting and advanced features, a revamped state-of-the-art customizable (skinned) interface. It is currently being tested in-house only, and it "will" require DirectX 8.0+.

In other news, I've updated the graphics for this site as you can see, I'm toying around with a new design, and I'll be working in menus for screenshots, skins, additional presets, textures, and more as I finalize the layout.

I would like to thank all of you for the kind words, and positive feedback that I have received from users of my program, it's what has fueled it's existence to this day. If you wish to contact me in EverQuest, my main character is a level 60 Shaman named Greybur on Brell Serilis (my Wizard Thomus is now my "alternate" character).
