For the ultimate in stealth surfing at work, setup sshd, and apache on your home PC. Configure your firewall to only allow connections to apache from localhost. The only port exposed to the outside should be port 22 (ssh). Configure Apache as a proxy server.

Then, from work you can ssh back to your box, and setup a tunnel on your local machine that connects to your proxy server. Configure your browser at work to use the tunnel as the proxy server, and you're golden.

I don't reccommend using this for visiting sites you wouldn't normally visit.

A simple tip for getting around blocked domains and the like is to use a web translation service.

At the last place I worked, everything was locked down and crippled to the point that you couldn’t alter anything – you couldn’t even pick your own windows backdrop, let alone alter the security settings in IE. Pretty much the only thing you could alter was the desktop resolution.

The same went for web access. Seeing as it was a tech support job, they granted us access to company web sites, technical information sites, and other stuff of that nature. Certainly no access to EQ or games sites. You couldn’t even get to News sites, such as the BBC. Whole domains, such as geocities were blocked.

However, for all their security and nanny-ing, getting to pretty much any site was so easy it was almost comical.

All we had to do was load up a web translation service, such as the one found at Google, and tell it to translate the site we wanted to see from Spanish to English. You got the odd strangely worded sentence, but apart from that it worked almost flawlessly.

Heartview. An old evil trick.

Move the default directory to, say:

C:\boringstuff\bangles\ab372a38b899cs1kl12ssdfasfasddfa\

Then, set the permissions on "bangles" so that NO ONE can read it.

Programs that have the "secret" "ab372a38b899cs1kl12ssdfasfasddfa" can do whatever they want. And indeed, if the user knows the "secret" name, they can delete all the files, read them, whatever.
