"The last version of Redhat (7.2) is extremely bloated and seems to run much slower than Windows apps performing similar functions."

Well most of that slowness is due to something many Linux users hate with a passion: X. For those of you who don't know X is the program which runs the GUI for Linux. It can do cool things like run apps remotly, but that is something most of us wont ever use. X is VERY slow and bloated. It eats up quite a bit of RAM which slows down your apps a bit, and without finely tuned drivers from your video card maker any graphical functions are terrible. There are some things being developed which run off the framebuffer, this usually extremely fast and offers some cool MacOSX/XP like features to the GUI which is hardware accelerated, such as alpha-blending menus and such.
