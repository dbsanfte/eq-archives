I agree with you completely - which is why I worked on the program today and the command lines are now custom. You can currently specify 5 strings and the file for each is selected through a dialog box so you don't have to type the path yourself (I found that to be annoying as hell). Also, I liked Dysassembler's MSAgent prog, so I incorporated Microsoft Text-to-Speech in the prog. It doesn't do what his does, because I didn't want to completely steal his idea, so I made it a little different. You specify the string to detect, and the string that will be spoken when the first string is found. For instance:
If this text is seen: "INC"
Say this: "Incoming!!"
or
If this text is seen: "You are low on food."
Say this: "Eat something you whore!"
It's pretty useful Also, the thing with MP3's where you had to alt+tab out.. I found a way around it. Simply bring up the inventory screen (press "i") and it should play fine. Why? No idea. Lol. One last thing.. spaces in path names are allowed, and when you set the files to be opened by a certain command line, they can be MID/WAV/MP3/EXE - AdminEQ will figure out how to open it by itself. The new version of AdminEQ isn't uploaded yet, because I'm fixing some cosmetics and such, but it will be available this week.
