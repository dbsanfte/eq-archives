I have never messed with satellites so I wont be much help there. However chances are your hub has a crossover port that automatically crosses it over for you. Either the last port will actually have spots for two separate cables (DO NOT plug two cables into it at once) or it has a button. Typically they don’t say crossover, but rather say something along the lines of uplink or downlink or “connect to another hub”. It is the connect to another hub port that you want to feed the satellite through.

It may be absolutely nothing that is related to your connections though. Getting two microsoft machines to see each other is an exercise of luck and patience. Some things to check though are that your machines are both in the same workgroup, and they both have similar protocols. I have seen instances where; two machines may or may not see each other with no difference between the instances other than a reboot, one machine may see the other but the other does not see it, and my personal favorite is when two machines can see every other machine on the network but refuse to see each other. Basically, good luck and have fun!

Assuming that the satelite still hooks it self up as a Network adapter, this can be done, but the procedure depends on the operating system. Basically, if you are running any non-server Windows OS (i'm assuming windows, since i doubt the satellite company supports linux), you need to set up Internet Connection Sharing (ICS).

This is easy on 98, 2k, XP (probably ME, but i dont know). Search on google for ICS and you should find instructions. If you are running a server OS (NT Server, 2K Server, Whistler Beta), you should be able to set up NAT, which is much nicer than ICS. Once again, check google for exact instructions. If you have some special USB dialer, and it isnt set up as a standard NIC, then you are probably SOL.
