I actually wouldn't like to see spells and combat implemented directly into the emulator. Some easier-to-implement things (I hope)

- ability to choose faces, and to equip different items and pieces of armor on the Avatar and/or Spawned mobs.
- ability to spawn Mobs by name rather than needing to remember the number

Possibly harder:
- ability to save "Game" state
- ability to have multiple zones running on the same server
- ability to enable client-side spell effects (levitate, shrink, Ultra-vision etc) - this doesn't necessarily involve the full spell code, just the bits that allow item-based (or potion-type) casting would do. E.G. Summon a Pegasus cloak and right-click, Client sends message to server "activating Levitate", Server sends back "Sure thing! Go right ahead!", Client makes the necessary changes, and the client-side effects take hold. Similar things would work for "shrink" potions, the various "Masks", and so on. I suspect some of this will be related to the "equip items" stuff above.
