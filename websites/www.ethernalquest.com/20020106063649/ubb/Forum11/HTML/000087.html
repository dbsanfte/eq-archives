A new version is out which includes a map editor.
To edit a map, it must first be defined in mapinfo.txt. If you find an area that's not in Mapinfo.txt, hop on IRC and tell me or Sparr; we'll guide you through getting the info we need.
The editor only allows inserting things at the "current player position". As it's kind of hard to get the Z value of the current mouse position, this is the only way.
A feature I'd like people to try out extensively is the "useless node finder", which tries to pinpoint a point in the map that's useless.
A good example of a useless point is a point directly between two others (no difference in line angle) or a point that's on top of the next point in the same line. The function tries to balance between these two when choosing the useless point.
Some of the maps currently in use have way too many useless points, so this feature was needed... It WILL require some tweaking before we get it right though, so please post feedback.

The weight of each node in a line is

(change of direction in radians + 0.1) * (sum of length of both edges)

Meaning that for LONG lines, an extra point with just a tiny change of direction will be kept, but for short lines the change will need to be greater to before it's considered removable.
Note that the function just FINDS (as if selected) the most useless node, it doesn't actually delete it. So if you disagree with the algorithm, just disregard it

The only "tuning" that's necesarry is the +0.1 I do to the angle, as that's the balance between length and angle change... People are hereby encouraged to try to alter it and find a better value (or a better forumla) to "always" select what's right.

An alternative to my "find useless node" algorithm (which is pretty much a quick hack ) is to use a proper geometry simplification algorithm in which the entire line is replaced by new points representing the same "shape" in a simpler way, but that's a bit off yet

I'm considering moving the entire map editing out to a separate program that would be usable by both SEQ and OE (and any other program that uses SEQ-type .maps) since map editing doesn't really "belong" in OE, but as the "link" part of it (linking to in-game player position) would need to be implemented in the other programs as well for it to be of any use.. Well.. Maybe in the future.

I added a Douglas Peucker line simplification algorithm, which by default is enabled with a tolerance of 50.0

This will reduce the number of nodes in each line in the map when it's loaded so it's much faster to display, but should have almost unnoticable visual impact.
Note that this algorithm is far superior to editing the map and removing "useless nodes", so the recommendation for now is to ALWAYS have it enabled (set the tolerance to 25.0 if you must) UNLESS you are editing maps. If you are editing maps, go turn it off and reload the map before you start editing, or you'll be editing a modified map

I also updated the useless node finder a bit, but please just remove the nodes that are truly redundant... Please DO remove the nodes that are truly reduntant, though
