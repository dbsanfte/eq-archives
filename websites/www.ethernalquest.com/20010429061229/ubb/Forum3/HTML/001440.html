You MUST be using logical block addressing otherwise the boot process is dependant on the interrupt 13h process, and partitions beyond the 1024th cylinder are not bootable, as well as a host of other problems. Modern versions of Linux expect LBA mode to be enabled on large hard drives (large, in this case, taken to be more than 8 gig, but have LBA mode enabled anyway) when installing to an IDE hard disk. (SCSI overcomes these kinds of problems by being more intelligent.)

Whether Linux is before cylinder 1024 set your hard disk geometry to LBA anyway!
Go into your BIOS and check that your hard disk geometry is set to use LBA. If it is not, then you will have to copy all of your important data from the drive, and then delete the partitions and then change the geometry in the BIOS to LBA and repartition and then reinstall. Be careful, the screwing up of your partition table and disk geometry is possible when messing with Linux on large disks without using LBA.

With LBA mode enabled (check it doesn't just say auto, it should say LBA) then install Windows 98 / ME to the primary partition. Then start linux setup and create two partitions, one / and one swap, unless you want a more complex installation than that. If you install Lilo to the MBR, it will allow you to boot Linux or Win98.
