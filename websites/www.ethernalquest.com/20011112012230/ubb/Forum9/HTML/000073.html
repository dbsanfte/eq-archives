1) the dupe method listed here is present in ALL current online MMORPGs. and will be theoreticly possible in any system useing todays technology. why?

if a server and/or zone crashes and dumps its current memory this pushes back any character in that zone to the last backup

how do you prevent that? as most games do, by not allowing your servers to crash.

why is this rampant in AO at the moment compared to other games? simple, there are AO zones so unstable they crash sometimes 10-20 times per day. compare that to EQs crashes of 1-2 per month (im talking actual hardware crashes where the server memory gets dumped, yes i know EQ appears to crash more often than this; however 99% of EQ crashes are connection dumps[yes often verants not just your own] but if a connection dumps players the server is still up and will treat all characters that go link dead as if they camped within a certain amount of time, thus they dont get rolled back to thier last save)

the fix? upgrade and stabalize the servers so they dont dump thier memory constantly and crash. this is something im sure they have planned even without the exploits enabled by it.

than what happens? well, with stability you end up waiting weeks for memory server crashes and in that wait youd have to keep transfering items after each backup(20 mins in eq, not sure what it is in ao)

unless you know how to crash the server, which btw is tracable if your repeating the same actions each time, as they can check the debug log on each crash to find out the cause
