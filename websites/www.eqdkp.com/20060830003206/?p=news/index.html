Security vulnerability fix for all current versions of EQdkp (submitted by tsigo at 10:11am PDT)

A security vulnerability was recently discovered in all current versions of EQdkp that could allow remote code inclusion on specific server configurations, if register_globals is enabled.

This issue has been fixed with the following newly released versions:

If you are running 1.3.0 to 1.3.0p4
Download 1.3.1.

If you are running 1.2.0 to 1.2.1
Download 1.2.2.

The files provided are the full, patched releases of EQdkp, not just the changed files. To upgrade, simply upload the new files over the old ones, but do not overwrite config.php. If you're looking for a list of changed files, or a unified diff, see the comments.

March 16, 2005
EQdkp 1.2.0 Security Update (submitted by tsigo at 11:10am PST)

A security vulnerability was recently discovered in all previous versions of EQdkp, up to and including 1.2.0. A patch has been made available to fix the vulnerability and bring the version up to 1.2.1 that is highly recommended for all users to install.

To Upgrade:
  • First ensure that EQdkp 1.2.0 is installed. This patch was only applied against the latest 1.2.0 code and may break earlier installations if applied.
  • Upload the selected files from this patch.
  • Because 1.2.0 is already installed, delete the install.php file on your web server.
  • From a browser, access the upgrade.php file and select 1.2.0 from the Previous Version drop-down.
  • Delete the upgrade.php file from your web server for added security.
If you encounter any problems please post in the support forums.
