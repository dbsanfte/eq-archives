EQdkp :: Auth List
 
Authorization Options
EQdkp stores a list of authorization options in a table and then stores user permissions in another table that relates to the options table. Each option must have a unique ID number, and so each ID number currently in use is listed below. If you are developing a plugin and need to use EQdkp's authorization system to validate your plugin's options, you need to make sure that the number isn't already in use.

Auth option IDs to be used in plugins should be greater than 100, anything <= 100 is reserved for EQdkp itself.
