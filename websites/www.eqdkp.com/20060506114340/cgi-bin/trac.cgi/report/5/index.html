#71 Using wrong Item value in additem.php for some locales. Items 1.3.0p defect normal 04/19/06

When using another Locale than English (I use German), there could be the case, that the decimal floating point separator is not ".", but for example ",". This leads to pre-entered item values on admin/additem.php, which are for example "12,34" instead of "12.34". The JavaScript-Parser for the generated HTML uses /[^\.0-9]/ as regular expression, leaving the number as "1234"! Either you should modify the regular expression or change the sprintf in admin/additem.php to produce always "."'s as separator. Deriving from a user comment, using "%F" in sprintf generates a locale-independent separator ".".

In other words: 

admin/additem.php, Line 433:
----
FIND
----
$format = '%0' . $floatlen . '.2f';
---------
CHANGE TO
---------
$format = '%0' . $floatlen . '.2F';
