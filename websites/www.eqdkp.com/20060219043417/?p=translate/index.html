EQdkp Internationalization
EQdkp supports language packs to allow users to view each page in the language that they are most comfortable in. If you are fluent in both English and another language, creating a language pack is a great way to contribute to the project without programming skills. The instructions follow.
  1. Download the English language files and unzip them. You will end up with a directory structure like the following: 
    |-- english/
|   |-- email/
|   |   |-- register_activation_admin.txt
|   |   |-- register_activation_admin_activate.txt
|   |   |-- register_activation_none.txt
|   |   |-- register_activation_self.txt
|   |   `-- user_new_password.txt
|   |-- lang_admin.php
|   |-- lang_install.php
|   `-- lang_main.php
  2. Make a copy of the english folder and rename it to the language you're translating to.
  3. Start with lang_admin.php, lang_install.php, and lang_main.php. They are plaintext PHP files and can be opened in any programmer's text editor, such as EditPlus.
  4. Open the first .php file. The data you'll be editing looks like this: 
    
<?php
$lang['admin_title_prefix']   = "%1\$s %2\$s Admin";$lang['listadj_title']        = 'Group Adjustment Listing';$lang['listevents_title']     = 'Event Values';$lang['listiadj_title']       = 'Individual Adjustment Listing';$lang['listitems_title']      = 'Item Values';$lang['listnews_title']       = 'News Entries';$lang['listmembers_title']    = 'Member Standings';$lang['listpurchased_title']  = 'Item History';$lang['listraids_title']      = 'Raids Listing';$lang['login_title']          = 'Login';$lang['message_title']        = 'EQdkp: Message';$lang['register_title']       = 'Register';$lang['settings_title']       = 'Account Settings';$lang['stats_title']          = "%1\$s Stats";$lang['summary_title']        = 'News Summary';$lang['title_prefix']         = "%1\$s %2\$s";$lang['viewevent_title']      = "Viewing Recorded Raid History for %1\$s";$lang['viewitem_title']       = "Viewing Purchase History for %1\$s";$lang['viewmember_title']     = "History for %1\$s";$lang['viewraid_title']       = 'Raid Summary';
?>
    Only the string in-between " " on the right side of the = sign needs to be translated. Do not alter the parts that look like %1\$s or %2\$d. Those are replaced by variables in EQdkp and must be present in your translated packages. If they have to be moved for context or grammar in your language then that is fine, that's what they were used for.
  5. Now that you've translated the main language files, all that's left is the five short .txt files in the email directory. Below is register_activation_none.txt: 
    Subject: {GUILDTAG} {DKP_NAME}: Account Activated
Charset: iso-8859-1

Please save a copy of this e-mail for your records.  Your account information is as
follows:

Username: {USERNAME}
Password: {PASSWORD}

Depending on what the EQdkp administrator has set as the default user permissions, you
may be limited in what actions you are able to perform until your permissions are
updated by an administrator.

Your password has been encrypted in our database and cannot be retrieved by anyone.
If you forget your password, you may request a new one that will be activated in the
same way as this account.

Thank you for registering.
    The values such as {GUILDTAG}, {USERNAME}, {PASSWORD}, etc. are variables and should not be changed. Everything else needs to be translated. The same goes for the other four .txt files.
  6. Once you're done, zip up the directory you renamed english to and mail it back to me. Or better yet, post it in the Internationalization forum for others to look over and use.

Thanks to anyone who volunteers to make EQdkp more usable by users around the world.
