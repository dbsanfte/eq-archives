File Manager - Reloaded Tuesday, August 17, 2004 - 11:07:28 AM

I just got finished deploying the new File Manager.  This new file manager that I created will allow you to organize uploaded images in directories, copy/paste, move files etc.  I think it`s a lot more intuitive than the other one (for the record, I didnt program the other one!!). 

There is a help file to explain the various new features, but honestly if you are a Windows user, you should feel right at home as the file manager is very similar to the Windows file manager.  I had actually created a pretty neat drag/drop component to allow you to multi-select files/directories and drag them into new directories, but it was only working in Internet Explorer, so I opted for a more tradition `web-application` model.  So, instead of drag/drop, you can multi-select, but you have to click the copy/cut icon then paste into the directory you want to copy/move files to. 

Next on my list is to start focusing on the forums again.  There is a lot of little things that need to get done on the forums, and I have gotten some great feedback (and some whining too!) on new features.  Hopefully after the next patch, most of the forum`s unfinished features will be done.

Image Button Generator Thursday, August 12, 2004 - 11:30:48 PM

I created an Image Button Generator in the Control Editor.  Basically, this allows you to select a blank button and add some text to it.  The when you select Image Button as they Control Type in the Control Editor, you can now generate a button as well as just uploading a premade image button.

Right now, there is about 75 button styles that you can choose from.  When you are in the editor, you can choose the button style by clicking the icon which brings up a new window with a picture list of all the available button types.  Double click the button you want to work with, add the text for the button, and click the Generate button which will generate the button and display it on the screen.  The image is saved, but the Control is NOT saved untill you click the Save button.

You can create some pretty good buttons with this tool.  Keep in mind, it is very basic compared to any half-decent image editor.  If you can use an image maniplution program like PhotoShop, you should create your own buttons.  This tool is for the people who dont know how to use these programs, and want a simple way to create some buttons. 

I am going to reprogram the file-manager next..

Guild Application Page Tuesday, August 03, 2004 - 10:54:46 PM

I just deployed the new guild application/recruitment request page.  This page allows you to create a form for guild prostects to fill out to join your guild.  The questions are totally configurable by you, as well as the order in which they appear.  You can also mark questions as mandatory/required, which will display an asterix next to the fields that are required if they are not filled in. 

How to setup your guild application page:

  • Login with your admin/officer account
  • Click the admin menu link and click Guild Application Setup
  • The first page sets all the non-question items such as Page Header Title, Description paragraph (this is the paragraph before the questions, maybe to explain your recruitment policy etc), and an email address to send the guild applications to.  Fill in all this information, and click the Submit button to save it
  • Click the Manage Questions selector to change to this panel
  • Add your questions that you want to ask in the New Question section, select if its mandatory, and click the Button to save each question.
  • You can change the order of the questions  in the grid below, as well as delete any questions

I might add some more features to the application depending on the feedback/usage it gets.  I know that most guilds are going to want to have a thread about the applicant, and thats where the voting usually happens.  Will play it by ear to see what people want and make some feature additions from there.

Guild Member last login Tuesday, July 27, 2004 - 1:43:06 AM

I added some logic that records the last time a member was online, to be displayed on the roster.  I created a new roster column type (custom rosters only) that will allow you to plug this information into your roster. 

It doesn`t record the login time, because you dont really login everytime you visit.  Instead, it records the last request of the user.  The information *is* delayed (up to 10 minutes on the roster page) with server caching, so dont expect instant reporting on the roster.  On other pages that this field will become available on such as the profile page, it will be real-time info. 

 

So I decided to just change the thing.. Monday, July 26, 2004 - 8:38:20 AM

A little late in posting this, but it was a hectic weekend!  Instead of upgrading the web-server that hosts Guild Wizard, I decided to just change it over to another server.  The new server has a much better processor, more RAM, and a faster hard-drive.  I`d be suprised if anyone sees the service running any faster though, as everything was running tip top anyway :)

I`ve been experiencing a very steady increase in new guilds signing up and using the site, so I did this as a proactive measure.  In the fall/winter (or who knows when), when Everquest 2 (EQ2) and Worlds Of Warcraft (WOW) come out, I am anticipating a sharp increase in new guilds.  This new system is much more versed to sit in a small web-server farm (group of servers) incase server load gets high. 

Guilds who have custom domains had a about 5 hours of downtime on their domain name while I was switching everything over.  Some domains are really taking their time to update their DNS info, so I went ahead and rigged something up so they can continue using the domain with the old DNS info.
