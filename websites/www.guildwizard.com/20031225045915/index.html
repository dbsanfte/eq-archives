Turn off paging on Roster Thursday, December 11, 2003 - 3:55:39 PM

Custom Rosters now have a configuration setting in their configuration panels to turn off paging for the roster grid.  When you turn off paging, your roster will be one long list of everyone in the guild, as opposed to the 35 names per page.  This is useful for when you are doing editing on the grid, looking to compare values etc. 

Again, this is only on the Custom Roster. To change over to the custom roster, log in as an Admin/Officer account and click the custom roster button that is above the grid.  You can switch back and forth between standard and custom roster btw.  That was patched a few weeks ago, I dont think I reported it though :)

Custom Roster deployed Tuesday, November 25, 2003 - 11:29:10 PM

Well I finally got some time to test the custom roster out.  To change over to the new custom roster, login as an admin/officer account and click the Custom Roster button.. confirm that you want to do it, and you will then start out with a Roster with only the Delete and Edit  rows (they are fixed columns).  From there, click on the Configure View link and add the columns you want.   Anyway, here is my patch blurb:

Custom Roster Info/Features:

  • You are able to add the columns you want to the roster
  • You can put your own heading description at the top of each column
  • You can change the left-to-right order of the columns that you add
  • You can fully sort data, ascending and descending, any columns - even custom columns
  • New Base columns can be added anytime by me, as I get suggestions for them
  • The grid is only binding the columns added, which is very efficient
  • Your columns are cached, which means the software isn`t calling the database each time your grid is shown, bigtime performance increase.
  • You can set security permissions on each column, even custom columns
  • Column headers can have images instead of just the text (broken, will fix next patch)
  • Support for other games will be done with this Roster.  Infact, I will likely make it the default Roster for any new guilds joining, once I see there are no bugs.

As for the FFXI support.. it`s there.  Currently I have 2 guilds set as FFXI guilds, and they can start fooling around.  Honestly the only difference is the Races, Classes, and Secondary Classes..
