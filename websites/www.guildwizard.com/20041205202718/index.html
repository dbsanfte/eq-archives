Guild Raid Encounter Manager deployed Monday, November 22, 2004 - 10:03:28 AM

Late last night I deployed the new Guild Raid Encounter Manager.  Features of the new guild raid encounter manager:

  • You can do all aspects of guild raid encounter management right on the one manager
  • Deleting encounters is back in. BE CAREFUL and make sure you read the confirmation box before you start deleting stuff.  When you delete an encounter, you are going to be deleting guild raids attached to the encounter.  Moreover, you will delete any purchases tied to the deleted encounters.  If you dont want that, you can use the new feature to disable an encounter (described next)
  • You can now set an encounter as "Inactive".  By doing this, it will be removed from your Active Guild Raid Encounter List.  So when entering raids/purchases, that encounter will not show up.  Setting to inactive will allow you to maintain old raids/purchases attached to the encounter.
  • I added an optional field to allow you to tie a picture to an encounter.  This will be tied into the new encounter "Front end" which will be a lot more robust that whats there now, which is merely a boring list of encounters.  You basically choose the image from the file manager and it gets tied to the encounter.  Then, when I redevelop the encounter/raid front-end , you will be able to flag if you want to display the "Encounter Icon/Picture" on the display screen.... i think its going to spice up the reporting a bit.

Oh, I also created a side navigation bar for quick access to the other (new) admin panels as I create them.  I was getting annoyed myself when adding an encounter, and then wanting to tie a raid to it, I had to open the raid manager from the parent page..   I will add the new Purchase & Item manager when I create them.

On the weekend I did a few ninja patches on some bugs.. also added support for the /who list raid entry for EQ1 guilds.  I didnt realize guilds still used it to manager their raids :)

Next up on the hit list is the Purchase & Item managers.. then I will get going on the front-end pages like the Roster, Raid List, Profiles etc. 

New DKP Guild Raid Manager Friday, November 19, 2004 - 2:50:42 PM

I deployed the new Guild DKP Raid Manager a little while ago.  The new manager fixes a lot of things that was wrong with the old panels, and brings together a lot of the functions that were spread out among the various pages that displayed guild raid information.  Here is some of the highlights of the guild raid manager:

  • You can select multiple raids at once and delete them
  • When you delete a raid, you no longer have to remove purchases attached to it, the new raid manager will take care of everything.  When you remove a raid, the guild raid manager will now take away the points given to members listed as attending the raid, as well as remove purchases for the raid and reimburse the points spent on the pruchase
  • You can now re-assign the encounter tied to a raid by clicking the Edit icon on the raid list in the Raid Manager
  • You can now change the point value of a raid and have the change update all members points automatically.  So if you entered a raid at a DKP value of 2 but later decide it should have been a value of 5 instead, all people in the raid will be awarded 3 extra points automatically.
  • I added a Raid Note field (optional) so that you can add notes to a raid.  Currently the raid display page doesnt use this field, but it will when I re-do that page, so I put it in there for that time
  • A super-fly-TNT high-tech calendar will popup when you click the icon to select the date.  Seriously though, you have to manually specify the date now because I was getting way too much bad data in from parsing the log files.. its just a lot simplier this way.
  • You can Add and Remove players from a raid on the same page now in the Raid Manager.  Click the Edit icon for the raid and you will see the options to remove/add members to/from the raid
  • You can select multiple users to be removed from a raid now
  • When you upload a Raid Dump file, the checkboxes on the attendance grid are checked by default
  • I removed support for the old /who list batch entry.  Raid dump file support has been in a long time so i see no need to support this anymore. 
  • The batch file upload is 100% setup now for any other file formats that might surface from EQ2 or World Of Warcraft. 
  • The layout of the screens are a lot more logical
  • I removed editing functions on the Raid List pages.  I want to contain everything to the Raid Manager page(s).  The Raid List pages are going to be re-developed next week or so, but wanted to get those functions out of there until then.

I will continue re-developing these dinosaur admin panels.  The Encounter panel will be next just to get it out of the way (simple panel compared to the others) and then will get to the item panel.  The item panel is going to take a while.. i want to spend some time on that to possibly integrate with some of the data sites like Magelo, Allakhazam, etc.  I also want to make the item listing screen configurable so you can put your guild logo`s on the item view.

New Guild Member Manager Wednesday, November 17, 2004 - 4:36:26 PM

I just deployed a very significant patch to the administration section for all guilds.  The new Guild Member Manager is management console that replaces the Add Player, Restore Player, Add Officer, Change Member Password pages, as well as all the edit functions of the Guild Roster(s).

Guild gallery upgraded Wednesday, September 29, 2004 - 11:50:09 PM

I just got finished deploying a complete re-make of the guild photo gallery page.  The old guild gallery was, very simplistic with next to no features.  The new guild photo gallery has some very nice features to make organizing and viewing screenshots/photo`s very nice.

Some key features of the new guild photo gallery:

  • Albums allow you to group images into logical sections.  If you want to organize all your raid images in one album, another gallery for guild member images, and another for guild events, you can do it all in the new Guild Gallery Manager (login with your admin account to see the link)
  • You have a LOT more control over the various settings and display of your gallery with the new guild gallery.  You can customize how many thumbnail images are displayed per page, how many columns, as well as specifying images for the Next & Previous image viewer.  Moreover, the various structural elements of the new gallery are fully CSS ready for the Guild Style Editor for even more customization!  I fooled around with the gallery you can make 10 galleries look VERY different from eachother with the various settings!   No cookie cutter look here, thx :)
  • You can enable or disable Paging on the thumbnail listing of an Album.  Of course, you can also specify the page size (how many thumbnails per page) if you have paging enabled.
  • THANK GOD ALMIGHTY (whoever your God might be) that the thumbnails DO NOT reload on every page load anymore.  A major hurdle with dynamically generated images like thumbnails is that most browsers wont cache them locally.  I developed a custom HTTP Handler to handle it, some real cool stuff for those geeks who have the slightest clue about what I am talking about.  I developed a component that dynamically generates the images for thumbnails etc, and properly complies with the browsers to cache the images locally. 
  • When you are browsing pictures by clicking the Previous & Next buttons to cycle through images, you will notice that the images are changed without the page refreshing.  Anyone who has ever used Coopermine gallery knows how slow and annoying it is to cycle through images.
  • You can set Permissions on each album.  You can set albums to be viewed by Everyone, Guildmembers or just Admins/Officers. 

One thing that isnt a feature, but kind of a side effect, is that you will have to create an Album to put your existing images into.  Don`t flip when you go to your Guild Gallery page and see the message "No Albums Configured".  You just have to go into the Gallery Admin page and create an Album.  Then go into the Image section and assign the images in the Uncatagorized Group to your newly created Album.
