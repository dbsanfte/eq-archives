Guild Application Page Tuesday, August 03, 2004 - 10:54:46 PM

I just deployed the new guild application/recruitment request page.  This page allows you to create a form for guild prostects to fill out to join your guild.  The questions are totally configurable by you, as well as the order in which they appear.  You can also mark questions as mandatory/required, which will display an asterix next to the fields that are required if they are not filled in. 

How to setup your guild application page:

  • Login with your admin/officer account
  • Click the admin menu link and click Guild Application Setup
  • The first page sets all the non-question items such as Page Header Title, Description paragraph (this is the paragraph before the questions, maybe to explain your recruitment policy etc), and an email address to send the guild applications to.  Fill in all this information, and click the Submit button to save it
  • Click the Manage Questions selector to change to this panel
  • Add your questions that you want to ask in the New Question section, select if its mandatory, and click the Button to save each question.
  • You can change the order of the questions  in the grid below, as well as delete any questions

I might add some more features to the application depending on the feedback/usage it gets.  I know that most guilds are going to want to have a thread about the applicant, and thats where the voting usually happens.  Will play it by ear to see what people want and make some feature additions from there.

Guild Application Form Tuesday, August 03, 2004 - 7:59:22 AM

I am about 60% done a new component that I will be deploying probably today or tommorow.  The Guild Application page will allow recruits to post an application to join your guild, with questions that you create in the administration panel.  Of course, you will have complete control over the questions, the order of them, and all the content on the page.  I also have a spot for a paragraph above the questions for you to have some preface/intructional text for people to read before they fill out the app.

Guild Member last login Tuesday, July 27, 2004 - 1:43:06 AM

I added some logic that records the last time a member was online, to be displayed on the roster.  I created a new roster column type (custom rosters only) that will allow you to plug this information into your roster. 

It doesn`t record the login time, because you dont really login everytime you visit.  Instead, it records the last request of the user.  The information *is* delayed (up to 10 minutes on the roster page) with server caching, so dont expect instant reporting on the roster.  On other pages that this field will become available on such as the profile page, it will be real-time info.

New Control : Fading Slide Show Thursday, July 22, 2004 - 5:18:47 PM

I created a new Control that will allow you to create a fading slide show.  I got the idea from  http://www.shadowedstrife.net  who has it on their guild wizard site (it`s on the right side).  Like any other control, you create it in the Control Editor and select Slide Show Control from the Control Type List.

A few things you should be aware of: 

  1. When you Add images to the slide show, the list is NOT SAVED until you click the Save button.  Just because it`s in the grid, doesnt mean it will be saved, so please remember that. 
  2. You really should have all the images the same size.  You can resize images in pretty much any image editor.
  3. The fade effect doesnt work in Mozilla browsers, but the control itself should (images should change on time etc)
  4. You can select the duration of each image being shown by selecting the desired duration from the list.  Default is 3 seconds.

Dude, two upgrades in one day?? Monday, July 12, 2004 - 11:58:09 PM

Well, you can tell that I am not playing any MMORPGs at the present time :)   I spent tonight doing some nice tweaks on the new style editor, based on a couple peoples suggestions (and a few peoples whining/death threats) hehe

  1. When you adjust the width of the Left or Right panel, the Center panel`s margin will automatically adjust accordingly.  So, if you make the left panel width 400pixels, the Center panel will automatically have a 400pixel margin.  Good suggestion whoever sent it to me.
  2. I fixed a little bug when you type in a URL for the background image of stylesheet item.  Before it was always assuming the image was in your guilds image directory, which is fine for most cases but I can see it would irritate an advanced user wanting to use an image on their own webserver.. so the system will check if a valid URL is entered, if not it defaults to look for that image in your guilds directory.
  3. Ok those who styled their guild site entirely with the visual editor is going to hate me when they read this.  You can now copy style settings between elements to ensure a consistent look/feel throughout your elements.  For advanced users, its still easier to just hop into the stylesheet and cut/paste the style settings, but for normal users this is going to save a LOT of time.

    Example:  Lets say you want all your grid headers to look the same as your Roster header.. no problem now.  Just select Roster Header from the new dropdownlist to Copy From, and then select the Raid List Header, for example.  Click the Copy To button and the Raid List Header automatically inherits the style settings for the Roster Header.  Rinse and Repeat for the various other elements that build the grids, and you have a consistent look/feel without having to remember colors, borders, font-sizes, padding, margins etc.  Just remember, you can *still* customize it after you inherit a style, its just a base to work with.

    This is going to save a lot of time for button styling too.. design your first buttons style, and then copy the style to all your other buttons..

Thanks for all the suggestions from the various guild leaders / web masters out there.  I really appreciate the help to make Guild Wizard better.  Believe me guys/gals, I am on your side.. if you want something added, changed, altered, or removed.. let me know!
