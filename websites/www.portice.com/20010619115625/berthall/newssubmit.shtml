Latest change in the works on Test - Looking for Group flag.
[ 0 comments] Submited by Porticemaster

 Hello all,

I thought I'd drop by and let you know of a new feature that we are trying out on Test. Last week it was suggested here on the boards that we implement a "Looking for Group" flag that players can set and view in order to help people get together quickly, allowing more time for adventuring. We've decided to try it out and it's on Test now.

The /lfg [on|off] command will set or unset your "LFG" flag. Players can then use all of the "/who" and "/who all" filters to find each others. For instance, "/who all rogue 45 50 lfg" will return all rogues between 45 and 50 who are looking for a group. "/who lfg" will return all players in the current zone who are looking for a group.

Your LFG flag will automatically unset when you enter a group. One final note, though anonymous and roleplaying characters can set their LFG flag, they will not show up in filtered /who lists. In other words, the first example above wouldn't return any anonymous rogues, however "/who all lfg" would return everyone in the game who is LFG (subject to the 20 person limit).

Assuming all goes well in testing, this change will most likely be in the next patch.

-Gordon

Ever wonder why?
[ 0 comments] Submited by Porticemaster

 Pressing a hotkey doesn't activate an item?

Pressing a key for something in a hotbox is essentially the same as left-clicking on it. That's why it works for spell gems. Activatable items require a right-click in order to be activated. If a right-click were applied to a spell gem, you would forget the spell.

That said, the idea for the key to operate as a right-click for activatable items is a good one, and I'll forward it to the programmer who's handling UI changes.

-Gordon

*Guide Application*

The Guide Application is now back up at http://guideapp.everquest.com. If you are over 18 years of age, have a good customer-oriented attitude, enjoy helping your fellow player, and are interested in joining the Guide Program, we will be pleased to review your application.

*Kunark Users*

A new file, emeraldjungle_chr.s3d, is included as part of todays patch. This patch adds additional displayable NPCs to the Emerald Jungle. Please run the "Kunark Patch" by pressing the "Kunark Patch" button on the bottom of the patch program UI prior to entering EverQuest.

*Patch Day*

Greetings everyone! Today's patch contains some exciting new changes.

*General Enhancements*

- Text messages that wrap will be indented on subsequent lines. This means that if a person or NPC says something that is more than one line long, all subsequent lines will be indented a few spaces. This should make it a lot easier to pick out separate messages.
- The spells "United Resolve" and "Heroic Bond" had their levels transposed accidentally. Rather than putting them back as intended, United Resolve has been changed to Heroic Bond, which means you might have two copies of that spell in your spellbook. Feel free to move one to the back. Clerics get a new group spell at 58 to compensate.
- The Wurmslayer is now a one-handed slashing weapon.
- The /help command will now list all /-commands available to you.
- Rangers, Paladins, and SKs now have mana equivalent to that of a pure caster with the same stats. This will give those hybrids a good deal more mana, regardless of their stats.
- NORENT items, items that disappear when you log out, will now show a "NORENT" tag when inspecting the item.
- Most summoned items now have the word "summoned" in their name.

*Changes/Fixes*

- Small Studded Tunic recipe now works
- Scaler recipe now works
- High Elves can now use the Moss Covered Twig and Branch
- Iksar can now wear the Crescent headband.
- The Shintl Lowbrew quest will now work
- The Rusty Rapier now looks like a rapier.
- Elven Wine can now be brewed
- Some pelt-dropping animals were always dropping ruined pelts, rather than pelts of quality. This has been repaired.
- The Guild-invite toggle in display options will now work.
- The faction problems with the Charasis Key Quest have been resolved.
- Many pathing errors have been repaired.
- Repaired a bug that caused some anomalies after being charmed or feared.
- Repaired a bug that kept the Unswerving Hammer of Faith and Flaming Sword of Xuzl from disappearing.

*Dragons?*

Rumor has it that some new dragons have been spotted in Kunark. It should be noted however that the dragons can be engaged from much further away than is intended. Rather than hold off implementing these dragons for another week, we've decided to implement them now and fix the problem in the next patch. Enjoy


*New Command For Tracking*

New option for those with tracking: /trackplayers. The command will work for all players with the tracking skill. The command has two settings:

- ON: Default setting, same as it is now. Players will show up in the tracking window.
- OFF: Players will not show up in the tracking window.

*Client Customizations*

Upon starting EverQuest, a new file (EQCLIENT.INI) will be created in your EverQuest directory. This file contains customizable settings for:

8-4-2000
New EQ Gameplay Forum Policy
[ 0 comments] Submited by Porticemaster

 Hello all,

We're quite pleased with the way the boards are working out so far, and hope that they will continue in this fashion. That said, there are a couple other issues that we need to address in order to facilitate the board's continued usefulness:

Nerf Requests: We're going to begin closing requests to nerf others classes. It becomes a fairly common occurance when it's apparent that we aren't going to upgrade another class. Nerf requests are very inflamatory and are a breeding ground for all-out flame wars. As we don't want to have to liberally ban people if it can be handled via other means, closing the threads is probably the best action we could take.

Dead issues: We'll also begin closing threads, explicitly or implicitly addressed to us, that we consider dead issues, issues that we've addressed already and consider "locked down". You all can feel free to continue discussing the issue amongst yourselves, but if it's a "Verant please read!" thread, and we consider the issue closed, we'll close the thread as well. We're going to do this in hopes of making room for new threads on other subjects that just can't seem to draw much interest on their first go-around. Rather than have a dozen threads pop up asking us to continue explaining the issue, this might be the best course of action.

In addition, I'm going to create a "dead issue" section of the FAQ for these things as they continue to come up. We realize that not everyone can read everything on the boards every day, so we should have the data available someplace.

-Gordon

More Player Concerns Addressed
[ 0 comments] Submited by Porticemaster

 >>>1) ghost/warping - can it/will it ever be fixed?

We consistantly work to improve these issues, however the level to which improvements can be made are sometimes limited by the size of the Kunark zones. We'll continue making changes until we come up with the best possible arrangement.

Keep in mind that we will never be able to "fix" these issues when they are caused by internet latency, as latency will always exist and is very rarely anything that we control.

>>>2) Massive RL time sinks traveling - Up to an hour to get to Kunark and other continents. Long RL time doesn't = challenge

There are things that you can do to reduce your travel time, such as by hooking up with a druid or wizard who can help you get around.

>>>3) grouping made easier - cross zone chat or looking for group flag that can be searched using /who all

We wouldn't implement a zone-wide chat channel that goes to more than one zone at a time due to both bandwidth and spam potential.

A "looking for group" or other flag is a possibility. I just spoke to one of the programmers about it and it doesn't appear to have any technical concerns on first glance. Possible issues include /anon and /roleplay. Neither would show up on any filtered list, while an unfiltered list (/who all group) would probably return far more than 20 results. If you think that it would help people hook up more swiftly in game, we can see about putting it in. Of course it goes without saying that the flag would be turned off automatically once you join a group.

>>>yes, perhaps the 'short' zone names can be integrated into the /who command, like

Due to bandwidth and system utilization concerns, we will not be able to implement zone-name filters in "/who all" for the public.

-Gordon
