We will not being seeing a command to see who is in a zone at all (it was hinted a while ago that players might). It's a shame really that we aren't.

This functionality already exists for those who are GM flagged, however due to bandwidth and server load issues, we've decided not to open it up to the playerbase at large. This command has to query the server of the requested zone in order to return the list.
Dozens of people on each server executing the command at the same time would presents the concerns mentioned above.

Test server player wiped
by Chris - 06/20/00 20:57 EST
If you had a character on the test server, you no longer do. All characters have been deleted.

I wonder what this means for the release of The Hole. Nobody will be anywhere near the level to test it.

For those of you who had characters on test, you have my sympathy. For 3 months or so, test was my home as well. I had a level 22 necro, and several other characters around level 10 on that server. I still played my necro every once and a while for a change of pace.

June 20th, 1:00pm
-----------------

*Character Wipe on Test*

Over the past 15 months, the Test Server has been a fantastic platform for testing new and proposed changes to the live EverQuest servers. However, in the past couple of months, with the release of several new zones, plus the entire Kunark expansion, the characters on the test server, and the equipment they carry, have been further deviating from the norm that is expected on the Live servers. This has made testing certain aspects of the game more and more difficult as time goes on.

With great sympathy to those that have built their characters on Test and made the server their home, it is now necessary that we "clean the slate" on the Test server and return it to a state that will allow successful testing to take place. To that end, as of today, all characters have been deleted from the Test server. Though no reimbursement of characters, equipment, or levels is planned, strict guidelines have been established to govern future artificial buffing and equipment allocations on the test server in order to insure that the environment remains a clean testbed for enhancements proposed for the Live servers.

We would like to thank all of the Testers that have been helping on the server since release, and we hope that you will continue to do so. An email address (testserver@verant.com) has been set up to receive your comments on this issue. We will make sure that your comments are forwarded to the appropriate people.

Thank you for your understanding,

The EverQuest Team


No Mac or Linux clients for EQ
by Chris - 06/20/00 14:14 EST
Looks like there definately won't be a port of EQ to the Mac or Linux. It's a shame, but at the same time, they are relatively small (especially linux) market shares, and I don't now what is involved in porting code over.

>>>I know this is cluching at straws, but any chance of having EQ run on Linux?
 We've no plans at this time to develop or release additional platform ports of the EverQuest client.

>>>Also, what system was EQ developed on? If you can reply to that one without revealing too much...
We use different tools for many different aspects of EQ. Outdoor geometry, indoor geometry, object building, character building, texture art, etc., all take place in different software packages. As for actual programming, it's done primarily in a combination of C and C++.

As for what particular hardware platform is used for each thing, it varies between WinTel stuff, various Unix platforms, and the like.
