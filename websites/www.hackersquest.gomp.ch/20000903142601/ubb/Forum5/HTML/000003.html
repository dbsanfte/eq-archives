I dont have 2 computers, but i do have 2 monitors on my win98 computer, so i run the winshoweq program in that 2nd monitor window. I have the keyboard remapped so, you can contol the winshoweq via the keyboard while playing eq.. =)

They only keys are (if window has the focus):
up for scrolling up
down for scrolling down
home for top fo the tracking list
end for the end of the tracking list
space to take out the killed mobs from the list and redraw it...

if you are playing showeq, and running this on the 2nd monitor, the keys are:
i for scrolling up 2 at a time
k for scrolling down 2 at a time
home for top
end for bottom
space for cleaning up the list..

take note, when eq is running, winshoweq runs a little slower, so you have to hold they keys down for a sec or 2 so they will register....

To get it to run, you first have to install the packet driver... this is done via the network in the control panels. you need to add protocal... and then click on have disk, and the point it to the driver included in the zip.


Then run the program.... and click on new game (yes, the original source code came from the mazelords game... i just have not taken the time to change all the menu names yet.. hehe.. get over it.. =P )

and then type in the ip address to sniff....

and watch it run.. it does all that I ever wanted.. ie, a good way to track mobs...

Ok, install steps...

Step1: Goto http://netgroup-serv.polito.it/windump/2.02beta/download.htm
and download the latest PCAP driver for your computer os version (95,98/NT/2000). Remember to do a restart afterwards.

Step2: Download Windump from the SAME site, and verify that it works with the default sniffing device.

Step3: Launch the winshoweq.exe. Click on file/new and then type in your ip address. and (1) recommended for packet threads/sec.

Step4: Lauch everquest. (it dont matter if you wont be able to see the winshoweq screen at this point. Log in, and select a player. and enter the game.

Step5: Log back out... Take a look at the winshoweq program... did it fill up with spawn info?

Step6: Move the sniffer to another computer/ or monitor, and repeat... so you can run eq, and see the SEC output.. =)


It dont get much easier than that..


Troubleshooting: I cant get it to work, it crashes.

A: Read the stderr.log file. What is the errror message? If it is it cannot initialize the PCAP driver, did you install it correctly? If yes, then you probably have 2+ sniffing (possible) devices.. ie, a modem, and a ethernet card.. etc.. You will have wait for the next release... or remove one from your control panels..


2: It just crashes... no stderr file...

A: post the memory address of where it crashed, so I can look into it.


3: It crashes after I run it.. ie when i exit the program.

A: This is a bug in the packet.dll program. You will have to a restart.

Something else I have been thinking of:

Until we get source code (I haven't run the proggy yet, forgot when I got home last night, and just started EQ up as per usual), any STANDARD registry entries, or for that matter, anything CONSTANT is a danger here.

If a registry entry is always at:

HKEY_CURRENT_USER/Software/WinShowEQ/settings/username = "abcdef"

Then the EQ client could easily search for this. Same goes for a constant install directory, or even a constant filename (albeit, it's almost impossible for the DLLs to be renamed, and many proggies might just use the DLLs as well).

The assumption I am making here, is that VI will go ALL OUT to stop this proggy. Yes, illegally scanning the registry, or even all attached storage devices.

(Yes, I am very paranoid, and not looking forward to risking a lvl 50 char + numerous other lower ones, but I feel this tool and the information it will gather too useful to ignore.)

I guess my point is that I'd like to see as much configuration/customizability as possible. I realize (programming wise) this is a waste of time, and less functionality getting into WSEQ, but in the long run, it will allow us to continue, unabated.

My 2 cents.
