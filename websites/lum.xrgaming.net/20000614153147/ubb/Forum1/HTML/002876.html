Partitioning your drive, which is what you are asking about, will have no effect on your speed. You will still be using the same drive on the same IDE controller, though it will be designated with another drive letter.

The best thing for you to do without purchasing any hardware is set your swap file to a specific amount of space. That way Windows doesn't waste any time trying to figure out what size it feels like making it.

Before you do that, though, run scandisk and defrag. Those two alone will provide you with some increase.

Other than that, you have to buy another faster hard drive and/or more faster ram. There's just no way around that.

Run Scandisk and defrag AFTER Removing the Swapfile
Then rebuild your swapfile
It's not uncommon for file corruption to occur on the protected portion of the drive. Scandisk and defrag will not even look in the area that Window uses for the Swapfile.
It also enables your system to use the largest, contiguous area of free disk space.

quote:
Originally posted by KoF_is_k3wl:
How do I make a swap file?

- Right click My Computer and select properties

- Select the performance tab

- Click on the virtual memory button.

There it probably has Let Windows manage my virtual memory settings selected. You can specify your own virtual memory size by selecting the other option. Virtual Memory is essentially the same as a swap file as far as you are concerned. If you look at the greyed out text in the minimum and maximum boxes, you will notice that they are different. This means that Windows makes your PC work harder to determine the most effective swap file size. By making them both the same size, you eliminate that process since it is relly unnecessary.
