I'm pretty sure that if you do a search in the EQUI_playerwindow.xml file... and find this section (this is new today)

<Ui2DAnimation item = "A_AttackIndicator">
<Cycle>false</Cycle>
<Frames>
<Texture>TargetBox.tga</Texture>
<Location>
<X>6</X>
<Y>2</Y>
</Location>
<Size>
<CX>116</CX>
<CY>28</CY>
</Size>
</Frames>
</Ui2DAnimation>


You should be able to change the size to look like this:

<Size>
<CX>0</CX>
<CY>0</CY>
</Size>

That should get rid of it for the time being... until people have a chance to play with it.

12-18-2003, 09:57 PM
well for one thing, the size of the "textureinfo" piece is simply indicating the size of the file.. it doesn't modify the size of the graphic you get out of the file..

Also I don't believe changing the size of a "Ui2DAnimation" will stretch the graphic either.. All you did was tell it to repeat until it fills up the entire space..

Only changing the size of a "StaticAnimation" actually stretches the graphic .. And I don't think you can turn this one into a static animation..

To do what you are trying to do, you have to modify the graphic itself to be larger .. you would need to open the file in Photoshop or some other program capable of editing a .tga (targa) file and change the paper and image size to match the size of your player window .. then your "textureinfo" would indicate the size you changed the file to.. and your "Ui2DAnimation" would also have the same size..

The other alternative is to simply change the graphic used in that file to be something more suitable to the style of your window.. You would probably need to do it in a program that can handle the transparent aspects of the window (aka alpha channels) .. again Photoshop is the most well known .. hehe
