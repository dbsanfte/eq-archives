The only way it could be done:

Write an application that monitors your log file for /notes and /loc.

Make tons of hot keys that when you click them it does something like /notes mapchange:scarletdesert

When the application sees the mapchange command it modifies the xml file or tga file (not sure how the maps are accessed in the mods that are out there now) for whatever window you've decided to put your maps in.

The application could use the /loc and some freeware scripted image editing software (like ImageMagik) to draw a dot on the image of the map.

Then do a /loadskin myskin 1

So..it's doable...sucks to high heaven but CAN be done. I think the drawing of the dot is the hardest part. The changing of the map is easy. It just sucks you have to reload the entire UI and can't just reload one selected window.

You could still do an ingame map. It would not keep track of your current location, but the pictures could still be used. This is how I Imagine it being done.

Take a window (Like the pet window, or the inventory window) and replace it's contents with a tab sheet. Say 5 or so tabs on the top (1 for each continent, plus the moon) Then inside each tab add a new tab for each map for that area. Then on each tab add an image of the map.

Now remember this would lead to alot of tabs, since there are alot of maps. But it's the only way I know of that doesn't involve using some external program to get maps, or checks you position.

You could decrease the number of tabs, by only using the maps you plan on using when you play. Basically you set like 10 images that would be used for maps (say 'map01' to 'map10' then copy the desired maps before loading.
