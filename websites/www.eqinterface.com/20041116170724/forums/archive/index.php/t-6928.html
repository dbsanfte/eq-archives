SoesUI 1.4

*** There is a bug with having Sound disabled that causes the UI to crash, make sure you have Sound ENABLED before you load the ui! ***

- Updated for new Legacy of Ykesha UI Files
- Updated Inventory Window
- Seperate ini files for 1280x1024 & 1204x768 resolutions
- Based of Drakah's Drakahmix (Excellent work Drakah!)
- Optimized for 1280x1024 resolution but ALSO WORKS in 1024x768 resolution.
- Provides 100 more viewable pixels than Drakahmix.
- Designed for all classes (Casters, Hybrids & Pure Melee)
- Features Customized: Buff Window, Short Duration Buff Window (Songs), Player Info/Clickables/Pet Window combined into one, Spell Book, Warcraft 3 Style Spell Icons, Spell Cast Bar, T.King's Arcane Guage, Hotkeys, Group Window, MP3 Player, Compass, Target Bar.

This is SoesUI 1.4, rejoice...

*** NOTE ***
1024x768 Resolution users, use the UI_YourChar_ServerNumber1024.ini file.

1280x1024 Resolution users, use the UI_YourChar_ServerNumber1280.ini file.

Copy the appropriate file to your _base_ eq directory (usually c:\Program Files\Sony\EverQuest Trilogy) and rename it according to your char's name and server number (for example mine is UI_Soes_35.ini, my char name is Soes, Veeshan is # 35...)

If you notice in the default UI, if you disable sound from Options that your Mp3 player will not come up. In this UI he is forcing a Mp3 player to come up so those of us that have Sound disabled, it is causing us to crash. If you have sound disabled your MP3 player line should look like this :

[MusicPlayerWnd]
LastM3U=***
Playing=0
Random=0
XPos1280x1024=580
YPos1280x1024=78
BGTint.red=255
BGTint.green=255
BGTint.blue=255
Fades=true
Delay=2000
Duration=500
Alpha=255
FadeToAlpha=128
Locked=false

Just change it to that, or delete his line all together and you will be able to load up. EQ will change it to what I posted either way.

Hope this helps :)
