Xymarra's UI Version 2.11

This is a full UI designed for a 1280x1024 window. The 3D rendering area is a 4:3 aspect ratio unobscured by any of the UI panes, so the full world will be visible. Currently it is 1000x750. I am working on a version with an 1120x840 rendering area.

The windows are dark green with gold borders. Most buttons that duplicate the functionality of keystrokes or /commands have been removed to reduce clutter. Title bars have been removed from most windows. A thin 2 pixel border is used to further reduce visual clutter. Buttons that do have to be pressed, such as Destroy in inventory, Yes/No in the confirmation box, and Track/Cancel in the track window have been made larger.

There is no Invite/Follow or Disband button for groups. I use Ctrl-i and Ctrl-d for those commands. I will not be adding those buttons to this UI, although I will probably make them an option in my next version because so many people miss them.

Target health bar, group health bars, and pet status are grouped in the player status window in the lower left corner.

Stats, resists and xp bars are removed from the inventory window and given their own panel at the upper right corner. Stats are labeled with icons for their buffs instead of with strings.

The target window (not shown) has a very large health bar, large font target name and a hp percentage.

Spell icons and the spell book artwork are from Thomus King. The rest of the new artwork is my own.

The home page for this UI is: Xymarra's UI Page (http://www25.brinkster.com/xymarra/Skins/Xym_UI.html)

Yes, actually that's what I did. I only changed three icons and they are in the first page. You will have to download the rest of TKing's icon set if you want them to match.

Was wondering if one is just able to use your spell-bar? If so, how please?

Yes, you can use it without the rest of the mod. You just need the EQUI_CastSpellWnd.xml file and GoldCurves.tga. If you want the gems to look right, you'll need to copy the definitions for the spell gem art into your EQUI_Animations.xml file. They have to be defined in the global file because they are also used in the hot box and for cursor attachments. These are the relevant items:

<TextureInfo item = "GoldCurves.tga">
<Size>
<CX>256</CX>
<CY>256</CY>
</Size>
</TextureInfo>

and the Ui2DAnimation entries for:

A_SpellGemHolder
A_SpellGemHighlight
A_SpellGemBackground
