Now for the info about the GUI I am developing. I am a newbie skinner so this is taking longer than I think it should, but I am not a programmer, nor am I a graphic artist.

Having said that, I have created the gui as it is so far based almost entirely off of hacked peices of T. Kings original inventory screen and celtic cast bar.

There are a couple things to know about it.

1. I need to resolve some layering problems (if I even can) that cause the group box to load ontop of all of the other windows, this prevents you from accessing them. (Includes buff bar, chat boxes, hotbutton box, stat box) I would like to a.) change the z-order somehow so they are on top, or b.) use Includes to embed the bulk of their logic into the group window.

2. Several of the "utility windows" like help, pet, and selector have been hijacked to create the border around the view port.

3. The friends window is still available for use. However, my intent is to turn it into a timer window.

4. ALL of these windows are created using backgrounds, I have in no way modified the default borders used by EQ.....at least not yet. ;-)

5. I am also working on an 80 slot merchant window, 30 slot loot window, bank, backpack, and inspect windows to match.

6. and finally...I also intend to mod the MP3 Player to fit exactly in the same spot as the stat window, so if you want you can have it up as a replacement for the stat window...though I may try and make one that surrounds the compass like the casting bar does around the target window.

It is important to note that in order to gain access to your covered windows, you will need to close them and re-open them using keystrokes. This places them ontop of the group window. Unfortuanatly there is no keystroke to open/close the chat windows, so you can only page up using the keyboard.

This ONLY for 1600x1200, however once I have it done, I will be able to create smaller resolution versions.

Compass totally fixed and working!

Thank you Xymarra! Your tip not only allowed me to get the compass working, but I have deciphered exactly what the rules are for creating borders in Verants warped XML schema.

To explain using algebra:

A windows Height = Tobleft Border Height + BottomLeft Border Height + LeftTop Border Height + LeftBottom Border Height + Left Border Height

AND

A windows Height must be greater than or equal to the sum of the Left Border Width plus the Right Border Width. Notice that you DO NOT ADD the width of the Top(Middle) border!

Knowing these 2 things you can make a border of ANY size IF you use blank space (transparent) beyond your visible border to make sure that the above equations hold true. It also does not matter if you put all or some of the transparent space on only 1 side of the border. For me, I put ALL of it above the compass (1388 pixels high worth!) Having done that, the bottom of the compass becomes the true bottom of the window.

Yes..I know this is confusing...sorry. ;)

In any case YAY!
