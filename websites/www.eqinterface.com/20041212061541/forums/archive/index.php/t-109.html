Yes Niik that is a bug on Verants part.

Not sure which buff window your talking about , but if it is mine there is a reason for the no resize. It will not be as readable should you resize it.

The problem with it is :
- the text needs a co-ordinate to print at, this is fixed and unmoving (unless you edit the XML)
- the buff icons even though they have default placings are hardcoded to move as you adjust the size and shape of the window.

This causes a problem for those who move their windows around. If you have the position and shape of the window set and don't plan on reshaping it, this would be easy to do.

Currently the buff window I made is wide enough to fit the largest spell name in the spdat.eff file while leaving the buff at a size that is remotely visible. To shrink it further you could try reducing the size of the icon (even to 0), or shrinking the font to 1 (which might make it unreadable)
