I would like the story window to remember the last few links clicked and provide "Back" and "Forward" navigation buttons to move around the list.

It is difficult to provide "backlink" tags on pages that are refered to by multiple pages. For example: a page describing "Minor Healing" might be linked to by a page listing "Cleric Spells" and another listing "Druid Spells". A backlink on the Minor Healing page cannot be created in this case.

Here is how a history might look after clicking 5 links and pressing "Back" twice:

array[0] = "file0.txt"
array[1] = "file1.txt"
array[2] = "file2.txt" <-- this file is currently displayed
array[3] = "file3.txt"
array[4] = "file4.txt"

Clicking "Back" now will display file1.txt, while clicking "Forward" will display file3.txt. Clicking a new link will replace file3.txt and remove file4.txt from the history.

The buttons should also disable/enable themselves when clicking them would have no result, like when at the start or end of the list.

As an alternative to Back/Forward buttons, perhaps a <back></back> tag we could insert into our files that would have the same functionality.

Sorry for editing so much. In the vain hope a dev sees this I felt it best my idea was presented as clearly as possible. I apologize for any posts following this that do not make sense as a result

I've got to agree... a complete overhaul of the story window would be nice... I understand why it is how it is, that it was intended to be easy access to game lore, as it was released... but it's way to useful as a repository for other info. True, it's not as essential with the ability to put EQ in a window, but it's still tons easier to have the pertinant information for tradeskills, quests, spells, or what-have-you at your fingertips.

So, as long as we're wishing, I'd like to see:

- Controls (Back, Forward, Home)
that work within each story document. If the document has only one page, then they obviously go nowhere.

- To make the above work, the abililty to relate pages in an intelligent way. Each story mod could consist of one main page and a potentially unlimited number of dependent pages; for example, a main page named storyFletching and dependent pages named storyFletching-01, 02, etc. (the main page defaulting to -00). The Forward button would cause the window to automatically look for the next number in sequence and load it; the Back button would look for the previous number, and the Home button would load the main page. Internal links would still allow users to skip around, then hit the Home button to start over, or go Forward or Backward as they please.

- The ability to sort the available documents by date or name... and possibly by Category (which would have to be added). Possible categories could be Lore, Tradeskills, Quests, and Other. I think categories would have to be set by the user (like in the Journal).

- To go along with sorting, I'd like to see the Name be listed first, Date last (maybe just a personal preference, I dunno... makes more sense to me that way).
