03-16-2004, 04:54 PM
The window is to high and i cant make it smaller.
I assume you mean the window is too tall, not too high. The pet window is hard coded to be resized only horizontally, not vertically. So, to resize it, you will probably have to edit your .ini file:

How to resize the pet window vertically

Open up your UI_character_server.ini file in a text editor (e.g. UI_Xymarra_eci.ini). You have to do this when you are not logged in as that character.

Search for the pet info section.

In that section, find the Height tag. There will be one for every resolution you have used with that character (e.g. Height1280x1024)

Either change the height to your desired window height in pixels or delete the line completely to get the window's default height.

Save the file.

Log in as that character and check the new window size. If you like the layout, you can copy the .ini file for any other characters that need the same layout to save yourself going through these steps again.

I will also check the included .ini file and change the default height to a reasonable value to prevent this issue in the future.

Ok, I have figured out the problem. To allow people to have many yes/no dialogs up at once, they added new categories in the UI_character_server.ini file for popup dialogs. The first time you brought up a dialog after the change, it was called PopDialog0. If you bring up more dialogs, they are called PopDialog1, etc.. Each one is given the Width and Height tags of the dialog in the UI you are using at the time. The next time you get a popup dialog, it uses the size from your ini file instead of the size of the window, despite the fact that the confirmation dialog is not resizable. This is obviously a bug.

So, the fix for this is to find the headings [PopDialog*] in your ini file and delete the Width and Height settings in all of them, or just delete those categories completely - it won't hurt anything. The next time you generate a dialog, it will save the default settings for the UI you are using when you generate it.

The default UI uses a much larger confirmation box than mine, so if you want to use the default UI sometimes and my UI sometimes, you will need to get a bunch of confirmation dialogs set up with the default box size. Do this by:

First clear out the PopDialog settings from your .ini file. You can't be logged on as that character while you edit the .ini file or it will just be overwritten by EQ.
Load the default UI.
Turn on confirm-on-drop.
Pick up an item from your inventory.
Click on the ground a bunch of times.
This should generate a lot of PopDialog settings in your .ini file with the larger window size.

In game workaround to the confirmation dialog problem:
Before the rez is cast or before you try to loot the no drop item, generate a few confirm boxes as above. Then the one you want will be created with the default window size for your UI because it will create a new PopDialog section. You have to get the new box created before your first filler box fades or it will just go back to PopDialog0 settings.

Would it be possible to provide a patch to convert the 1600x1200 to the left-hand version? Or is this something I can do myself?
It is very easy. Only four files are modified for the lefty version. You could perform the conversion on any resolution or the mel&eacute;e mod.

Open up EQUI_BuffWindow.xml. Near the end of the file:

Replace:
<Location><X>640</X><Y>480</Y></Location>
with
<Location><X>0</X><Y>480</Y></Location>

and
<DrawTemplate>WDT_Right</DrawTemplate>
with
<DrawTemplate>WDT_Left</DrawTemplate>

Open up EQUI_CastSpellWnd.xml. Near the end of the file:

Replace:
<Location><X>0</X><Y>480</Y></Location>
with
<Location><X>640</X><Y>480</Y></Location>

and
<DrawTemplate>WDT_Left</DrawTemplate>
with
<DrawTemplate>WDT_Right</DrawTemplate>

Open up EQUI_HotButtonWnd.xml. Near the beginning of the file:

Replace:
<Animation>Panel_BottomLeft</Animation>
with
<Animation>Panel_BottomRight</Animation>

In the HB_ButtonBox screen near the end of the file,

Replace:
<Location><X>0</X><Y>25</Y></Location>
with
<Location><X>4</X><Y>25</Y></Location>

Near the end of the file,

Replace:
<Location><X>0</X><Y>480</Y></Location>
with
<Location><X>640</X><Y>480</Y></Location>

Open up EQUI_PlayerWindow.xml. Near the beginning of the file:

Replace:
<Animation>Panel_BottomRight</Animation>
with
<Animation>Panel_BottomLeft</Animation>
and
<Location><X>640</X><Y>480</Y></Location>
with
<Location><X>0</X><Y>480</Y></Location>


EDIT: The only important changes are the appearance changes. WDT_Left and WDT_Right are window templates that add the gold curly things to the sides of the hot button and casting bar windows. Panel_BottomLeft and Panel_BottomRight are background animations that create the green curved panels with gold borders. The location changes make the windows appear on the correct side of the screen when you load them with default positions. The hot button location change centers the buttons within the green part of the panel, because they are a pretty tight fit.

The pet window is hard coded to be resizable only horizontally. If you want to change the vertical size, you will have to edit your UI_character_server.ini file. Look for the heading PetInfoWindow. In that section, either remove the line Height1600x1200=X, which will cause the pet window to reset to its proper height next time you load it, or change the value of X to whatever height you want the window to have.

Also just curious is it possible to change the font size in the chat windows
You can change the font size in any window by clicking in that window and typing "/chatfontsize 4". I think the valid sizes are 1-6. Mine are all 2 or 3.

This will only change the size of text that is printed in the window. If you want to change the size of text that you see as you type it, you will have to edit EQUI_ChatWindow.xml. Search for Editbox item = "CW_ChatInput". In that section change the font from 3 to 4.

I have tested the new code as thoroughly as I could and I do not think it is possible to get stuck in that fashion. However, these constraints severely limit my ability to arrange the augment window components.
