Here's a basic rundown of what's in the various XML and graphics files. I was trying to figure this out myself and didn't find any good info so I went through them until I understood them (a little.)


AboutSIDL.doc - documentation that very briefly explains SIDL
- ElementTypes are basically different objects the everquest UI knows how to manipulate.
- Each window is basically a collection of objects called ScreenItems that are made up of various ElementTypes

EQUI.XML - defines which other XML files to load and in what order.

SIDL.XML - defines the various elementTypes that the EQ interface understands (eg what pieces make up a button, window, mouse cursors, etc.) Unfortunately exactly how these things *behave* isn't defined anywhere. It's just something you need to experiment with until you understand how the SUITE works (and how it's buggy.)

EQUI_Aminations.XML - this file basically deals with loading all the graphics files and cutting them up into little pieces to be used later.
- loads graphics from the tga, bmp and jpg files in your skin directory into TextureInfo items
- Creates graphic pieces (2DAnimations) such as your mouse cursor, class animation, window frames, buttons, etc by specifying the pixel locations X, Y, DeltaX, DeltaY within the TextureInfo items and piecing them together
- Labels all the little graphic bits so that they can be used in later files when defining ElementType templates.

EQUI_Templates.XML - this one creates basic elementType templates out of the graphics pieces loaded by EQUI_Animations.XML which are then refered to in individual window XML files.
- Graphics and other settings for standard windows, buttons, sliders, gauges, etc. are all defined in here.
- Graphics for these items are taken from the pieces loaded in EQUI_Animations.XML

EQUI_a window name.XML - these files basically define each individual window within the UI. The file contains the various parts that make up the window. They are usually made up of different ElementTypes usually cloned from a template defined in the EQUI_Templates.XML file and then tweaked by modifying various parameters such as color, size, position in ways specific to this window.


When modding the UI, people generally do something a little different. To make your mod compatible with the default UI, there are a few things that are important:
- If you don't have a file in your skin directory that EQ needs, it will take it from the default skin. To make your skin compatible, only modify the files you need to and don't have a copy of any other files in your skin directory. That way, if Sony changes something in the defaults, it won't affect you unless they changed the XML file you modified.
- Even though the EQUI XML files load graphics from EQUI_Animations.XML, you don't have to. To make your mod as modular (bad pun) as possible, make a custom graphics file, and custom template and load it in your window XML rather than modifying EQUI_Animations.XML
- If you are modifying lots of windows, then you might be better off modifying the EQUI_Animations.XML and EQUI_Templates.XML files to load custom graphics but doing so will be more maintenance (whenever Sony changes anything) and will make your skin a lot less compatible with other people's mods.
- If you want to change the way the default UI looks (colors) but want everything to basically be the same (structurally), you can probably just modify the graphics files. Look through the EQUI_Animations.XML file to find the exact pixel coordinates you'll need to modify. For example, there are some really nice drop-in spell gem (gemicons01 and gemicons02), spell icon (spells01 - spells05), and compass (window_pieces04) replacements.

Hope this helps give you a starting point for looking into the XML files and graphics.
