No, these story files do NOT go into your custom UI folder.. simply unzip the files so they keep the folder names in the zipped file and place them into your \everquest\storyline folder. They should then show up as a new story when you open your story window in your UI.

If they dont, then perhaps your custom UI has done some modification to the Story window by design and you may need to use the default UI to see the menus in your Story window.

Ok, went back and looked at the way the files unzipped and realized that the problem may be in the way you are putting those files and folders into place...

Example using LothBake:

Step one--- unzip files
Step two--- open the LothBake folder... move the file named "StoryLothBake" into the /storyline folder of the EQ directory
move folder named EQTRADERS into /storyline folder

That should allow the recipes to work as intended.

The author of the Loth recipe lists should have probably included a readme file in his download that explained this better... the folder named LOTHBAKE and LOTHBREW should NOT be kept but can be deleted after you have moved the storyLothBake and storyLothBrew files into the /storyline folder where all the others are.
