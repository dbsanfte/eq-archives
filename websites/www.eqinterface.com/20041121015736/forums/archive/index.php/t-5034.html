I made the same mod for myself and had the exact same problem.

The Problem:
Using /LOAD <MOD> 1 causes the windows properties to load from the characters ini files. Since your character already have an entry for the LOOT WINDOW in its INI file, it used that entry rather than looking at the XML mod.

The Solution
Go into each of your characters INI files and DELETE the LOOT WINDOW parameters. Then go back in game and loot something. This will create a new entry for the LOOT WINDOW in that characters INI file based on the XML code for whatever UI is loaded.
