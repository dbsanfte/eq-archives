One thing I discovered that screws up the way the game tiles backgrounds: If the background texture is of a non-standard size, or is called incorrectly.

What I mean is, tga's must follow the doubles rule... they must be 16, 32, 64, 128, 256, etc pixels on a side.

Normally, if you make a texture that is not a correct size, like 92x64... the game will just warp the tga to fit. But in the case of tiled backgrounds, I've noticed that it leaves strange artifacts like little boxes or circles.
Also, if the file is 64x64 and you call it as a different size, like in the animations file you call for it to be 32x64... the same thing can happen.

I hope this makes some sort of sense. =\
