There are many resources to editing xml files available but the tracking window may be a bad spot to put this.

I would really consider putting it onto a tab in the actions window.

XML is a definition file for objects. A tag is any item surrounded by <tag> followed by a closing </tag>.

Here is an example of a tiny piece of make believe xml.
<bowl>
<sizewide> 40 pixels </sizewide>
<sizetall> 40 pixels </sizetall>
<locationdown> 100 pixels </locationdown>
<locationover> 100 pixels </locationover>
<pictureofobject> mybowl.jpg </pictureofobject>
<specialstuff> sizable </specialstuff>
</bowl>

So if I were to read this piece of code in plain english
It would read: A bowl that is 40 pixels wide and 40 pixels tall. It is 100 pixels off the top of the screen and 100 pixels from the left of the screen. The graphic associated with my bowl is mybowl.jpg and it has a special tag that allows it to be resized.

The organization of the file is layed out so you can determine what is an object and what is a property of an object. Notice bowl is at the begining of the line and everything else is indented then /bowl finsihes the description of the object. All the lines between bowl and /bowl are properties of that object. Html works the same. The only difference is xml you can give any tag any name as long as the program reading the file knows what to do with the tag something will happen.. Html all tags are predefined. If you were insane you could type all your xml or html on one line.

The actions box has tabs along the top of it you could add one tab called tinker and put anything you like inside that tab. heck go crazy and add 5 tabs with classes of tinkered items or names of vendors locs or prices.

When you go to make your edit just find the code that creats a tab.. in the actions window in the actions.xml file
Copy a whole tab

<definetab>
<tabname> sometab </tabname>
<sometext> xyzzy </sometext>
</definetab>

Paste that copy into the same file...
<definetab>
<tabname> sometab </tabname>
<sometext> xyzzy </sometext>
</definetab>

<definetab>
<tabname> sometab </tabname>
<sometext> xyzzy </sometext>
</definetab>

Rename the copy of that tab to your new name.
Put you text into tags that are text descriptors.

<definetab>
<tabname> tinker </tabname>
<sometext> Put you text into tags that are text descriptors.
</sometext>
</definetab>

Near the bottom of the everquest xml file is a list properties and objects associated with the window you put your tab on. You will need to add one line to tell the window to actually draw the tab you just added up above.

<drawactionswindow>
<pieces> sometab </pieces>
<pieces> tinker </pieces>
</drawactionswindow>

I am sure if you examine the smaller xml files first them move up to bigger ones you will start to understand them.
Make tiny changes to the items between tags and try interperate a tag before you change it.

This is all from memory and I have never messed with the actions window so don't take it totally verbatim.
Good luck
