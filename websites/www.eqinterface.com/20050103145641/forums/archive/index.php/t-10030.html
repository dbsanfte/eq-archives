A lot depends on the complete set, some don't have issues with adding modified windows and some do. Very heavily modified UI's will be the least likely to accept other mods, due to changes in things like the animations.xml file and such.

In this particular case, it appears you have a duplicate label. In this case "TW_FireNumber" seems to have two instances. This is a fatal error, and will cause the default to load instead. There can only be one instance of a particular label.

There are two possible solutions:
(Note: Highly recommended you copy the UI's folder into one you can fiddle with - without the worry of totally destroying your original archives.)

1) Rename one instance of the label to something else. (IE: TW_FireNumber and TW_FireNumber2. You would need to make sure to edit both the begin and end part of the tags) then load it up and see if you get continued errors, make sure it looks ok.

2) Remove one instance of the label from the file(s).

Do not make a copy of your default folder. The only things that should be in your custom folder are the actual custom files. If you copy the default folder, it means that every single time SOE changes anything you will almost definitely have a non-functional UI.

As well, re getting the the error message in game (files not compatible, may not work correctly, etc) ... if you get a message like that, you want to find out what is causing it (find your UIError.txt log file in your main EQ directory) and get it fixed. While it may not cause a problem right now, it will later on.

A perfect example of this would be the mess a lot of folks were in last week. After the patch on 26 Aug, many people found that their UIs no longer worked. After looking at their UIError.txt log files, they discovered that the problem went back to a patch on 14 May. Until last week's patch, they were still able to load their UIs, but all had the "your files are not compatible", and had to retro-actively make those corrections. The same will happen to you sooner or later if you have that in-game message and don't fix it.
