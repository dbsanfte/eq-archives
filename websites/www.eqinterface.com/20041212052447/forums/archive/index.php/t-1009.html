There needs to be a better way to detect changes to UI files.

As an example, Lucy (http://lucy.fnord.net/index.html) provides a very useful Spell Changes service which details the differences between pre-existing spells and changes introduced with a patch.

A similar UI Changes service would greatly serve the community by eliminating the guesswork/mystery surrounding patches that change the UI. It only really needs to do a diff between one set of default non-graphic files and another.

Anybody want to tackle this project? It would be right up the alley for Unix programmers.
