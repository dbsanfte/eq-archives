I will try to be more specific in my instructions, of course I only know it helped me with my 9600XP so yrmv (your results may vary).
When you run the shortcut on your desktop, it runs the "EverQuest.exe". So to patch, goto your eq dir and run the "TestEverQuest.exe". You only need to patch until you see it FINISH getting a new "eqgfx_dx8.dll". Once it has gotten that file, hit cancel. Now you goto your eq dir again, find that file and copy it. I went back a directory and pasted it, for safe keeping. You could also paste it to the desktop.

To use this file on live servers, you have two options. First, for both you WILL need to run the patcher to fix any test files back to live. Once this is done, BEFORE you press play... paste that eqgfx_dx8.dll file BACK into your eq dir. Make sure you paste a copy and don't just move it, so later if you need to do it again you don't have to bother with the test server.

Now, your eq is using the test version of eqgfx_dx8.dll. If you EVER run the patcher again, it will over write it so here is our two options.

1) Run the patcher every time, and when its done copy the file into the eq dir.

2) Copy your everquest shortcut, then modify it to point to "eqgame.exe" with the word "patchme" at the end. The following is an example, based on MY system.

The current shortcut reads "C:\Games\EverQuest\EverQuest.exe" and I would modify it to "C:\Games\EverQuest\eqgame.exe" patchme

The "patchme" part HAS TO BE OUTSIDE THE "" in the shortcut. The "" let windows see file/dir names with spaces as a file, if you put the patchme inside the "" it would think you were looking for a file named "eqgame.exe patchme" which obviously doesn't exist. by putting it OUTSIDE the "", it becomes a parameter instead. Notice it isn't -patchme or /patchme, but just plain "patchme". Despite the "sly" naming, it actually tells eqgame.exe to run w/o patching




enjoy ati users ya now have targeting rings atleast temporary with a bit of copy and pasting...
