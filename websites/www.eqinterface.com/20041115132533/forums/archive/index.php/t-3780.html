Just wanted to get this out to all mod makers if possible. I thought this was a great idea that Statix ( Quartz Skin ) came up with.

<quote>
All of Quartz's files have been moved out of the EQUI_Animations.xml and out of the EQUI_Templates.xml file , they have been placed inside of a file called EQUI_Quartz.xml
This file is 'Called' from the EQUI.xml file like all of the other files, this allows Quartz to not 'Break' if verant changes graphics or tweaks a window, etc. With this setup the only way that Quartz should possibly break is if verant adds in a new 'Required' window , such as the RaidWindow.xml file.

By using the EQUI.xml file to load the Quartz files as an include, then Quartz+other mods will merge together easily, I think that ALL UI makers should use this method to make work on them less stressful and to make modding (or taking parts of mods) much easier :)
</quote>
