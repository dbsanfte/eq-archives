I am planning on making a few variations to the target indicator when I have some time.

The first will have the same 8 frame animation but will use the built in features in EQ to create each con color. This should cut down on memory usage because EQ will only need to load 8 frames, not the 48 current frames. The downside to this will be that the runes will glow a static color and the con colors that are drawn into the images will not be there so it might not be as bright as the original. The EQ Target Indicator engine may also put a tint on all the gold rings with this version. I'll have to experiment on it some and see how it looks.

The second will be the same image for each con color without any animations. This version will only have 8 images to load so it should not use that much memory.

The third would most likely be static and I would just remove the middle animated ring that rotates counter clock wise in the original version.

Just some of my ideas I've been thinking about since I finished it. Personally, I created it because I wanted a Target Indicator with some "Eye Candy". With that in mind I figure any variations I make will be tailored for low graphics usage for use on raids etc...

I like the simplicity the majestic ring but I did notice that the animation is jumpy whereas this one is smooth. I suspect its just a matter of frames.

Yes I agree the majestic ring is also very nice. Both basically work the same in that they have 8 images that create the animation, the only difference being the Elanthian Ring uses a different set of images for each con color. The only thing I noticed with the majestic ring and mine was that the animations were a bit jumpy. This is in fact due to a bug in the target indicator code in EQ. If you were to open the TargetIndicator.ini file you would see a line for each con value that says:

FrameCount=8

When I was creating the indicator screenshots for the red animation I noticed that it was skipping a frame. I replaced the images with ones that had a number for each frame and noticed that it was only displaying frames 1 thru 7. By setting the

FrameCount=[Number of Frames] + 1 (ie. 9 for 8 frame animation)

I was able to fix this problem. If you edit your TargetIndicator.ini file and change the FrameCount to 9 I'm pretty sure the animation will not be jumpy any more. Hope this is of help to you.
