That would create a situation where you would click on the group window and your pet window would fall behind the group window in function. IE the buttons are no longer useable untill you bring the pet window forward again by closing and reopening it.

Simply from what ive found there are 3 types of things in the EQ GUI.

Information elements. (gauges, labels, compass)
Function elements. (buttons, check boxes, search windows)
Graphical elements. (things that look pretty)

You can stack informative and graphical items almost freely because you dont need to click on them for them to work. Functional elements however must always be in the formost layer of your skin to work otherwise the UI thinks youre clicking on something thats in front of it.

05-12-2003, 04:38 PM
Right.


The outside borders of that chat windows were removed for looks. No border = no movement.

Open your EQUI_ChatWindow.xml file with whatever you edit with or notepad if you dont have an XML editor.

6 lines from the bottom is this...

<Style_Border>false</Style_Border>

Change it to...

<Style_Border>true</Style_Border>

Save your changes.

reload the UI by relogging or /load EQPDA 1

Your chat windows will now move. Once you have them where you want them, change it back to keep the look.


I'll add this to the readme for future reference.
