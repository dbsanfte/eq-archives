You don't need to add the graphic, EQ will pull it automatically from the default folder. The only reason I included the image in the patch thread was simply to show folks what had changed, so that if someone modified their dragitem.tga files, they would know what needed to be added/changed. (As far as I am aware, no one has ever modified the dragitem files, there would be no point to it.) The only thing you need do is make sure that the appropriate coding for the new dragitem is defined in your custom EQUI_Animations.xml file.

I think I know what Noqtue is referring to.

When I followed the patch fix posted, it listed some edits to the animations file. I did those edits in my mod file, but what I found is that I needed to go back and add the drag items for 26 and 27 deeper in the animation file. I had to have done something wrong previously.

Before I did this, my new bag from the expansion was showing up as a pearl necklace. I even made it disappear completely while i was editing.

Here is what I did:

<TextureInfo item = "dragitem27.tga">
<Size>
<CX>256</CX>
<CY>256</CY>
</Size>
</TextureInfo>

and...
<Frames>
<Texture>dragitem27.tga</Texture>
<Location>
<X>0</X>
<Y>0</Y>
</Location>
<Size>
<CX>256</CX>
<CY>256</CY>
</Size>
<Hotspot>
<X>0</X>
<Y>0</Y>
</Hotspot>
<Duration>1000</Duration>
</Frames>

both had to be pasted in their appropriate homes in the document.

Not sure what I did to cause this problem but hope this helps.
