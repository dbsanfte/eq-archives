SARS 4 does something a little funny with guage fill lines... it doesn't use them. It uses "solid" gauges which gives a rather unique look.

I know there are other UI's that do the same thing, so I'm creating a seperate upload of my pet window to handle that, rather than make people go into the XML and make more changes.

All fixed now, thanks for the heads up :) And thanks for the compliment!

Should be approved soon... enjoy!

EDIT:
If you want to make the changes yourself, rather than wait, open up the EQUI_PetInfoWindow.xml file and search on A_GaugeLines ... look for:

<GaugeDrawTemplate>
<Background>A_GaugeBackground</Background>
<Fill>A_GaugeFill</Fill>
<Lines>A_GaugeLines</Lines>
<LinesFill>A_GaugeLinesFill</LinesFill>
<EndCapLeft>A_GaugeEndCapLeft</EndCapLeft>
<EndCapRight>A_GaugeEndCapRight</EndCapRight>
</GaugeDrawTemplate>


You want to delete the <Lines> tag and <LinesFill> tag, so it looks like:

<GaugeDrawTemplate>
<Background>A_GaugeBackground</Background>
<Fill>A_GaugeFill</Fill>
<EndCapLeft>A_GaugeEndCapLeft</EndCapLeft>
<EndCapRight>A_GaugeEndCapRight</EndCapRight>
</GaugeDrawTemplate>


Save that and start EQ ( or just reload the UI ). That will do the trick for SARS 4 and any other UI that uses "solid" gauges ( no fill lines ).

Enjoy!
