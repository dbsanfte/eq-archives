There's really no way to fix it, but a patch from VI.

There is two alternatives, nither I really suggest unless you don't plan to edit UIs:

1) Change the line 'UISkin' in your eqclient.ini to point to your druid skin

2) Copy over the wnd_bg_light_rock.tga and the window_pieces[01-03].tga into the Default directory.

Problems with these:

1) If you're editing files. EQ will not have any 'default' skin to fall back on in case something goes wrong.
If your druid directory is missing a file, EQ *might* not load, that's unknown.

2) These files will get patched everytime you start-up EQ.

That's it .. Only work arounds, imho. Nothing I would suggest leaving as a 'default' setting. Just until VI gets the bugs fixed.

First, change the name of all the files that want to load from the default directory. For instance, change window_pieces01.tga to Druid_window_pieces01.tga

Second, ADD, don't overwrite the texture info to your EQUI_animations.xml It needs the filename and the size. Example:

<TextureInfo item = "Druid_window_pieces01.tga">
<Size>
<CX>256</CX>
<CY>256</CY>
</Size>
</TextureInfo>

Third, go through EQUI_animations.xml and EQUI_templates.xml, and change all mentions of window_pieces01.tga to Druid_window_pieces01.tga EXCEPT for the first part where you tell it the texture info, which I mentioned above.

Repeat the process for all the files that want to load from the default directory.

WARNING: You will go insane trying to do this manually. Use a text editor with a search & replace function. If nothing else, use DOS Edit.

WARNING 2: The XML files are case sensitive. Make SURE you have the exact same capitolization as the actual filename or it won't work.

make a copy of your default skin directory but with the name "druid" then copy all of the mods you want into that folder. Open your eqclient.ini file and scroll down a little ways till you find the line that says "UIskin=default" and make it read "UIskin=druid". Save the file and log back into the game. Once in the game type /loadskin druid true and camp out of the game to your desktop(the true command retains your current window position settings). Then log back into the game ad your new skin will be fully active. Note: there is a bug in the Bazaar with non-default skins. If you are in the Bazaar using a non-default skin and you type /bazaar, you will crash to the desktop. To remedy this remember to type /loadskin default true before you use the /bazaar command. Just type /loadskin druid true when you are done shopping and log out of and back into the game from the desktop.
