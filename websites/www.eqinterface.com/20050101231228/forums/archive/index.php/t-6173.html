01-31-2003, 07:56 AM
This is usually because of a graphics problem. To check and see if this is the case take a look in your Everquest folder for the Texture.txt file. If this is also growing pretty quick then the problem is that somewhere in your xml files of your skin there is a call to a XXXX.tga file that is not in your skin or the default directory. What ends up happening is that your xml file goes looking in the skin directory for the file and when it is not there is looks in the default folder. But when it is not there either it just goes in to an endless loop of looking back and forth between the two folders.

Check the Texture .txt file for it's size. In general it should be somewhere around 1-2K in size. If it is megs in size then this is your problem.
Chronwaas
01-31-2003, 08:02 AM
Opps, almost forgot. To fix this about the only thing you can do is look through all of your xml files for entries calling xxxx.tga. Where xxxx is the name of the TGA file. As you find all of these entries verify that the TGA file is either in your skin or in your default directory. If not then that is your problem.

Hopefully this fixes the problem. Unfortunately there is one other way that the problem could exist and with luck this is not the problem. The other way is that in your animation sections in your xml files(not just the Animation file though because a lot of custom skin xml files have the animation code right in the new xml files) the <Location> statment within the Animations section may be looking in a section of the tga file that does not exist. This is not what the problem is most of the time but if the first solution does not work then you may need to check the <Location>values for all of your animations to make sure they are inside the size limits of all of your tga files.

If you get this warning repeating:
[Fri Jan 31 09:06:16 2003] Warning: file UIFiles\FolderName\ not found in directory UIFiles\FolderName\. Attempting to use file from Default skin.

My explanation from my limited knowledge of 3d programming and the everquest client:

It's not loading info from a file into a Texture.. The UI doesn't even know that the texture is susposed to be there.. Thats why the filename is blank..

When the 3d Hardware draws the polygons (The triangles that make up everything in 3d) It doesn't know what to texture them with! So it just draws blank triangles..

So, you are 99.9% likely to be missing a <TextureInfo> block for a file that is trying to be used by your UI..



If you get this warning repeating:
[Fri Jan 31 08:53:05 2003] Warning: file UIFiles\test\FileName.tga not found in directory UIFiles\test\. Attempting to use file from Default skin
Every time it draws the item using the file it gives this warning..
Cause: It can't find the file in question to load into a texture
