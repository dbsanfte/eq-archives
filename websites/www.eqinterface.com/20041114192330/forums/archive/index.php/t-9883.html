Known issues:

The loot & give windows appear behind the inventory window when they are opened, preventing them from being seen. As far as I know this is not something that can be changed. I just press the "I" key or the Done button on the inventory window to bring them out of hiding.

The three buttons next to the spell casting window, above the class animation are unfunctional. This window was one of my earliest coding attempts, when I was still trying to make my UI have the pretty much the exact same layout as the old one, which I since gave up all notions of, since there's just way too many hard-coded roadblocks to such a goal. The spell casting window doesn't allow these types of buttons on it, thus why they don't work. I might split it in two and make the left side the window selector window, but then I'd lose the spellbook button, which doesn't function on the other window... <sigh> I'll probably end up just removing them.

The class animation shows a static warrior graphic, no matter what class char is logged in at the time. Opening the inventory window causes it to display the correct graphic, though once it's closed it is static as well.

The compass window keeps disappearing whenever something gets targeted etc. This is because it didn't seem to want to cooperate with what I wanted it to do as far as a background behind it, and I had to instead stretch the bottom of the target window down to fill in the background gap. I only really put it in because the actions window didn't let me do it the way I wanted to, so I ended up with extra empty space there, and I decided to squeeze the compass in. I just hit the keyboard command to close and reopen the compass when I want it, to make it appear.

The text on the hotbuttons, and some of the actions window buttons, is force-coded to be white, and is thus difficult to read. I told it to use dark brown text but it insisted on ignoring me. <sigh> I didn't feel it was a big enough nuisance to redesign my buttons to be dark instead of light; though I did decide the item info display window, that has the same issue, was too difficult to read and changed that background to be dark.

The actions window is in two layers, that I had to de-anchor from one another because of positioning problems I was having with the coding. Since the UI isn't designed to be moved about, this shouldn't be that much of a problem. If it does get hopelessly out of place accidently, I just reload the skin to reset the coordinates.

The spellbook window has a problem with spell names that take up two lines being hidden under the spell graphic underneath them. The spacing in between the lines appears to be forced to a fixed distance, so I'll tweak with the book graphics and see if making the book a tad bigger will correct the problem.

Planned changes:

The buffs window has been giving me nothing but headaches since I first tried it (but it was one of my earliest coding attempts when everything was pretty much still Greek to me so hopefully it'll be easier to deal with now) so I decided to release it with only minor changes with the beta. I will continue to attempt to adjust/design it so that all fifteen possible buffs can be seen, without taking up half the viewport area. The s.d. buffs window will also be changed.

Now that I've been using the newest coding changes for a bit, I've noticed that the quantity window isn't used just for what I thought it was when I designed it. I'll be de-standard positioning it (not that EQ paid any attention to where I told it to put it anyway...) - making it a bit smaller and changing the border so that it doesn't look out-of-place in the random spots it pops open.

The first windows I attempted to design are a bit messy, since I really had no clue what I was doing at the time, so I'll be going back over them looking for changes I want to make.

That's about all I can remember at the moment...
