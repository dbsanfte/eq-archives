Windows are drawn in a default order. Certain windows are drawn 'in front of' others, meaning that if the windows overlap, one will seem to be 'on top of' the other.

There are three windows that show their tooltips when you hold down the alt key: the castspell window, the buffs window, and the shortdurationbuffs window. The problem is that those three windows are drawn 'behind' most others so any nearby windows are drawn 'on top of' their alt-tooltips.

Clicking on a window brings it to the front when it gets focus, but it's temporary... as soon as you zone all the windows go back to the default order. Also certain windows like group window and target window tend to gain focus quite often, returning them to the front... but most windows don't.

The best fix is to make tooltips always appear 'on top'. The mouseover tooltips already do, but the alt-tooltips don't. Lacking that, you could at least change the window draw order so that castspell and buffs windows are near the 'front'.
