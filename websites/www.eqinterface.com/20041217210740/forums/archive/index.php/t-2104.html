kronas, you are probably chaning the size in the gauge definition in EQUI_TargetWindow.xml

That doesnt work.

You have to make your graphics the right size or at least set all the UI2D bits used by the Gauge _TEMPLATE_ to the size you need them.

The size that is given in the Gauge Definition (notice 'definition', not 'template'), is basically just a invisible 'cropping' rectangle, outside of which nothing is drawn.

I have myself had the same problem and I now have BIG gauges like the one on the patcher :)

I have defined UI2D's for the gaugefill, gauge background etc.. all the same dimensions, and replaced the endcaps as wlel, then used those instead of the default ones, so I get the size I want.
