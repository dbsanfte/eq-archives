Sometime today, hopefully, an upload for this UI will be approved in the Minimalist section (though it may be moved to the mix & match section, I'm not sure.) I can't post the zip file here, as it is too large.

I would really appreciate some feedback on this UI, as I know it has a few issues to be ironed out, and I can't get a few minor aspects to work on it.

This is a complete UI package I have been working on, with an emphasis on compact window elements and transparency of window elements, so as not to occlude the viewport whenever possible. Simply put, I hate clutter. I like to keep everything tucked off as compactly on the bottom and the sides as possible.

It has been tested and includes a UI_playername_server.ini file for 1600x1200, 1280x1024, 1024x768, and 800x600 resolutions. Simply change the name of the .ini file to include your characters name and server.

There are two Combat Ability Window options--one with four slots for hybrids and one with 8 slots for meleers. Simply rename the appropriate one EQUI_CombatAbilityWnd.xml.

***IMPORTANT NOTE***: This UI was designed to use the following elements other people have created:

Spellbook graphics by TKing
http://p206.ezboard.com/fmaximus94788frm10.showMessage?topicID=217.topic

Spell Gem graphics by TKing
http://p206.ezboard.com/fmaximus94788frm10.showMessage?topicID=219.topic

Class Animations by TKing
http://p206.ezboard.com/fmaximus94788frm10.showMessage?topicID=220.topic

Keilha's Compact Map Window
http://www.eqinterface.com/downloads/showthumbs.php?id=2697

Rax's AYA Media Player
http://www.eqinterface.com/downloads/fileinfo.php?id=2866

I didn't upload the files for these modifications, because I hadn't altered them, and to save space, however some of the other files in this package may reference them and cause failures if the above graphics and modifications are not installed. Of a certainty, the Book and Spellbook modifications will fail.

This UI has the following issues (that I am aware of):

The borders and/or titlebars are missing on the following window elements, despite the fact that the XML files have been modified so that they SHOULD be there, and the borders/titlebars show up when viewed in SidlWidl..

Hotbutton Window
Combat Ability Window
Loot Window
Alarm Window
Tribute Benefit Window

The background on the spellgems in the Spell Bar and Hotbuttons is black with a thin slice of the appropriate shade around the edge, rather than being entirely the right color. I am sure this has something to do with the way I fiddled around with the TGA files, but at the moment, I can't determine how to fix it.

The SocialEditWnd is supposed to be sizeable. I set it to be sizeable in the xml file, but for some reason, it's still not.

The Buttons on the Adventure Leaderboard window are missing. They show up in SidlWidl and disappear in game.

The offsets for inventory item graphics on the hotbuttons are off, so that the items look horizontally stretched. I am trying to figure out what these settings are, but they are different than the offsets for most graphics within buttons or spaces.

When you right click on a window to change it's settings, lock it in position, etc, the menu that appears is completely transparent, as is the menu that appears when you right click on the spell gem bar. This gets to be a bit of a pain, making the text in this window very difficult to read. I am still trying to figure out the settings that will make this window the transluscent gray of the other window elements.


The follow elements are untested as of yet:

The Health of Target's Target window. Ideally, this window should end up being like the Target window, in that it will only appear when a Target is selected. Unfortunately, I have not been able to put this to the test as of yet, seeing as I have been too busy designing the UI to group with someone with the HoTT ability. So, as of this moment, I have no idea if the HoTT window works, or where it is on the screen.

The Book window. I haven't had a book to open yet to test this. Ideally, it will look like the Spellbook window when opened.

The Note Window. I haven't had a note to test this element either. I fiddled with the TGA files a bit, and if all goes according to plan, there should be no "window" around the note scroll--just the scroll with a tiny "DONE" button at the bottom edge.

PVP windows. I don't play on a PVP server, so I haven't had a change to test these. Looking at them, it looks like maybe these windows are identical to the default Adventure windows, which means my mods for the Adventure windows could probably be applied to the PVP windows as well. But I'm not sure.

Please send feedback if you can confirm or deny that the untested elements are working, or if you know how to fix the aforementioned issues.

Thanks!
