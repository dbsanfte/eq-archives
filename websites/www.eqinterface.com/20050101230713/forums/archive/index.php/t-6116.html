The usual cause for the infinite uierrors.txt file is because of a graphics file missing that one of your xml files is asking for. It ends up looking for the file in an endless loop and just continues to right an entry to the error file. The best thing to do is look and see if there is also a texture.txt file that is quite large in your Everquest folder along with the large UIErrors.txt file. If the texture.txt file is also large then you can be pretty sure that there is a TGA file missing somewhere along the line.

Some one else here may be able to help narrow down what file it is looking for but I have always just used my debugging skills and looked through all the xml files I have in my skin for all entries of .tga. Then I just check to make sure that all of the .tga files that my skin is asking for are either part of my skin or are in the default directory. I uaully find the culprit and everything is fixed.

The best thing I can suggest if this is the case is to use the Search function in your xml editor(whether it is notepad or wordpad or something else) and search for all entries of ".tga". then make sure you have them all.
