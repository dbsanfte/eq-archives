This is the latest release of my /viewport UI for 1024x768 resolution. Completely updated for the LoY expansion and revamped to look better.

It is designed for a /viewport 0 0 842 600 setting with the UI windows on the right side of the screen and chat boxes on the bottom.

The zip file contains instalation instructions and a copy of a .ini file that can be used to insure the windows line up correctly.

The UI is primarily designed to be mouse user friendly, and for any class.

Download it by clicking here. (http://www.eqinterface.com/sdownload/fileinfo.php?id=1476)

http://home.attbi.com/~mbeiser/Images/Caleal1.4.JPG

The hotbutton window has a tabbed section that gives full access to all inventory and armor locations.

The group window includes all player stats and has large clickable areas for targeting self and group members.

The buff and song windows are compact with small icons, buff names, custom icon backgrounds and numbered buff slots.

The "target" bar is very long and has 10 sections to it. There is a casting gauge under the target lifebar that also has 10 sections to it. I find it usefull for heal timing.

The "target" window is made from the player window, so it will have a flashing red outline around the target and casting gauges when attack is on.

The inventory window is very compact and has no stats in it. The autoequip area is the class animation.

Containers have the primary, secondary, ranged and ammo slots in them.

The spell cast bar has a transparent background and number labels for the spell gems.

The chat window contains the template for its custom borders, as well as the templates for the borders for all other windows in the UI. All custom graphics animations used in the UI are defined in the xml code for the chat window.

The UI is fully compatable with the default EQUI_Animations.xml file. The custom one included with the UI is for the custom buff icon backgrounds. If a patch makes changes to the EQUI_Animations.xml file, you can just delete the custom one and the game will use the default backgrounds.

The modified xml files are:
EQUI_Animations.xml (only modified to use custom buff icon backgrounds)
EQUI_BuffWindow.xml
EQUI_CastingWindow.xml (reduced to size 0 0 so it is never visible)
EQUI_CastSpellWnd.xml
EQUI_ChatWindow.xml
EQUI_CompassWnd.xml
EQUI_Container.xml
EQUI_GroupWindow.xml
EQUI_Inventory.xml
EQUI_PlayerWindow.xml
EQUI_SelectorWind.xml
EQUI_ShortDurationBuffWindow.xml
EQUI_TargetWindow.xml (reduced to size 0 0 so it is never visible.)

Modified graphics files are:
window_pieces01.tga
window_pieces02.tga
window_pieces03.tga
wnd_bg_dark_rock.tga
wnd_bg_light_rock.tga
scrollbar_gutter.tga

Added graphics files are:
buff.tga (Custom buff icon background)
lunbar.tga (For the gauges in the "target" window.)
vertgauges.tga (For the gauges in the group window.)

Custom graphics files are from Raxthorn, Vertebrea, and Lunakin.

I have gotten a few emails from people complaining that they are unable to move the chat windows.

I don't use title bars in my chat windows because I think they are ugly and waste space.

Unfortunatly resizable windows with no titlebars cannot be grabbed onto and moved around, they have to be resized into the new location.

The borders on the chat windows are only 1 pixle wide so, depending on your mouse and drivers, you may have dificulty resizing the chat windows. Change your mouse speed to 0 and you should have no problem resizing the windows. /mousespeed 0

If I can make it work correctly without changing the looks of anything, I'll ad a 2-3 pixle wide transparrent title bar to the top of the chat windows soon.

03-01-2003, 12:05 PM
There isn't a custom spell book, or custom graphics for one, in this UI. The game loads the default one if you installed the UI correctly.

It sounds like you installed the new version over the top of an older version. Versions 1.1 and 1.2 had a variation of the vertmod EQUI_Animations.xml file and a custom spellbook and graphics for it.

Delete all the files in the directory you are installing my IU to, or start with a clean directory.

Extract all the new files to the clean directory, you will no longer have the problem.

Only the modified files should be in the directory you install the UI to.

Exept for the change needed to use custom buff icon backgrounds, I madeno changes to the EQUI_Animations.xml file.

You can install any spell book mod, or any other mod, you want, so long as it is a stand alone mod, or works with the default EQUI_Animations.xml file.

I'm going to be doing some updating this weekend to get rid of the error message that comes up when you first lot in or /load my UI.

The patch had no impact whatsoever on the function of this UI, or my UI for 1280x1024.

If you are having problems, make sure you have deleted the EQUI_Animations.xml file that was included in the .zip file for my UI.

If you still have problems, make sure that the directory you have my UI in ONLY has the files that came in the .zip file for the UI.

100% of the problems with the function of my UIs, that have been reported to me, were the result of people having files in the directory with my UI that were not suposed to be there.

It has been nearly 6 months of patches, and one expansion, since the last time I updated my "patch resistant" UIs. The only problem encountered, that "broke" my UIs, was fixed by deleting the custom EQUI_Animations.xml file. My UIs do not need that file and function perfectly with the default EQUI_Animations.xml file.
