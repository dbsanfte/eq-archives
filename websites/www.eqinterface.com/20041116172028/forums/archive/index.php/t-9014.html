Any custom UI is going to write to the UIerror.txt file. It is because not all graphic or xml files are included in the custom UI, so it is going to look in the default folder for the missing ones. These lines in the error file are not actually errors...more like warnings telling you that it didn't find such and such a file so it is going to use the one from the default folder.
