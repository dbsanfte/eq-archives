I'm not sure if you already noticed, it took me the better part of 2 days to figure this out....

SidlWidl sets boolean variables as "True" or "False" (if you change them with the combo box option) but XML requires the boolean variables to be all lower case, i.e. "true" and "false", it IS case sensitive...

So do the changes on those variables manually to get it right...

I've been searching for the reason why the window title didn't appear after setting the variable to "True" like mad till I finally figured out that it was case sensitive...
