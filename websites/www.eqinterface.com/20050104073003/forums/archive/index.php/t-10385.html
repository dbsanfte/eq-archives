I'm currently using mostly the latest Quartz gui (1_6_1) without the custom chat window and with NinQuartz's group window. This problem first showed up when I was using the Alyquartz gui and is still there with my current mix.

The problem is that clicking in the text entry box of any chat window puts the cursor in the "Always chat here" chat window. This is more an incovenience than anything since I have to keep track of every channel I'm in and start each message with /chat n or something similar.

With the default UI I'm able to click in any chat window and just type, sending the message to whatever the default channel is for that window.

I did a search and couldn't find anything describing this problem, my UI loads without any incompatibilities, and I haave no idea where this behavior is handled in the UI.
