I'm playing around with this on two systems I have. I'm starting to understand what it does.

When you create a library, it uploads basic information to your online library. Stuff like checksum (I think--some type of "auth" method. Checksum is most valid) and your client information (for p2p connections). When the check sum (or what ever) changes, the FolderShare application will prompt you to download a new version on a remote machine-it will also automatically upload it after you've edit it. When accepted FileShare opens up a pier to pier connection to Xfer the files across.

The Peer IP information is stored within the misc data that's stored on the remote library.

So far I'm unable to find a way to Sync up individual files that exist and must remain in a nonSync'd directory (ie the ini files). It seems to be all or none. FileShare doesn't even follow Sym links (shortcut in windows term).


This isn't too bad for basic functionalities. Can even be used for sync'ing websites, if FileShare gets a linux/unix port.

But for detailed syncing, select files in different areas, etc, not sure it's there just yet. However, it's getting there judging from the posts on the forums. They seem to be taking these feature requests seriously.
