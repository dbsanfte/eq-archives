The patch in the next post fixes this error:

If you are going to augment an item or do a quest combine, switch to the default UI before you open the container.

The tiny sticky containers currently have a bug that can cause the background to come to the foreground covering the container slots. When that happens, simply close and reopen the container. This should not cause a major problem unless you have placed a no drop item in a world container. If that does happen to you, don't reload the UI or close the container -- that would destroy the item. Your only recourse would be to petition for a GM to come take the item out of the container and place it back in your inventory.

I will try to fix this bug as soon as I can, but it might be a few days.
