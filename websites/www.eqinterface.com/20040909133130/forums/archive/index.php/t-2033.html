09-12-2002, 11:24 AM
Thats what allows the empty gauge to be visible on empty group members lines or mana on char that dont have mana. I wanted to show the empty gauge all the time (for the look).

For example, lets say your ungroup. Normaly, you would see the F* and 0% labels but not the HP gauges because EQ automaticly hides them when they are not used. So I made other gauges but not link to any eqtype. I didnt draw the fill partern on those so it looks like an empty gauge. And since its unlink, it shows all the time.

The catch is you have to "call" them before the actual link gauge in the screen definition (bottom of the xml) so they are drawn "before" the real gauge. When there is no group member on a line, only the empty gauge is drawn and when there is a group member, the real link gauge is drawn "over" the empty one, replacing it.

-------------------------------------------------------------------------

The biggest positive feedback I have so far is about the center target/grp/player windows that allow quick and easy monitoring of important data and the 12 customazible containers inv slots that allow fast and easy switch between resist gear and weapons.

Also, you may not notice it, but the /view 0 0 1024 600 was used (in 1024x768) so you dont have to make the bottom of screens windows transparent. Since casters need more screen space, they can try /view 112 0 800 600 (in 1024x768). It will make room on both sides to put windows while allowing you to not loose anything on the play screen. The drawback is you see in a smaller window. You will notice that most of my windows are design to fit exactly on the sides of the screen when using that view.

Be sure to explore the extra folder, you may find some interesting windows (not shown in the SS). I have an interesting inventory window in there for use with the /view 112 0 800 600

Every windows at the bottom of the screen are used by all characters. I will always try to put info that is common to all in that section. So it easyly free up screen space when your not a caster for example...

Xirin
09-14-2002, 07:03 PM
The full command is /viewport. It defines which area on the screen is usable for EQ, and which is black.
So, say my screen resolution is 1024x768. If I do
/viewport 0 0 1024 560
it will allow EQ to use the top 560 pixels of the screen (0 0 is left top corner, 1024 560 is bottom right), and the bottom 208 pixels will be black. I use something like that to put chat boxes on the bottom
