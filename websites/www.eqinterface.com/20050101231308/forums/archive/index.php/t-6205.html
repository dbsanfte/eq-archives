It looks like you have 2 windows in your UI that both have Ui2DAnimations named TK_ExpTube_Overlay.

It is actually quite easy to fix, just follow these simple instructions.

Go to the directory you have your UI in.

Find the file named EQUI_Inventory.xml

Right click on EQUI_Inventory.xml and click "Open with" on the popup menue, then click "Notepad".

The xml file will now open in Notepad so you can edit it.

At the top of the window Notepad opens in, click "Edit", then click "replace".

A little box will then pop up that has 2 boxes for you to enter text.

In the box labeled "Find", type in "TK_ExpTube_Overlay".(leave out quotation marks)

In the box labeled "Replace With", type in "IW_TK_ExpTube_Overlay".

Now click the "Replace All" button.

Now click "File" at the top of the screen and click "Save" in the drop down menu.

You are now done and should not get that same error anymore. :)

Try it and see if you get any more errors. Unfortunatly this particular problem causes the game to imediatly load the default UI without checking any further, so you could still have other things causeing errors. Just post your errors here and someone will help.

While Caleal's fixes work, and are technically correct, the better option (imo) would be to open the EQUI_Animations.xml file included in the Kao mod and delete the duplicated information. It is at the very top of the file, very easy to find.

The reason I suggest doing it this way is that t.king and Remelio include the coding in *all* of their inventories. So if you use Caleal's method, every time you want to try a new one of theirs, you will have to make the changes again. By deleting the duplicated code from Kao's Animations file, you will be able to drop in any of the t.king/Remelio inventories and not have this problem crop up on you again.

Attached is the exact information that you need to *delete* from the EQUI_Animations.xml file, should you choose to use this method.

Again, as I said though, Caleal's method *is* another correct way of resolving this error and I am in no way trying to disparage his suggestions.
