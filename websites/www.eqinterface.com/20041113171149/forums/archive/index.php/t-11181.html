(AKA: how to be over critical of your UI and how it loads)

tired of that "some of your windows might not be compatable" message? here is one solution to how to stop them all together ( at least untill there is a patch )

this works only with a UI that only gets warnings, no errors.

make a copy of the EQUI.xml file from your default folder, open your modded folder, make a list of the windows in your modded folder and sort them like seen in screenshot.

http://www.eqgui.com/attachment.php?s=&postid=58774

this is a variation on Quartz ../default/EQUI.xml

I have found useing the load key like Quartz has it, the game tends to try and load the default versions of the windows in your modded folder.
forming the EQUI.xml the way I have it formed forces the game only to load what it needs to. ( coder opinion, SOE has not approved this yet )

|||||||||||| warning for UI authors ||||||||||||

useing this in your UI, is risky, many people like to take a UI apart and mix and match, useing the loading code in this fashion does not allow other modded windows to be added without changing the loading code. please post this warning in the readme if you plan to use this key.
