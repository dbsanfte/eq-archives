I looked around and see that he only has the map as part of the total UI... find his UI in the FEATURED downloads for warcraft and try the following:

*** Copy EQUI_MapViewWnd.xml ***
*** Copy windows_pieces04.tga ***
*** Copy drakah_wow_map.tga ***

There is a chance your particular custom UI may not display the map properly, thanks to that windows_pieces04.tga file. (especially true if your UI uses its own file of the same name instead of the default version)

Looking at the XML file, it does not appear any other map-related files are drawn from his complete UI than those above. Try it, but be sure to first backup or copy the files with matching names if you see them in your custom UI, since pasting these into your custom UI folder will overwrite the ones already there. You want to be able to put those original files back if you find the copy/paste fails to produce the finished product you desire.

02-05-2004, 06:30 AM
EQKobara is correct. Those are the only 3 files related to the Map Window. If you are having client problems, I would suggest the following to test your system:

1) Backup and/or Delete your current UI_charname_servername.INI file from your EverQuest main directory. This way you start fresh.

2) Make a new folder in your "uifiles" directory, name this "newmap".

3) Copy those 3 files into that new folder.

4) Load EQ


When you get in game, you should be automatically using the Default UI.

Now type:
/loadskin newmap


If your client crashes, then it is not UI related as your computer is having problems parsing the data using the TGA graphics with your current videocard/driver. Although this is highly unlikely as 256x256 tga files have rendered fine even on old Geforce 2 cards.

You can check your "UIERRORS.INI" file to see if there was any errors in parsing.

Hope this helps in troubleshooting some.
