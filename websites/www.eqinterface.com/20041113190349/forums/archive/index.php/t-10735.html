Download it here (http://www.eqinterface.com/sdownload/fileinfo.php?s=&id=2614)

Wanted to post the GUI I am currently using. This was inspired by Forry, Vert, and my own prior work.

Rather than going through the extreme challenge of modifing Daellig 2 to meet everyones demands, I built this. It offers significantly more flexability. I have included my ini file if you want the same layout. Here is a preview of what it looks like with the map window up.

The entire goal was to be as efficient as possible while adding as much functionality as possible. The only difficulty some may have is with the pet buff window, song window, and buff windows. These all are effectivly perma fixed in their locations. They can be moved my GUI designers or by modifing the included ini file. Personally I like em where they are. The ini file is designed for a 1600x1200 resolution.

Lastly, the song window which you can't see since I am not grouped with a bard, makes the buffs appear directly to the left of the standard buff window which is on the right side of the screen. The pet buffs are on the left.

Guess that's it.

Ok, this interface is exactly what I have been looking for, but it has some inherent problems.

I tried both version, patched and unpatched from this thread, for melee and for casters respectively.

The spell window detached on the patch version is great, it should be like that for custom placement anywhere on my screen. I hate showing stats on my UI, i don't need to look at them, and will use inv win if I do.

The first probelm is the window filename usage. I have a large target window which I usually replace for any skin, and I cannot, since your target window is the clickies window. If I try to use my custom target as the player window (your target window) the skin will crash. Target should be the default EQUI_TargetWindow.xml for the target window. Make a new window name for clickies window EQUI_ClickiesWindow.xml and add it to the main EQUI.xml include list.


Secondly, the pet window is just the self player stats window, so the pet commands window is inaccessible altogether. Again, the stats should be a new window name, like EQUI_StatsWindow.xml and should be on the include list as well. This way it can be closed for those who do not use it. This way, the pet window can function normally as well.

Third, if you are a melee/pet caster like an SK, you need to see the pet window, and on the player/group window I need sta + mana bars.

Fourth, your target window should show the casting meter on it, so a cleric can see if the CH rate is happening fast enough as the target's HP's are decreasing. Currently the casting meter shows on top of the mana bar.


Aside fromt eh above, I love this skin, and would use it if it would work properly for my toons.
