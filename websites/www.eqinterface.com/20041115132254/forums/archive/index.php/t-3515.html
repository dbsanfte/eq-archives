I can't believe someone actually had the patience to create 100 gauges for this. But then, I can't believe I created a circular gauge that took 51 gauges..... anywho, yes, this is about the only way to make a completely vertical gauge. Problem is, obviously, alot of gauges and a nice big xml file. Another problem I've considered is that this might cause 'laggy' effects on less robust computer systems.

I'll give you a 'pat on the back' for actually sitting down and putting in all 100 gauges tho.

Oh, and if I'm not mistaken, you don't need a seperate animation for each line of the gauge. Infact, since each gauge encompasses a single percent of the total gauge, you could probably get by with a single animation that's 1 pixel high, and use the same animation for each gauge.
