99% of the time when people have a problem with my UI or another UI that uses a custom INI file, it's because they did something wrong or have incorrect settings on their PC.

Just for kicks, I downloaded some of the featured UIs on EQInterface and pretty much all of them (XymUI, AYAUI, Cairenn UI, all of Drakah's UIs etc) use CUSTOM INI FILES.

"What happens if I don't rename and copy the ini file correctly?"
The UI will still load, but none of the windows will be where the author originally designed them to be. Some windows like the Pet Window in SoesUI simply cannot be moved because of the way they are coded. The only way to have these UI's set up 100% in the originally designed way is to follow these instructions.

Instruction Manual: How to Install a UI with a custom INI file

Step 1:
Make sure your folder settings are correct
Open any windows folder and click on Tools: Folder Options
http://www.abouttahirihinc.com/pic/soeshelp1.jpg
Make sure that "Hide File Extensions for Known File Types" is not checked!
http://www.abouttahirihinc.com/pic/soeshelp2.jpg
I have had people not install my UI correctly because they had this option checked (it's checked by default when you install Windows), so what was happening is they were renaming the file correctly but they weren't able to see the extensions. So their files ended up being "UI_Character_##.ini.ini" and hence not working correctly.

Step 2:
Unzip and install your UI to the UIFILES sub-directory within your EverQuest Folder.
http://www.abouttahirihinc.com/pic/soeshelp3.jpg
The program I use is WinZip, you can get a free version from www.winzip.com. Make sure "Use Folder Names" is checked, this will automatically create the "Soes" subdirectory.

Step 3:
Browse to your EQ Directory and into the UiFiles/Soes sub-directory.
http://www.abouttahirihinc.com/pic/soeshelp4.jpg

Step 4:
Locate the custom ini file "UI_YourChar_YourServer.ini" and rename it according to your char's name and server number.
http://www.abouttahirihinc.com/pic/soeshelp5.jpg
If your char name is Bob and you play on Veeshan, your file should be renamed to UI_Bob_35.ini ... if you don't know your server number find it here (http://www.eqinterface.com/modules.php?op=modload&name=FAQ&file=index&myfaq=yes&id_cat=8&categories=Server+Numbers).

Step 5:
Finally copy the file to your base EQ Directory.
http://www.abouttahirihinc.com/pic/soes-install-help.jpg
If you've ever played the char you are trying to setup the UI for, it WILL ask you to overwrite your existing ini file... if you don't get a box like this, you did something wrong (or deleted the existing file already). Select "Yes".
http://www.abouttahirihinc.com/pic/soeshelp6.jpg

Step 6:
Log into EQ, but make sure you are in one of the supported resolutions.
For SoesUI you have to use either 1280x1024 resolution or 1024x768 resolution, no other resolutions are supported nor have been tested so if you use another one you're on your own. So goto Options before you log into EQ and check to make sure!
http://www.abouttahirihinc.com/pic/soeshelp7.jpg
Similarly, if you were installing another custom UI make sure you use the supported resolution(s) the author specified.

Hope this helps some people out. Good luck.

OK, was just wondering if you were planning on doing yet another patch to fix the problems that have been incurred with the LDoN patch (the UI isn't working anymore, I plan on copying over any missing files from the default directory to my Soes directory later today).

To the people who mentioned that the spellbar and the buff window are off the screen, here's what I did:

1) Type "/loadskin default 1" (no quotation marks, of course -- this brings you back to the default UI, but the "1" keeps all windows in place)

2) This makes unlocking and moving windows much easier since the title bar is back. Right click the title bar on both your spellbar and buff window, unlock them, then move them towards the center, aligning the appropriate edge to the edge of your viewing area (i.e. the edge of your screen). You can relock them if you wish, but you may want to wait.

3) Type "/loadskin soes 1" (once again, no quotation marks and replace the word "soes" with whatever you have named your Soes UI directory. In my case, the directory is named "soes" so I just tyep in /loadskin soes 1). This will revert you back to Soes and your spellbar and buff window should now be viewable. If the alignment isn't perfect yet (in other words, you're anal retentive like me and want the edges to be perfect), you can CAREFULLY right click in an unused area of the window (for the buff window, right click where the names of the buffs are or in that general area -- for the spellbar it's a bit more tricky, you have to right click in the area in between spell gems) and unlock the windows (provided you locked them back in step 2). Click and drag the windows (the area to click and drag to move the windows is the same area where you were able to right-click the window and get the menu to appear) to where you want them and relock them (so you don't accidentally drag them away).

I know step 3 looks long and drawn out, but it really isn't, just a lot of explanation which is common sense and a lot of it is in case you didn't get the windows properly aligned in step 2.

Hope that helps! If it only makes you more confused, well then that's my fault for being so confusing. :p
