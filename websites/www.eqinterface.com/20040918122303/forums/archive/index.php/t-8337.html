Haliken
05-08-2003, 12:53 PM
Load time is probably in the realm of 1 to 2 seconds longer in EXTREME cases. My RAM is standard SDR (not DDR) and MY UI (Enoks UI v6) which uses window independent coding takes a few seconds to load. This UI (Enoks UI tking dedara v2) loads faster than it, and, even after I make the windows independent, won't load or run any slower since each window has its own graphics anyway. Loading the actual texture files (*.tga) is what takes time and memory (also, processing the XML takes time, that's why Enoks UI v6 takes longer to load), cutting them up into components takes very little. To make these windows independent I'll have to set up the color-coded squares and gauge graphics in each window graphics file, which I have plenty of blank space (in each windows texture file) to do.

Basically, in the next week I will make the windows work independently of each other, write detailed instructions for mix&matching, and make a mirrored version of this UI.

Enok

P.S. - You will be able to mix&match elements from the normal and mirrored UI to suit your tastes.
