Hi All,

For those of you having problems with setting this compass up, I just thought I would set down what changes I made to the files that are used by the compass.

Basically I think the problem would be in the EQUI_Animations.xml. The file I used is with all the expansions enabled. Also, if you are running an already existing UI that uses the animations file you will need to cut and paste the code changes to the animations file to the file that you are currently using.
The changes to the animations file are as follows:

<TextureInfo item = "compass.tga">
<Size>
<CX>255</CX>
<CY>255</CY>
</Size>
</TextureInfo>
<Ui2DAnimation item = "A_CompassStrip">
<Frames>
<Texture>compass.tga</Texture>
<Location>
<X>0</X>
<Y>94</Y>
</Location>
<Size>
<CX>203</CX>
<CY>94</CY>
</Size>
</Frames>
</Ui2DAnimation>
<Ui2DAnimation item = "A_CompassOverlay">
<Frames>
<Texture>compass.tga</Texture>
<Location>
<X>0</X>
<Y>0</Y>
</Location>
<Size>
<CX>203</CX>
<CY>94</CY>
</Size>
</Frames>
</Ui2DAnimation>

The only real change that might effect anything is changing from windowpieces_04.tga to the new compass.tga file. This is done in the Texture element. If you have existing UI mods that use the animations file, just delete the existing compass code (found at the bottom of the file) and replace it with the above lines of code.

The changes to the Ui2DAnimation are only changing the Texture that the strips use and the size and position within the Texture file.

The other changes to the Compasswnd file consist only of changing the size and location of the Overlay and Strip.

If you dont have all the expansions enabled and arent using any other UI Mods that use the animations file then I suggest you make a copy of the file from your UIFiles\Default directory and make the changes to the code that way. (Ie. replace the compass code with the code supplied).

I hope this helps.
