Thanks for all the comments guys. Ill try and answer your questions as best I can.

Q1. Will I make versions for other resolutions?
A1. Atm I'm really busy with my classes and I dont have the time to invest into doing that. It requires rescaling the size of all the background images to fit in those resolutions, and I would have to do them for each resolution. The default file only uses a texture to fill each window, were as WinMode uses an image of each window as its backgroud, and thus cannot be rescaled without resizing the images manually. This would also require recoding all of the texure size declarations in the animations and template files for each resolution. Basically it means i would have make avalible a version for each resolution. For that reason I tried to choose a size that seemed like a happy medium 1024x768. If a lower resolution is what you looking for, plz refer to Caleal's post. He is correct. By using the viewport command to make the window 671x510, you are actually running a much lower resolution than the what the UI is set at.

Q2. Error Loading player window using SidlWidl UI editor?
A1. Im not sure why you are getting an error loading the player window. I believe that program is still in beta phase so it might be a glitch. I coded all of my files manually using Peter's XML editor so im not real familiar with how SidlWidl works. Also, I'm not quite clear on what you are trying to do to the player window. The percentages are there already for health and mana. What are you trying to add to it?

Q3. Actions window toggle in selector window?
A3. Yes im aware it doesnt appear. I could only fit 8 icons in the window so I chose to push the actions window button off the screen since it was my intention to always have it on in the design of the UI. You can still hide the actions window using Alt+M but It will look pretty weird without it.

Q4. Can't move actions window?
A4. The problem with the actions window is that you need to have the borders turned on to be able to grab hold of it. Because of the multi page setup in that window, you cant just grab the window itself. Since my actions window has no boarders it can't be moved but it is set to load in its intended spot.


Q5. Is there a necro animation?
A5. The class animation is dynamic to the class your using the UI with. I believe all animations except beastlord exist and will show. There is a minor bugg with the animations though. It usually loads the warrior animation as default and doesnt load the proper animation until you open the inventory window. I believe this is a minor glitch in VI's coding. Its really something you would never notice unless you put the animation outside the inventory window.

Q6. Will I add a casting window?
A6. Not sure what your talking about here. If its the cast time window you speak of, then it is already there. It is the default window and appears only when you cast a spell. I plan on keeping it that way, but I will be adding a background to it that matches the one in the original windowed mode. I could make it permanent in the player window but my intention of this UI is to keep it as similar to the old UI as possible. I want it to fit like an old shoe. Even with that said though, this is something I definately will consider doing.

- Character Picture would have to be added to the background image file manually using photoshop or a similar program. VI took out the char picture in the old UI as well so you would have to add a static image of your choice.

- To get rid of the old style arrows, you would have to use the default window_pieces03.tga file. You would also loose the old style buttons. You could also use photoshop to post the default style buttons from the default file if you wanted to keep the current buttons.

-You could also extend the selector bar background image file using photo shop. Just copy and paste a section to lengthen it. Then use a smudge tool to blend the textures so it looks like a solid pieces. After that you will have to increase the length of the window in the EQUI_SelectorWnd.xml file.

-if you /load winmode, the actions window should automatically go in the bottom right hand corner, then all you have to do is like up the other windows above it. If you use the /load winmode 1, it will not go to its defined location.

Sounds like most of the changes you want to apply require some photo work.

i fixed it.

all you have to do is make the changes in the general 2/24 fix section of the patch fixes board on this board, then dont forget to add a call for charms in animations.xml, this is what you need to do, it was forgettin in the 2/24 post:

find <Ui2DAnimation item = "A_InvEar"> in animations.xml

right above it paste:

<Ui2DAnimation item = "A_InvCharm">
<Frames>
<Texture>window_pieces04.tga</Texture>
<Location>
<X>180</X>
<Y>0</Y>
</Location>
<Size>
<CX>40</CX>
<CY>38</CY>
</Size>
<Hotspot>
<X>0</X>
<Y>0</Y>
</Hotspot>
</Frames>
</Ui2DAnimation>

hope this helps
