In this section of the fix:

<Ui2DAnimation item = "A_AttackIndicator">
<Cycle>false</Cycle>
<Frames>
<Texture>TargetBox.tga</Texture>
<Location>
<X>6</X>
<Y>2</Y>
</Location>
<Size>
<CX>116</CX>
<CY>28</CY>
</Size>
</Frames>
</Ui2DAnimation>


Change the size to look like this:

<Size>
<CX>0</CX>
<CY>0</CY>
</Size>

That'll make it so that the flashing bar doesn't show up at all, until the author of your UI gets a chance to edit the .tga file to fit their window.

Re the lag, I think I've found the problem everyone has been having, see the stickied thread at the top of this forum, or the "Addendum" thread in the Patch Fixes forum.
