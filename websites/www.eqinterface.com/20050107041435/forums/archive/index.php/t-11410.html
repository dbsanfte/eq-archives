I don't know if this helps or is totally correct but this worked for me. The target indicator seems to be an auto fill type of graphic, similar to the flashing border.

To make it work, you must manually adjust the graphic definition to the size and shape (alpha channel) you want. Then when you call it using the static animation lines, it is sized 1,1. This makes it only show when you actually select a target. No matter where you move it, it seems to be hard coded to start drawing (displaying) the graphic from 0,0.

To see what you will get in Sidl, resize the static animation to your definition size. This will show you how it will look in game. But make sure to change it back to 1,1 or you will see the graphic even when you don't have someone selected.

Hope that helps.
