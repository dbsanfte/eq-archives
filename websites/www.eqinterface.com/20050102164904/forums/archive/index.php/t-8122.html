If using the default Animations File (loadskin default) allows you to see the icons, then one of two things is (could be) wrong in the custom folder.

The Animations File
The TGA file associated with the Icon.

To Test:
1) Rename your Custom\Animations File.
Loadskin Custom
2) Rename the Dragitem.TGA files in Custom
Loadskin Custom

One of these should allow the skin to load with the Icon intact and assist you in identifying the problem area.

Also should check your UIErrors.txt file. It may indicate a missing something that is not causing an absolute crash....

04-24-2003, 08:11 AM
From your posts, I assume that you copied your default folder to the custom directory and then installed the modifications over that.

You really should delete the files that are duplicated in the custom folder. Only the modified files should be in custom as the UI will automatically see that it needs to refer to the default folder for the stuff that is missing.

(IE: if you have the exact same dragitem#.tga files in custom...delete them. The ones in the default folder will be used. If you have the same exact EQUI_Animations.xml file in the custom folder, delete that. Once again the UI will do what it needs to refer to it.)

Check out the FAQ on UI loading. This will greatly assist you in making sure that only the file you have and want modified are modified and the others are updated as needed when patches come through.
