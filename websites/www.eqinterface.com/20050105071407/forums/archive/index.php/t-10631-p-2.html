OK I think I found the problem:

In the AYA EQUI
<TextureInfo item="AYAepic.png">
<Size>
<CX>128</CX>
<CY>128</CY>
</Size>
</TextureInfo>

This needs to be changed to:
<TextureInfo item="AYAepic.tga">
<Size>

This is the "Cursor line in the file:
<Ui2DAnimation item="A_DefaultCursor">
<Cycle>true</Cycle>
<Frames>
<Texture>AYAepic.tga</Texture>
<Location>
<X>0</X>
<Y>0</Y>
</Location>
<Size>
<CX>37</CX>
<CY>45</CY>
</Size>
<Hotspot>
<X>1</X>
<Y>1</Y>
</Hotspot>
<Duration>200</Duration>
<CX>128</CX>
<CY>128</CY>
</Size>
</TextureInfo>

As you can see the Line points to AYAepic.tga Yet the "Texture Info" Points to a AYAepic.png.

If you make the above change your cursor of choice should work.

I used to change cursor myself to use my own, But I like the one used by Kanon in this update.

You can use "Notepad" to make the changes.

Hope this helps :rolleyes:

The work around for combining augments is to

1. Exit Everquest
2. Change the name of your ui file in the main Everquest folder that has to the naming convention charactername_servername to a new name like characternamebak_servername.
3. Start EQ.

This will force EQ to create a new ui file for your character which will support augment combines. After you do the combines,

1. Exit Everquest
2. Delete the charactername_servername and rename (F2) characternamebak_servername to charactername_servername.
3. Start EQ.

You may think hey I can shortcut the process by just ingame switching UIs but that will not work. Hope this helps until the UI is patched to support augment combines.

For those who want to change their colors back to the original or to something else they'd rather have, You can search through any of the files with gauges in them. Now, just look for the different gauge sections in the XML (I'd do it here, but I'm away from home and was unable to bring EQ with me. :( *Cry*) and in the code you'll see a section saying FillTint and LinesFillTint, IIRC. Inside that you'll see the RGB numbers <R>###</R>, etc. Change those three to whatever you'd like. The lines fill tint will change the color of the lower boxes on the bars (the darker grey in the picture). That's not on all of the gauges, though and I'm pretty sure this doesn't work with the rainbow gauges (group and target). Fiddling with the colors, unless do something like put a letter or something instead of a number, shouldn't interfere at all with the UI, so feel free to experiment and see which colors you'd like. You can do this with the casting gauges, as well. Just have to find 'em. I usually just open a file, do a find for "gauge" and just look for the Gauge Item sections, then redo the colors. Hope this helps!

There is a way to change this:

Use notepad (or Text Editor of choice) and change the Selector Window XML. You will be replacing the Journal Button with the Help Button. The Journal can be called up using Alt J.
Once the change is made the Help Button will show in the Journal Button Location, But it will be a Light Gray Color ( I don't know how to change that).

Changes you need to make(Within the Selector Window XML File)

Find.....
"SLEW_HotButton"
<X>420</X> change to 375
<Y>16</Y> change to-13 (That's Minus 13)

Find.....
"SLEW_JournalButton"
<X>375</X> change to 420
<Y>-13</Y> change to 16
Hope that helps :rolleyes:
