Well, I finally decided to do a complete UI. I'm placing it here in the Beta forum because there are a few things I want to fix yet, but figured I would let anyone try it. I have not uploaded this to EQInterface yet though, so I will link to my Tripod site for anyone who wants it.

Remember, this is a Tripod site...so you might get an extra pop-up window and there is a bandwidth limit. Just try a little later if you have any problems.

Specs:

- The UI was designed at 1280x1024, but the type can be a little small at that size (I like it like that!) Should work on other sizes.

- Two timers included: 5 and 20 minutes - These are part of the Target window and are the slide timers from Arista's Webpage (I just modified the code to fit the window)

- I tried to make as much as I could be see through. The chat windows can be transparent or have a background color.

- Included are t.kings G2 spell icons

- There is a fixed mana bar on the player window, so this may cause a problem for the pure melee players. (I'm working on this)

- The UI "should" be up-to-date with the upcoming patch (May 14th) I have changed my naming convention to that of the test server, so unless that changes, it should work.

- I have included a custom EQUI.xml file, but that should be a lot easier to fix from a patch than making changes to the Animations.xml file.


Hope everyone likes it, but let me know what you think. Thanks.

Actually you can make a patch proof EQUI.xml file using this code:


<?xml version = "1.0"?>
<!-- Monolithic Parameter File -->
<!-- The composite section must come first and be followed by the schema -->

<!-- defines for gauges set the EQType tag in Gauge definitions to one of these numbers -->
<!-- GAUGETYPE_HP 1 -->
<!-- GAUGETYPE_MANA 2 -->
<!-- GAUGETYPE_STAMINA 3 -->
<!-- GAUGETYPE_EXP 4 -->
<!-- GAUGETYPE_ALTEXP 5 -->
<!-- GAUGETYPE_TARGET 6 -->
<!-- GAUGETYPE_CASTING 7 -->
<!-- GAUGETYPE_BREATH 8 -->
<!-- GAUGETYPE_MEMORIZE 9 -->
<!-- GAUGETYPE_SCRIBE 10 -->
<!-- GAUGETYPE_HP_PARTY1 11 -->
<!-- GAUGETYPE_HP_PARTY2 12 -->
<!-- GAUGETYPE_HP_PARTY3 13 -->
<!-- GAUGETYPE_HP_PARTY4 14 -->
<!-- GAUGETYPE_HP_PARTY5 15 -->

<XML ID = "EQInterfaceDefinitionLanguage">
<Composite>
<Include>..\default\EQUI.xml</Include>
<Include>EQUI_Forry.xml</Include>
</Composite>
<Schema xmlns = "EverQuestData" xmlns:dt = "EverQuestDataTypes"/>
</XML>


I HAVE NOT tested this, but I believe this was the way to do it...
