I recently completed a project aimed at displaying EQDKP’s DKP standings inside EQ through a UI window (currently the feedback window). I figured that maybe this would be a good place to post in case others were interested.

The projects itself can be found at http://www.lokorin.com/dkplu/
It can be downloaded at http://www.lokorin.com/dkplu/download.php
A readme that explains how it works (with pictures): http://www.lokorin.com/dkplu/readme.php

It’s a program that (once correctly configured) downloads the EQDKP standings, converts it into an UI file and (optionally) launches EQ while closing itself down. All in all it causes no extra work/clicks (once configured) and always provides an up to date DKP list that can be accessed at any time through one of many different UI windows.

It is possible in the current version but it includes a bit of extra configuration.

The simplest way in the current version to get both lists at the same time would be to make two instances of the program (in separate folders). Then set one to list the names and current DKP and the other to list the names and total DKP in the EQDKP reader tool (select the row with total DKP instead of current DKP). Then make the two programs write into two different windows (i.e. bug and feedback) and make the first instance automatically start the second one and close itself. Then set the second instance to start EQ and automatically shut down it self.

It would all result into something like this:
The first instance starts. --> The first instance writes current DKP list into the feedback window. --> The first instance automatically starts the second instance while shutting the first instance down. --> The second instance writes the total DKP list into the bug window. --> The second instance automatically starts EQ and shuts the second instance down.

So it would involve an extra bit of configuration but no extra work running it. I will keep it in mind for the next version though so a user can add additional columns to the list.
