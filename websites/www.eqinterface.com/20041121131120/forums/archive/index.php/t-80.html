For those of you who are having problems getting a certain XML file to load in SidlWidl her is what is posted by the creator on his message board:

Some files will not open if they are not 100% Xml conformant. EQ does not require this, but I do for now. What this means is that a few of the mods out there need to be manually edited to insure that the ?xml version="1.0"? tag is the very first, and the root XML tag is next. That's been the only problem I have found so far.

I have found EVERY time a file wont open there is a coment entered by the author in the first 3 ines or so.
just remove them and it will load fine.
here is what the top of all my XML files look like:

<?xml version="1.0" encoding="us-ascii"?>
<XML ID="EQInterfaceDefinitionLanguage">
<Schema xmlns="EverQuestData" xmlns:dt="EverQuestDataTypes" />

if it looks like this your window will open in SidlWidle...
Hope that helps

Tuoneves
