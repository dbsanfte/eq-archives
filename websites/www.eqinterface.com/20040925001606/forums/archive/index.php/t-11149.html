Thought I should just put this out there so those people modifying there 3D target indicator would know. When EQ looks at the TargetIndicator.tga, it doesn't display the bottom 16 pixels. These bottom 16 are cut off where the circle meets itself...so for anyone making a moving ring, those bottom pixels will make an offset.

It's kind of hard to explain, but here are some pictures that I hope will help:

http://jason.forryweb.net/ui/3_swoops.jpg
This is a clockwise moving 3 pronged swoop that is smooth at the intersection of the ring.

http://jason.forryweb.net/ui/3_pronged.jpg
This is the TargetIndicator.tga that is used to create the above picture. See how I left the last 16 pixels black and made everything line up for the first 240 pixels instead of 256.

Hope this helps when creating those rings!!
