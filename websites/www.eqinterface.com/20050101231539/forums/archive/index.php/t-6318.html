02-05-2003, 12:14 PM
Make sure that your vert version of the EQUI_Animations.xml file has

<TextureInfo item="lines.tga">
<Size>
<CX>32</CX>
<CY>32</CY>
</Size>

in near the top.

Dont copy the EQUI_Animations.xml file from the default to your vert custom directory. You may have done this. It doesnt work.

You need to manually add the new lines which VI create into your custom EQUI_Animations.xml file.

Hope that helps.

Tils

Using your text editor's search or find function, check your EQUI_Animations.xml file. I have a suspicion that you will find the coding definitions for the Song buttons in it, AS WELL as in your selector window, hence the duplicate item error.

If that is the case, there are two easy solutions:

Either rename the duplicate coding in the selector window to change the name (ie A_SongBtnNormal to SELW_SongBtnNormal, or some such), remembering to also change it down in the pieces section,

or just delete the duplicated code from the selector window.
