From what I've seen so far there are no numbers in the XML file that you can change to move the actual color graphic. It appears to be hard coded to display the color image starting in the upper right corner of the target window.

So far the only fix I've been able to come up with is to edit the TargetBox.tga image and adjust the alpha mask. I basically went in and changed the size of the default image so that it covered the entire target window then masked the parts I didn't want colored to transparent.

There was a tutorial on here that explained how to go about editing the alpha mask of the images you might look at that and see if you can figure out how to tinker with your picture.
