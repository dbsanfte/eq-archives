I looked through the zip file of your groupwindow.xml, but I couldn't find anything in it that referred to the tube gauge overlays you want, so I'm just going to start from the beginning.

The first thing you need to do in your Groupwindow.xml file is declare the tkinvgauges.tga file, like so:
<TextureInfo item="tkinvgauges.tga">
<Size>
<CX>256</CX>
<CY>128</CY>
</Size>
</TextureInfo>

Then you need to declare the properties for each image in the tkinvgauges.tga file that you wish to use.

The tkinvgauges.tga file that KaoUI uses contains two different gauge graphics. The code in KaoUI refers to the top one as the Exp Tube Overlay, and the bottom one as the AltAdv Tube Overlay.

If you want to use the Exp Tube Overlay, declare its properties like so:
<Ui2DAnimation item="TK_ExpTube_Overlay">
<Frames>
<Texture>tkinvgauges.tga</Texture>
<Location>
<X>0</X>
<Y>30</Y>
</Location>
<Size>
<CX>149</CX>
<CY>34</CY>
</Size>
<Hotspot>
<X>0</X>
<Y>0</Y>
</Hotspot>
</Frames>
</Ui2DAnimation>

If you want to use the AltAdv Tube Overlay, declare its properties like so:
<Ui2DAnimation item="TK_AltAdvTube_Overlay">
<Frames>
<Texture>tkinvgauges.tga</Texture>
<Location>
<X>0</X>
<Y>70</Y>
</Location>
<Size>
<CX>160</CX>
<CY>28</CY>
</Size>
<Hotspot>
<X>0</X>
<Y>0</Y>
</Hotspot>
</Frames>
</Ui2DAnimation>

You can declare both if you wish to use both graphics, just be sure that you refer to the proper animation (I'll explain further in a minute).

Next you have to tell your UI what the fill for your new gauges is supposed to look like, by adding the following lines to your groupwindow.xml file:
<Ui2DAnimation item="A_TubeGaugeBackground">
<Cycle>true</Cycle>
<Frames>
<Texture>tkinvgauges.tga</Texture>
<Location>
<X>0</X>
<Y>100</Y>
</Location>
<Size>
<CX>82</CX>
<CY>8</CY>
</Size>
<Hotspot>
<X>0</X>
<Y>0</Y>
</Hotspot>
<Duration>1000</Duration>
</Frames>
</Ui2DAnimation>
<Ui2DAnimation item="A_TubeGaugeFill">
<Cycle>true</Cycle>
<Frames>
<Texture>tkinvgauges.tga</Texture>
<Location>
<X>0</X>
<Y>10</Y>
</Location>
<Size>
<CX>100</CX>
<CY>8</CY>
</Size>
<Hotspot>
<X>0</X>
<Y>0</Y>
</Hotspot>
<Duration>1000</Duration>
</Frames>
</Ui2DAnimation>
<Ui2DAnimation item="A_TubeGaugeLinesFill">
<Cycle>true</Cycle>
<Frames>
<Texture>tkinvgauges.tga</Texture>
<Location>
<X>0</X>
<Y>20</Y>
</Location>
<Size>
<CX>100</CX>
<CY>8</CY>
</Size>
<Hotspot>
<X>0</X>
<Y>0</Y>
</Hotspot>
<Duration>1000</Duration>
</Frames>
</Ui2DAnimation>

Now, you have to declare a StaticAnimation item for each gauge that you wish to use these graphics for. I'll just include one as an example. Be sure to name each one differently (for instance, GW_PlayerHealthTube and GW_PlayerManaTube), and to include each one as a piece of your window at the bottom of the file (for instance, <Pieces>GW_PlayerHealthTube</Pieces>).

<StaticAnimation item="TK_PlayerHealthTube">
<ScreenID>TK_TargetTube</ScreenID>
<RelativePosition>true</RelativePosition>
<Location>
<X>-10</X>
<Y>0</Y>
</Location>
<Size>
<CX>149</CX>
<CY>34</CY>
</Size>
<Animation>TK_ExpTube_Overlay</Animation>
</StaticAnimation>

The <Animation> part of the StaticAnimation is where you tell it which gauge graphic you want to use, so if you want to use that 2nd gauge picture, make sure you refer to the TK_AltAdvTube_Overlay instead.

You'll have to adjust the location of your gauge graphics in order get them to display properly. I'd try setting the <x> and <y> of each of your static animations at the same <x> and <y> of the <gauge item> it corresponds to. You will have to move your static animations around, but this gives you a good starting place.

Lastly, you have to change the <GaugeDrawTemplate> in each of the Gauge items that now has a TK gauge graphic, using the fill animations you already created. The default gauge draw template will look like this:
<Background>A_GaugeBackground</Background>
<Fill>A_GaugeFill</Fill>
<Lines>A_GaugeLines</Lines>
<LinesFill>A_GaugeLinesFill</LinesFill>
<EndCapLeft>A_GaugeEndCapLeft</EndCapLeft>
<EndCapRight>A_GaugeEndCapRight</EndCapRight>

You'll need to change it to this:
<Background>A_TubeGaugeBackground</Background>
<Fill>A_TubeGaugeFill</Fill>
<!--<Lines>A_GaugeLines</Lines>-->
<LinesFill>A_TubeGaugeLinesFill</LinesFill>
<!--<EndCapLeft>A_GaugeEndCapLeft</EndCapLeft>-->
<!--<EndCapRight>A_GaugeEndCapRight</EndCapRight>-->

Note: if you want to use this gauge graphic in another window, you need to include in that window xml file the Ui2D animation referring to the picture of the gauge (though named something other than what you called it in your groupwindow.xml file) and a static animation item for each gauge for which you wish to display the graphic. You should not copy over the texture info item, or the Ui2D animation items referring to the fill pieces of the gauge.
