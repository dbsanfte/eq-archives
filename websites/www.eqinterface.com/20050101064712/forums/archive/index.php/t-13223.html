ANY time you add more graphics to the screen, you slow down zone/load time. Simple.

More to load = more time to do it.

The only way to speed up the load time of the interface is to make it as load-friendly as possible. Graphics no bigger than they need to be, in the smallest number of files. XML code clean, no extraneous items in the files.

Uhm, Izaru you say you know about custom UIs, but, ANY custom UI that doesn't replace ALL the files used by the default UI will generate those "warnings" you are talking about. They are how custom UIs are SUPPOSED to work. You CAN'T get rid of them. Unless you copy all the files from the default folder into a customer folder, then copy the custom UI files into that folder overwriting files as needed.

HOWEVER, as anyone who understands custom UIs knows, you should NOT do this since it precludes you from getting updates to non-custom windows and files. EQ only updates files in the default folder, so you would have to then copy those updated files into your custom folder EVERY time Sony made a change to ANY file, even if your custom UI doesn't specifically customize that file.

Which is why, as it says in those "warnings", if EQ doesn't find one of the required files in the custom folder, it just loads it from the default folder...as it is SUPPOSED to.
