Please, modders:

Stop including modified EQUI_Animations.xml files in your mods. If you are adding graphics information to a specific window, define the graphics at the top of that window's file. If you are changing definitions that are already in the EQUI_Animations.xml file, also put those in your modded window's file.

The Animations.xml file is loaded up before any window definition files. If there is duplicated but different info within the individual window files loaded later, it will overwrite info found in the Animation.xml file.


Not making changes to the EQUI_Animations.xml file will do a few things:

1.) Allow your mod to coexist with any other mod without the user having to make extensive changes to the code.
2.) Allow your mod to survive most changes made by Verant during this Beta test process.
3.) Make your download files smaller.


Thanks :)

What I had done to break everything: Deleted the <texture> definitions in each window file. The Ui2dAnimations were loading, but the files they referred to were never loaded...


1 problem with my initial post: You can't redefine textures or Ui2DAnimations in your specific window file that are already defined in the animations file. /loadskin fails with an error "duplicate items." This fact lead to a problem with Asmyr's buff box...

I thought everything was fixed. After getting the "duplicate item" error due to my attempt to redefine <Ui2dAnimation item="(Blue/Red)IconBackground">, I renamed them to (Blue/Red)LabelBackground. The mod loaded up without error, but the buffbox was displaying the wrong images. Instead of the (Blue/Red)LabelBackgrounds I defined in the EQUI_BuffWindow.xml file from the buffbttn.bmp texture file that Asmyr includes, it was loading up the default definitions ((Blue/Red)IconBackground) from the EQUI_Animations.xml file. I double checked that I had changed the buff buttons themselves to reflect the "Icon" to "Label" name change, and I had. Conclusion: Buff buttons are hard coded to pull the image defined in (Blue/Red)IconBackground for the buff buttons, no matter how those buttons are defined in the BuffWindow.xml file.

If you understood that, or care, I've sent a request to the developers to remove that particular hard-coding, as its really not necessary.


In any case, my original post still stands for the most part. Most new texture and animation definitions can and should be placed in the window files they are used it, rather than in the animations.xml file. I also hope Verant provides a little more modularity before freezing their code at the end of the beta test. Would be a lot more convenient for modders and end-users alike.
