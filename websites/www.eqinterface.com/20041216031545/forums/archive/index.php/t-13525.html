To allow people to have many yes/no dialogs up at once, new categories were added in the ini file for popup dialogs. The first time you brought up a dialog after the change, it was called PopDialog0. If you bring up more dialogs, they are called PopDialog1, etc.. Each one is given the Width and Height tags of the dialog in the UI you are using at the time. The next time you get a popup dialog, it uses the size from your ini file instead of the size of the window, despite the fact that the confirmation dialog is not resizable. This is obviously a bug.

Please fix popup confirmation dialogs to use the size of the window specified in the xml, not the ini, since it is not a resizable window.
