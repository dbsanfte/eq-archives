Mostlikely a 0x0 size castingbar window...

Many UIs chose to hide it like that...

What I would do is:
1) rename your EQUI_CastingWindow.xml temporarily.
2) get a default EQUI_CastingWindow.xml file from default.
3) Move the casting window to some corner of the screen.
4) rename the original file back.

That way you get rid of it from middle of screen :)

Just something to try atleast...

It is possible that one of your existing customized files has an "element" outside it's viewing area that is actually "showing up" (via right click menu, as you said).

Okay, that didn't make sense. Let me try again.

Say in your custom Player window, the author wanted to "hide" the mana bar. Two ways of doing this. One is to set the size to 0,0. The other way (and the one I'm suggesting might be the problem in your case) is to put it "off the screen", ie setting position to some ridiculous number (either positive or negative) that puts it "outside" the visible viewing area of the window. It's possible that you are "finding" that "invisible" mana bar. Did that make any more sense?

So, how to deal with the problem, options:

a) if what I describe turns out to be the problem, then you could go through, find the elements that are "off the screen" and pull them back onto screen, then size them down to 0,0;

or:

b) whether that's the problem or not, you always have the alternative of going into the Options window (Alt + O) and turning off the "Allow Context Menus". That turns off the right-click menu.
