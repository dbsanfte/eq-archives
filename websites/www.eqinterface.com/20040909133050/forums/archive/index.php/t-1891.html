Hi all,

I wanted to announce the release of the GUI XML Creator for use with maps. You can download it from my site at http://www.eqatlas.com/gui . I've left it on my site to spare this poor site's download section. :) This is a Java program that will create a EQUI_HelpWnd.xml file (Help Window mod) that includes those images. It reads the size of the images and generates the file automatically, so you don't need to know a thing about XML or creating these things for it to work. You just need to know how to follow my instructions. ;)

The purpose of this program is to allow people to have a Help Window with a tabbed pane that will show images (jpg only right now) in the game. Please download the images you want from my site or any other to use. I do NOT want to see people passing around zipped versions of my maps, so keep that in mind. I require that you still go to my page and download them yourself for your personal use.

I've also set up a message board section for talking about this on my site as well.

Download and enjoy!

Version 1.3 Available since 9/22. This fixes some sizing issues, and lets you put fixed-width tabs in a scrollable pane. It also has some beta-level code for Voodoo3 users to see it correctly.
Version 1.2 Available since 9/10. This lets you resize the whole window, and corrects the tiny map problems in version 1.0.

Originally posted by Muse
Aandiian: As far as I know there's no way to do that. The EQ client decides how to show those, we have no way to tell it that it has to size the tabs differently. I mention somewhere in the readme.txt that you probably shouldn't use over 20 maps cuz of this problem.
If you size the <TabBox> element so that it accomodates all your tabs at the size you want, you can then horizonally scroll to access those tabs.

What I mean is:
Set the <CX> of the size of <TabBox> to the following:

(number of tabs) * (desired size of each tab)

So for example, if you have 50 tabs and you want each tab to be 40 pixels, then you would set the size to 2000.

As long as <Style_HScroll> is set to true in the window, you'll now be able to horizonally scroll to each tab. Of course this will also scroll all the contents of the window, but you can now have lots of tabs, while keeping all of them at a width that is readable.

Edit: forgot to mention that the above only works if the tab box is not auto-stretched to the size of the window, that is, set <AutoStretch> to false.

Well, I have some bad news for Voodoo users. I've been doing some tests with Vandyl and his Voodoo board, trying to get various combos to work. It appears that Voodoo boards will never be able to show these things correctly, because of the way they load textures.

I plan on tweaking the GUI so that voodoo users will not have their maps cropped, but they will always have them distorted. What happens is that Voodoo cards are stuck with 256x256 texture spaces. Not a huge problem, because they can allocate more than one to fit a larger texture. The problem is that, in doing so, they stretch the texture to fit that multiple of 256x256 space (like 512x768 or 1028x256, whatever it is). This causes distortion that I can't fix through any manipulation short of creating maps that all fit some multiple of those dimensions (which I have no plans on doing).

Sorry for the bummer news. Hopefully this doesn't affect too many people.
