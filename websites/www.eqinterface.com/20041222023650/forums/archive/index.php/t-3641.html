After todays patch (10/31/02) I was crashing to desktop after character selection screen trying to load a character using a Vert 2.2 based UI. Looking around for a fix, I collected some tips from different threads and posts and now have my vert mod working. Hopefully this will help some of you.

1) Copied the file EQUI.xml from my default folder to my custom UI folder (where you are keeping your vert mod files).

2) Once EQUI.xml is copied over to your custom directory, delete line 20 and 21, should be these:

<Include>EQUI_LoadskinWnd.xml</Include>
<Include>EQUI_AlarmWnd.xml</Include>

These lines should be towards the bottom of the text after:

<Include>EQUI_PlayerNotesWindow.xml</Include>

3) I also followed the directions in this thread

EQUI_Animations.xml changes (http://www.eqgui.com/showthread.php?s=&threadid=3192)

OK... here is a last ditch fix for anyone who just wants to get in.

Go to the UI_<charname>.ini file for the character you want to sign on with.

There is a setting called 'UI(something)=<name of your custom ui>

Change this to 'UI(something)=default'

This will reset the ui for that character to the default ui.

This will of course not fix the problem with your custom UI, but will at least allow you to get into the game and zone reliably.

Also, once in a zone, you should be able to /load <customui> 1 while you are in the zone. Don't forget to /load default 1 prior to zoneing or you will get booted.

If you are using certain mods (mine, Kaori, Vert and others included) that modify EQUI.xml or EQUI_Animations.xml, they WILL NOT work if you copy the default one over. Same goes for various other files, depending on the mod. If you want to use a complete custom UI developed by somebody else, you have two choices when Sony patches: either use default till the developer comes out with a fix, or know enough about xml that you can go into your UIErrors file, figure out what's wrong with which file, and fix it. If you start just deleting/copying over whatever file somebody tells you to, you'll most likely only end up making things worse for yourself.

Oh, and re: the playerwindow thing, I had same problem, you just need to put the pethp gauge back in and set it to size 0,0.

Right now the UI I use has been updated, so you can download a new version...well the main one. I use a mixed version.

Let me explain. Perhaps 'download' was not the correct word.

I did a select all\copy in my custom UI folder. I pasted it into a file on my desktop. Then I deleted my custom UI folder and made a new, empty one.

So, I "select all\copy" from the "uifiles\default" file.

Pasted it into my brand new, empty, custom ui file.

Then did a "select all\copy" of my custom files that I had moved to the file on my desktop.

Opened up my new custom UI folder, which now has all the default files in it, and pasted the custom files back in....when it prompted me about overwriting I selected "yes to all".

What that did was put ALL the default files in my custom folder, then placed my ui files over them. Any NEW files made by the patch on the 31st were now in place in my custom folder, allowing the UI to function properly now.

But understand, when you have all the default files in your custom UI, it leaves your UI open for getting screwed with each patch that changes any of those default files. So this was only a temporary fix.

Now, my main UIs, Zeremus and Vert are both updated, so I just use the updated version now and have deleted that other folder.

It worked for many, many people who were in the chat room, freaking on the GMs, who could do nothing but sit in there with their tongues hanging out..

If I were you, and hadn't been able to get my UI working this long after the patch, I woul delete my UI completely, go to Vert, or Zeremus (ZW3), or whoever, and download their version anew, as if it were for the first time.

If you like some of the things, but not all, then do like me....use their UI as a base UI, and add changes to it as you like.
