I'm terribly sorry... but my original code had some problems in it. I've released a newer version of it. Nothing that would cause crashes or anything. But it was just annoying the crap out of me.

Here's what happened:

I had used a hacked up version of Remelio's code when I created my spellbook. I'm a graphics guy first and a coder second and was trying to take a bit of a shortcut. After releasing the mod I started to notice there was something wrong with the graphics of the mod in game. Other people might not have noticed it, but I certainly did. The game was squishing my graphics. I figured out why...

Aparently the game only really likes to work with images that are 256x256. It may be able to work with others, but I noticed that using any other size of image the game would do strange things to the graphics (leave spaces, or squish them). Because I had used Remelio's code, I was suffering from the squished graphics (gave my skull a pointy looking head).

I also noted that the size of the area for the spell names was too small... long spell names were getting cut off a little. So I re-worked that as well... Shouldn't be a problem anymore. I'm really sorry to those of you who already downloaded my mod. Please look for the updated version to show up in the downloads section. One other thing I changed is the edges of the paper in the corner... my first version didn't have a good enough 3d effect. So I fixed that. I also added a nice alpha blended drop shadow to the book itself, hopefully making it look like it stands out from the background a bit. Anyway... here's a screenshot of the updated version. You can compare it to the original screenshot to see what I mean.

You should avoid using textures that are over 256x256 because some video cards can't take textures that are larger. That's why the default UI textures are broken up into 256x256 and smaller TGAs. I myself use textures which have dimensions which are a pwoer of 2. 64x64, 128x128, 256x256, 64x128, etc.

I have never had and problems with EQ squishing or stretching any of my graphics as long as I correctly specify sizes in the XML files.
