Beta Version 1.0 - June 2004

Resolution for 1024 x 768 ONLY
No other resolutions are available...YET.

All graphics have been re-created by hand to simulate the EverQuest II UI as well as additional ideas implemented to enhance the interface. There are some portions that came from public screenshots and user submissions that were re-textured for clarity. You can visit the EverQuest II website at http://everquest2.station.sony.com/.



Installation:

* Do not have EverQuest loaded when installing
* Be sure you are running EverQuest in 1024 x 768 prior to installation
* Create a Folder named "eqii" in your EverQuest\uifiles directory
* Extract ALL files to the "eqii" folder
* Use the included UI-INI file for your UI-INI Character

PS - Please make sure you make a fresh install after each Beta!

PSS - If you move the BuffWindow in-game, you will notice the Bordering will come foward, blocking everything. This is NORMAL. Simply re-load the interface, or zone and everything will go back to normal. You can set the transparency of the window to see through the icons if you wish, but DO NOT hide the window from view (using the window selector). Doing so will also remove the bordering around the interface.

Well... I looked at it again...

The Chat window is the one XML window that is always behind EVERYTHING. So, if I place the borders in that window instead, everytime you try and move the chatwindow, that border will popup in front, which is totally bad.

I could code the graphic parts in each window at the bottom to be within the spellbar, hotbuttons, whatever, but then you are forced to have those windows always exactly there, and so many people have different setups, so I do not want to try that route.

So, in the released version, you will not be able to place a chat window over the bottom graphic unless you click it up everytime you zone. My suggestion is to place the chatwindow above the graphic so its not in the way of it as intended. All the miscellaneous windows will be housed within the black areas, so having the Chat window where it is supposed to be should not really be a big ordeal.

Now, as for those happy modders, I am sure you can mod the file to remove the bottom graphic and place the spellbar and hotkeys where-ever.

In the 1280x1024 version, having the chatwindows where they are intended to be should be even better since you have more screen space.

Well, I have some interesting news.

The UI has undergone many revisions with code and graphics as well as style. Thus being said, there will be 2 versions

Emulated EQII Interface (simulates the live version of EQII with borders)
Emulated EQII Interface - Transparent Version (user defined)

The Emulated EQII Interface (now on hold) will be undergoing some heavy code and graphics from scratch since the bordering, in my opinion, does not work well at all being coded into the buff window. Instead, I am going to make 2 static bars going across with the use of the /viewport commands. This type of interface has been put on hold until we can gather some new graphics to finalize the emulation. This UI will be for 1024 x 768 and 1280 x 1024 only.
