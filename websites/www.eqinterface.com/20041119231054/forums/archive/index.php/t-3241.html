10-23-2002, 02:31 AM
The reason your image isn't showing is because you haven't actually specified it as part of your window yet. You have to add in a Static Animation item as well, like so:

<StaticAnimation item="PW_Background">
<ScreenID>Inv_Background</ScreenID>
<RelativePosition>true</RelativePosition>
<Location>
<X>0</X>
<Y>0</Y>
</Location>
<Size>
<CX>256</CX>
<CY>256</CY>
</Size>
<Animation>p_window</Animation>
</StaticAnimation>

You can call the Static Animation item whatever you want, just make sure that it's the same name as you use in the <Pieces> part in the Screen item section. So in my example above, I'd put <Pieces>PW_Background</Pieces> in the Screen item section.

The <Animation> part is where you put the name of the Ui2D Animation that refers to the image you want to use. Make sure that the size is the same in both the Static Animation and the Ui2D Animation or your image will get stretched/distorted.

Hope this makes sense. :)

Sounds like an issue with the transparency option of your image file. I recommend reading through the various posts regarding this in the Help - Graphics forum. I was having a similar problem the other day and the fix was to get the transparency setup properly. (For example, simply going into Paint Shop Pro 7 and making a new image with a transparent background and then saving it as a TGA file will NOT enable a transparent background within EQ)

Hope this helps...
