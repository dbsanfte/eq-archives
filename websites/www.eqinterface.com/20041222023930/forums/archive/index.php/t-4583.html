Last night I was monitoring the serverwide.eqgui:eqgui channel and got a cry for help. The person was having problems with the new bug window. Since they were running a small network, they were using 1 machine to swap the files on the other.

However, the bugwindow.xml (12/4/02 version) was still causing an error and failing to load skins.
uierrors.txt - ERROR - CLASS::ITEM A_CHECKBOX_<I forget- there was more :)>

essentially, the EQ executable had started (using the old bug window), the person then swapped in the new bugwindow.xml. However, because the EQ executable did not START with the class definition it was UNABLE to make the new assisgnment at the /loadskin event. Therefore, even though the person properly replaced the XML file on the fly, they still needed to actually leave and restart Everquest for the CLASS::ITEM to be initialized. Once the person quit and restarted, everything loaded just fine.

Since I myself don't have 2 computers networked, could someone else here verify/confirm this behavior.
