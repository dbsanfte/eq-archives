Go to your custom UI folder, find the EQUI.xml file. Make sure that these two lines are included:

<Include>EQUI_SkillsWindow.xml</Include>
<Include>EQUI_SkillsSelectWindow.xml</Include>

As for the messages in your UIError.txt log file, don't worry, those aren't errors, they are normal "warning" messages saying that EQ is doing exactly what it should be doing, which is pulling files from the default UI folder if it doesn't find them in your custom folder. We want to see those messages in the error log. :)

If you can't open it in default, then there is likely a problem with your ini files.

Remove your UI_<charactername>_<servername>.ini and your defaults.ini files. If it works after that, then check the entries in them for the location/size of the Skill window to be sure that it isn't off the screen, or shrunk to nothing.

Oh, and don't copy/paste xml code form your browser view...it adds extra characters. Use notepad, or another text editor, instead.
