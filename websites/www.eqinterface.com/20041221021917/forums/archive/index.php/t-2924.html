10-31-2002, 02:56 PM
There is a limited number of default windows you can later for custom windows. The hilt with the gem, skull, and about the first inch is the spell gem windows. The image map for it is already 256x256, in order to use one of those other spell_gem holder designs, or in order to line the gems up along the blade of the sword, i'd have to increase the dimension of the spell gem window to 256x512, 512x256, or 512x512. That would make it entirely to big and bulky. It would also elliminate the option to have the buff icons overlay on the blade itself.

I'm trying to keep it so there is a single unifying image merging it all together. There are other sk/necro themes out there that the only thing that ties the elements together is the fact that there are skulls somewhere in the images, or the windows are all purple or such things. Using that circular pet window you linked wouldn't really go with what i'm trying to do.

As for the #% being at the end of some gauges and in the middle for others. It's in the middle for the groupmembers and exp bars for space reasons. It looked cleaner and more symetrical to have the "0" sitting in the middle of the groupmembers while empty, and it's like that for the exp bars simply cause there's so little space where they are located. I put the #% at the beginning for HP and MANA, and above the Target because that's just what I wanted to do. hehe For gauges that empty fast, or is constantly changing I just prefer seeing the whole bar instead of having 1/4 of it covered up by numbers.

As to using extra chat windows to add a transluscent glow to the sword, I don't think that would work. It's a very creative idea, though. Once added in, the pet window (unless I make some big changes) would always be up whether there is a pet summoned or not. That's also one of the things i'm pondering 'cause, I too, would prefer to toggle it off when I don't need it. I'm still working on it. I had not clue the pet window would be giving me the most trouble when I started this. Probably because when I sketched out the image layouts in the beginning, I didn't plan on adding the pet window. So it didn't get it's own special place at the time of conception. It's kinda being added as an afterthought now, but i'd still like to do it well.

I'm planning to make a final run through of all the images and tighten them up. What i'm using now is just for draft purposes, the final ones will be cleaner. I looked for weeks before starting, for artist renderings of the SK sword, that were stylized or altered for artist effect. I couldn't find one that 100% true to the actual sword. A SS was my only choice. I took the one I used myself, so it's the closest I think is possible to what I wanted......short of getting my hands on the 3d database object from SOE hehe

Thank you for the reply, i'll take it under consideration. Keep it coming.

I've seen a lot of UI's out there that look absolutely fantastic. here's some things to think about, imo, when making on.

Place all common use buttons as close to the mouse as possible.

This does two things; one, it keeps all the buttons together so you don't have to move the mouse to opposite sides of the screen to use spells then back to switch weapons, or use a skill like bash; two, I think it's a psychological thing, but I seem more coordinated when said buttons are close to the mouse (the bottom right corner, I'm right handed w/the mouse)

I know most Ui windows can be relocated as most see fit, but designing the ui with these things in mind seems like the ui would naturally be more beneficial.

On the other hand, I've also downloaded some ui's that looked great, but functionally they were aggrovating, like strings of buttons spanning the lenght of the screen.

Anyway, hope this helps with development ideas.
