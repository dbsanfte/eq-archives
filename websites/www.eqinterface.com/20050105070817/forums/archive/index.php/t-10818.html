to answer dolbys question.. there is a flavor of Linux called Lunix that I helped develop long ago.. it had a special version of DirectX 7a built into it... but with microsoft not wanting to give away their DirectX source codes at a reasonable price. we haven't been able to update the OS in a long time.. EQ is currently unplayable on it even with winex.

<video card optimization>

you optomize for certin video cards by figuring out sizes of the graphics, in this case Nvidia works best on a ratio of square even numbers (256x256) where as what I can tell about ATI is you can use non-square even numbers and have it run faster (256x128).

also you can finetune the XML code to run faster on what card as apposed to another.. what I've found in Nvidias case is that having the heavier code ( stuff that renders large on the screen ) as the first bit of code in the window makes that paticular window render faster ( by about 200ms ), ATI appears to work the same way.. 3dlabs wildcat 3 wants the lighter code first.

the optimization is just minor tweaks in reality... I guess I could just be too picky when it comes to making the UI error free and light as a feather. :D
