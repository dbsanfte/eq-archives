Actually you can add more screens - there just no way get them open. I seem to recall someones interview with Verant and this was a topic.

Also, I've successfully transformed the inventory window into a parent window with the real inventory window as subwindow floating witihn it. Heres a run-down of the behavior exhibited.

The child window was only accessible as a control when the parent window was open and behind the child. This includes any mouse interaction such as sizing, moving, etc...

The parent window didnt need to be fully behind the child- only the area being used on the child had to be in contact with the parent.

The child window could be closed, but there was no way to re-open the child window short of re-loading the mod. Recommend CLOSE being set to False and MINIMIZE set to True.

Parent window could have several children windows.

Controls available to the parent window were also available to the child window. For example, auto equip worked on the child inventory window. Pressing DONE on the child inventory window closed the parent inventory window (darn hard coding).

I placed the Auto-Equip in the parent window by it self and it worked. However, this cause the Auto-equip on the child to stop working. No UI errors either. LOL.

Parent window remained affected by all open and close events (selector window, CTRL-I, I from keyboard)

Minimizing Parent - removed child windows from screen.
