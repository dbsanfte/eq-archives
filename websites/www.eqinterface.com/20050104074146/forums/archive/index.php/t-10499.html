<AlignRight>true</AlignRight>

Here's an example in context...

<Label item="Player_AC">
<TextColor>
<B>255</B>
<G>255</G>
<R>255</R>
</TextColor>
<Font>3</Font>
<RelativePosition>true</RelativePosition>
<Location>
<X>13</X>
<Y>46</Y>
</Location>
<Size>
<CX>22</CX>
<CY>16</CY>
</Size>
<Text>AC</Text>
<NoWrap>true</NoWrap>
<AlignCenter>false</AlignCenter>
<AlignRight>false</AlignRight>
<AlignLeft>true</AlignLeft>
</Label>

<Label item="Player_ACNum">
<TextColor>
<B>255</B>
<G>255</G>
<R>255</R>
</TextColor>
<EQType>22</EQType>
<Font>3</Font>
<RelativePosition>true</RelativePosition>
<Location>
<X>38</X>
<Y>46</Y>
</Location>
<Size>
<CX>35</CX>
<CY>16</CY>
</Size>
<Text>1888</Text>
<NoWrap>true</NoWrap>
<AlignCenter>false</AlignCenter>
<AlignRight>false</AlignRight>
<AlignLeft>true</AlignLeft>
</Label>

These are aligned left... but you get the idea. :D

It would do the right thing because <AlignLeft> is not a valid tag..

EQ just ignores it. I think the devs put it in there so they remembered what the text would do.. Kind of like a comment.

These are valid Label alignment tags taken from the AboutSIDL.doc..

NoWrap - Don't allow this label's text to wrap - default: false
AlignCenter - Center the text. By default, the text is left-justified - default: false
AlignRight - Right-justify the text. If AlignCenter is true, this value is ignored. default: false

Xymarra
10-07-2003, 04:50 PM
Ok, so it wouldn't do the right thing then.

If you said Right-false, Center-false, you would get the desired effect of Left-alignment, since that is the default value.

If you said Left-false, Right-false (or Left-false, Center-false), you would still get Left-alignment, since the false tag on Left would be ignored and the default for Center is false.

The safest and easiest thing is still to have exactly one alignment tag, the one you want to be true.
