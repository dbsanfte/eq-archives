/loadskin currently has the following functionality

/loadskin
lists all sub-directories in the ../EverQuest/uifiles directory.

/loadskin <skin directory> or
/loadskin <skin directory> 0
sets the active UI definition files directory to ../EverQuest/uifiles/<skindirectory> and resets the UI_<playername>_<servernumber>.ini file to the game default values.

/loadskin <skin directory> 1
sets the active UI definition files directory to ../EverQuest/uifiles/<skindirectory> and keeps the existing UI_<playername>_<servernumber>.ini file.

The problem is the default value of 0 for the second parameter. If you forget to type 1 a significant amount of work in placing and adjusting various windows is flushed into the electronic aether never to return.

Please change the default for the second parameter to 1 and not 0. This way you have to intentionally type a parameter to flush the .ini file.
