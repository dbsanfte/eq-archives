************Fozz_Infinity_1.0************

Instructions:
1)Create a new folder in your EQ directory... call it whatever you want (ex. Fozz)
2)Unzip all files in the Fozz_Infinity download
3)copy the files only (no folders) and paste them into the new folder you created
4)Enter the game and type /loadskin (your custome folder name)
5)arrange to your liking. This UI is all in pieces and almost all are easily moved

To Install Mods:
Included in the download are several mods to the UI. These are in the mods folder, and each mod is in it's own folder who's name describes what the mod is.
1)Open the mod folder that you want to use (ex. vertical buffs folder)
2)Copy the files
3)Paste the files into the custom UI folder you created for this UI.
4)Your computer will ask you if you want to overwrite. Click "yes to all"

12-31-2003, 04:02 PM
Which player window are you using? That one I included as default, or one from the mods folder? If you are using the default one... here's how to get the attack border back.

Open up your EQUI_playerwindow.xml with notepad, and find this section right near the beginning of the code:

<Ui2DAnimation item = "A_AttackIndicator">
<Cycle>false</Cycle>
<Frames>
<Texture>TargetBox.tga</Texture>
<Location>
<X>6</X>
<Y>2</Y>
</Location>
<Size>
<CX>0</CX>
<CY>0</CY>
</Size>
</Frames>
</Ui2DAnimation>

Change the <size> section so that it looks like this:

<Size>
<CX>116</CX>
<CY>28</CY>
</Size>

Save the file and load EQ.

The blinky attack border will only work in the player window... in order to have a blinking border on the target window you'd have to switch all the code for the two, while maintaining the needed information for each. A bit more complex, but I could do it for you if that's what you really want. I would just need to know which playerwindow and target window you are using.

I just uploaded version 1.8.

This version includes a fix for the container window (the augment window should work as advertised now). I also added a fixed-up target of target window... it now matches the rest of the visible UI better.

DarkFather,
The target of target window may not be showing when you bring it up because it is off screen. If you are using 1024x768, that is probably the issue. If you try it with version 1.8 and you still can't see the target's target window when you bring it up, log out and look for a file in your everquest folder called UI_(your charactername)_(your server).ini

Open up that file, scroll all the way to the bottom, and you should see the entries for TargetOfTargetWindow. Set the Xposition and Yposition numbers to like 500, 500. This should hopefully do the trick.

As for the melee target window... I tested it again, and it works fine. It must be a glitch with your computer. Try it again and see if you crash. Make sure you don't reload the UI too many times... that will crash anyone's computer.
