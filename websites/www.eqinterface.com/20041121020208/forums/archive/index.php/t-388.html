Pini Uldar
07-31-2002, 05:25 PM
Non-file notes:
It seems like the tga files are cached now. They don't seem to be changing on zoning. This means that window_pieces01 that gets loaded on character select stays loaded even if you have a custom version of it in your skin.

Also new, if a file isn't found in your skin directory, it uses the default version. So if you haven't changed EQUI_BankWnd.xml for example, you can just leave it out of your skin directory and it will fall back on the default one.

File Change notes:

[window_pieces02]
Edges of the Spell gem background in the lower right corner were cleaned up.

[gemicons02]
The files aren't binary match, but I couldn't find a visual difference.

[EQUI_TrainWindow]
Minimal width readjustment.

[EQUI_TextEntryWnd]
Fixed up. I believe this wasn't used at all before and was a cut and paste job from QuantityWindow. Now it is used for renaming Chat windows (right click on title bar - Rename Window).

[EQUI_SkillsWindow]
Minor anchor offset changes make the window layout change a bit, but nothing major.

[EQUI_BazaarSearchWnd]
Had the listbox changed from absolute sized to being autostretched. Note that the listbox contents that are hard coded request A_DividerVertical. If EQUI_Animations doesn't contain this, then you will crash to desktop when doing /bazaar.

[EQUI_Animations]
Had A_DividerVertical added. This is used by the new resizeable multi-column listbox in the Bazaar Search Window. If this is not added, you will crash on /bazaar.

[EQUI]
Had EQUI_TextEntryWnd.xml added as an Include.

[SIDL]
[EQUI_SocialEditWnd]
[EQUI_SelectorWnd]
[EQUI_OptionsWindow]
[EQUI_BazaarWnd]
[EQUI_ActionsWindow]
I couldn't find any actual changes to these files except for whitespace and carriage-return changes. diff -b -B said the files matched.

- Pini
