Problem has to do with it referencing a graphic file that does not exsist. If I recall right it's trying to look for 07 of the window pieces.

This is not a bug if you're using the latest full version of the UI, but if you're running any version pre2.6, it will be.

Problem is caused from the renaming of the files when SOE added a window_pieces06, etc files. This broke many UIs that relied on the fact that 05 was the last piece file. So, to assist in making UIs more patch proof, they started using their own naming conventions. 2.6, if I recall right, is when this change was put into effect.

Try installing 2.6e and patching that. It should work.
