Here Is What You Do To Update or Convert To DDS

1- Get the UI you want to update or convert.
2- Using PSP, PS, or one of the converters listed in the Stickyed post by Kudane, Convert all TGA to DDS.
3- Change all references to the TGA versions to reflect the new DDS versions.
3B- If any of you windows contain refrence to a default file (default UI componet) then save a converted(from TGA to DDS) version with the updated/converted UI.
4- Delete(or store elsewhere) the TGA versions.
5- Your UI is now updated/converted!!!

Here's an example*

Files
WindowPieces_1.tga
EQAnimation.xml

Change WindowPieces_1.tga to a DDS. It will look like this.
Windowpieces_1.dds
Delete( or save elsewhere) the TGA version.

Now go into EQ animations and

Change
<TextureInfo item = "window_pieces01.tga">
<Size>
<CX>256</CX>
<CY>256</CY>
</Size>
</TextureInfo>
To
<TextureInfo item = "window_pieces01.dds">
<Size>
<CX>256</CX>
<CY>256</CY>
</Size>
</TextureInfo>

and now your done.

*- For simplicity sake, we are imagining that there is only one referance to the file we are using.

RULES TO REMEBER!!!

1.)No UI can have both DDS and TGA in the same UI. You must have all TGA or all DDS.
2.)All refrences to the TGA version in XMLs MUST be changed to reflect the new DDS.
3.)Any default images, files that come from default folder, must be in the custom folder, as DDS, if you want to use them as DDS in the UI.
4.)Be sure to save you DDS on the highest resolution setting to prevent picture disfigureing

I will update this post as information becomes avalible.

When I try saving a few of my TGA files, I get this error in the Nvidia option screen:

"DXT is supported for power 2 only."

Any ideas?

Actually what this means is that the width and height must be to a power of 2. (e.g. 1x1, 16x4, 64x128, 256x256, ...) up to 1024x1024. So the only acceptable numbers are: 2, 4, 8, 16, 32, 64, 128, 256, 512, 1024. Images larger than 1024x1024 aren't supported. This is easily fixed if you are using Photoshop. Just enlarge, (or shrink), your canvas size and anchor everything to the upper left hand corner. Then go in if you have an alpha channel and fill in the new part of the canvas. Save them as DXT 1 ARGB (1 bit alpha) if you have an alpha channel and no MIP mapping. (Save them as DXT 1 RGB, if you aren't using an alpha channel).

Do not save them as an 8:8:8:8 scheme. This is saving them as 8 bits per channel with an 8 bit alpha, (which is not necessary, unless you have fades). This is where the increase in file size is coming from as these are basically uncompressed files.
