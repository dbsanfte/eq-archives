GENERAL INFORMATION

What is EverMon ?

EverMon is a small utility that monitors the servers for the EverQuest multi-user game developed by Verant Technologies.  EverMon has several features, which include the ability to automatically launch EverQuest, when your desired servers are up, Play sound files when your servers are up, monitor the number of players on each of the world servers, and chat server, provide you with up-to-the-minute broadcast messages about game status, and much more.
Who Develops EverMon ?
EverMon is directly sponsored and supported by HydraSoft, a division of Hydra Computers, and Programmed by Bryon Quackenbush (a.k.a. Zackron).  Neither Verant, 989Studios, or Sony Interactive have any role in the development of EverMon, nor is it officially supported by them.  However, Verant did develop the Status Server that EverMon uses in order to get the majority of it's game server status and information.

INSTALLATION INFORMATION

Full Install vs. Update Install

There are two different distribution files that are released for EverMon.  The First file is a rather large file, upwards around 2 Megabytes in size, this is the Full Installation file, and meant for first time EverMon users, who do not have the Microsoft Visual Basic 6.0 libraries installed on their computer.   The second version is rather small, normally around 100-150 Kilobytes in size.  It is the update version, and meant to users that already have EverMon installed.

You should download the Full Install version when :

  • You have never Installed EverMon before, and you do not know if you have all the Microsoft Visual Basic 6.0 Run Time Libraries insalled.
You should download the Update Install version when :
  • You already have a previous version of EverMon that has already been installed, and had at one time properly worked.
  • You know for sure you already have all Microsoft Visual Basic 6.0 RTL's installed on your computer. (i.e. you have Visual Basic 6.0 installed).
If you try to use the Update Install without using the Full Install first, and the Update version will not run for you, then it is advised that you download and install the Full Version, even if you think you might have all the needed libraries.
Full Install Instructions
If you are downloaded the full install version, there are very few instructions that you need to know about, since almost everything is completely automated.  But here is some information for you to go by.
  1. The Install file is shipped in an .EXE file format, which automaticlaly begins the installation process when you run it.
  2. The installer will check the version of some of your system files.  Specifically OLEAUT32.DLL, OLEPRO32.DLL, and MSVBVM60.DLL.  If any of these files are out of date, the installer will ask you if you would like them to be updated, and it will then restart your computer.  You will then need  to re-run the install to continue.
  3. The installer will default to using C:\Program Files\EverMon\ as the default directory, although you can select any directory you please to install it to.
If you are having problems with the installation, or the Installer Keeps asking you if you want your files updated, then make the following :
  1. Make sure no other programs are running during the install, this includes programs in your System Tray, and programs you may have set up to auto-start in your Start Menu.  If any other program is using any of the .DLL files the installer needs to update, the update will fail.
  2. Make sure that you are running in the standard Microsoft Win 95, or Win 98 shell... Some programs, such as Neo Planet which may modify the shell environment, and system files, may cause the installer to function incorrectly.
  3. If you still have problems, the files may already be updated, try downloading, and running the Update Version.
  4. If you still have problems, send me e-mail at BryonQ@hydracomp.com, and I will try to help you further.
Update Install Instructions
The Update versions are set up slightly differently than the Full Install files.  Update Installs are Self Extracting Zip files.. which means that they come with as an .EXE file, but when you run the file, it will create the EverMon.EXE file in what ever directory you run it in.  the EverMon.exe file which it creates should be copied to replace the olf EverMon.exe file, where ever you installed EverMon at.

If you used the default installation, it would be in the C:\Program Files\EverMon\ dirctory.  You can either use a DOS window, or Windows Explorer to move the file to the correct location.

Options Menu
The options menu is where you will configure EverMon to function the way you want it to.  There are 3 options in this menu :
  • General - General Path and Nickname Options
  • Monitor - Selcts servers to montior
  • Event - Selects servers to prevorm events for
GENERAL OPTIONS

This menu provides a way to select paths and other information :
  • EverQuest.exe Path - In order to use the Auto-Launch features of EverMon, you must list your full drive, path, and filename to everquest.exe, an example would be : C:\Program Files\EverQuest\everquest.exe
  • Path and Filename for .WAV - If you would like EverMon to play a wave file when all your servers are up, you must list the full drive, path, and filename here.
  • User Nickname - This is only needed if you use the Interactive Menu options, and allows a name to be associated with you when communicating with the Stat Server in Sending But Reports, Comments, etc.
MONITOR OPTIONS

This menu allows you to select which servers you would like to monitor, or show information for in the main window.  Only servers that you select here will be shown.  Also, In order for you to use a server in the Events Options (explained next) it must be selected in the Monitor Options.
  • SELECT THE SERVERS YOU WITH TO MONITOR - Servers selected here are monitored in the main window.
  • DO NOT CONNECT TO STAT SERVER - Check this if you do not wish to access the EverMon Stat Server - This will prevent you from getting broadcast updates, auto-downloads, and other non-Verant provided information.
  • DO NOT DISPLAY BROADCAST MESSAGES - Check this if you don't want a window to pop up with game-related broadcast news.  If you have the 'Do Not Connect to Stat Server' checked, then this option is auto-assumed to be checked as well.
EVENT OPTIONS

The Events Menu allows you to select which servers will cause a server connect event :
  • SELECT REQUIRED SERVERS TO BE UP TO PREFORM EVENTS - All the servers that you check here must be online before EverQuest is launched, or your selected .WAV file is played.  NOTE : The servers must be selected to be monitored, in order to be able to be selected as event servers.
  • WHEN SERVERS ARE UP LAUNCH EVERQUEST - If you would like EverQuest to be launched automatically when your selected servers are up, select this box.
  • ASK ME BEFORE LAUNCHING - Will give you a message box asking you if you would like to launch when your servers are up.
  • CLOSE EVERMON AFTER LAUNCH - Will automatically close EverMon when it launches EverQuest.
  • WHEN SERVERS ARE UP, PLAY A SOUND FILE - This will play the file you select in the General Options window
  • PLAY SOUND ONLY ONCE - If this is NOT checked, the sound file will be played, each time the server check is done, and the servers are all up.
Interactive Menu
This menu ONLY appears when you are connected to the Stat Server... If you are in Direct Mode, you will not see this menu.

When you do see it, you will have the following options :

SUBMIT BUG REPORT - Allows you to send a message in regards to an EverMon bug you have found.

SEND COMMENT - Allows you to send a general comment about EverMon to the EverMon development team.
 

VIEW BROADCAST MESSAGES - This will re-display the broadcast message window, and allow you to view any previous broadcasts that you may have missed, or that were sent before you connected to the EverMon Stat Server.

Web Sites Menu
The Web Sites menu contains links to some of the most popular EverQuest sites, including the official EverMon site, and the official EverQuest site.  Select any of these links, and your default browser should automatically jump to the site you selected.
Help Menu
This is where you can select to get information About EverMon, or where you can select to view This Online Help file, which will also automatically launch your default browser.

Once EverMon starts, on the main form, in the lower right corner, you will see the current mode that EverMon is running in.  This can be one of 3 possible modes :

Appendix A - Why can't you PING or TRACERT EverQuest Servers?

(updated 02/25/1999)

Perhaps one of the questions I get asked most often is people asking for EverQuest server addresses that they can use the PING or TRACERT command or, or people asking me why these commands fail when they try to use them...

If you go to your MS-DOS prompt, and type PING CHAT.EVERQUEST.COM  you will see what I'm talking about, in that your requests will all time out, or fail to reach their destination.  If you type TRACERT CHAT.EVERQUEST.COM, you will notice that a trace route starts, but seems to die just short of getting to Sony Interactive's network (Where EverQuest.com, and the rest of the everquest servers live).

The reason both of these fail is because there is a large firewall set up on the gateway computer which leads into the Sony Interactive network.  This firewall blocks all incoming ICMP packets.  ICMP packets, otherwise known as Internet Control Message Protocol, is the means by which PING and TRACERT commads are able to return information to you.   Since the packets are blocked, the computer your trying to send the packet to never gets it, and thus never replies.

So why would someone wish to set up a large firewall which prevents people from being able to PING them?  There are many answers for this.  One factor is that of network security.  Another is that of server stability.  Game servers when not being flooded with PING's can focus more on serving game players.  In the end, it all works out for the best for the players.

So how can you get PING and TRACERT to work with EverQuest servers?  Unfortunately, you can not.  Although a return trip time is being worked into one of the future versions of EverMon, which will act like a PING, there is no way to get TRACERT to work on any of the game servers.  The closest you will be able to get is the computer right before the Sony Interactive gateway computer that blocks the ICMP packets

Appendix B - What is the difference between real PING and EverMon PING times?
(updated 03/10/1999)

With the new ability of EverMon to display Chat and Game server ping times in Direct Mode, I've started getting several questions regarding the comparison to real pings, that you would get from DOS, and the numbers that EverMon reports.

First of all, it's important to understand that the ping times that EverMon reports is not in any way the same as the time that would be reported if you used the DOS PING instead.  The reason they are different is in part because true PING's and TRACERT's can not be done on EverQuest servers... See Tech Report #1 : Why can't I PING or TRACERT EverQuest Servers ? for a good explanation as to why this is not possible.

Next, DOS PING times are reported in milliseconds... which means that a ping time of 1000 would be the same as a 1 second time.  The times reported in EverMon are not millisecond.... instead, a 100 ping in EverMon is the same as a 1 second round trip time.

Your next guess would be to multiply the EverMon ping by 10, to get the normal PING time, so if you see a 60 in evermon, you would think this to be a 600ms real PING, right?  Nope!  this is not accurate either...

Because a DOS PING is done at an operating system level, the times are going to be more accurate.. the PING times in EverMon are done at an application level.  There are events that occur in the application before the ping time is calculated, which can cause the time to be slightly longer than if it were done at the operating system level.  Since we are talking about very short periods of time, it does not take much to make that value higher.

So, the final question is what good is the ping time in EverMon, if it is not the same ping time that the DOS PING would return?  The main, and only purpose to the EverMon ping times is to allow users to get an idea as to what servers are running faster, and to give them a general idea of how long it is taking to make a round trip from their area...  Generally, ping times of 40-50 are pretty good, where as ping times greater than 100-110 may not be as good.  users must compare their ping times to those of their friends, and check to see at what ping times players are getting good preformance at, and what ping times players are finding it hard to play at.

I'll start making this ping time ranges avalible here as soon as I'm able to start getting feedback from other players as far as what ping times others are getting, and how well they can play EverQuest.
