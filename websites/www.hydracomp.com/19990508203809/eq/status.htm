Revision Histroy - Listing the new features added for each version of EverMon

  • Version 2.1 Build 32 (04/22/1999)
  • EM Stat Server protocol completely re-written to provide a more solid structure
  • Added better Error Trapping and Debugging Support
  • Added a new splash screen with thanks from ColdDarkNord
  • Added new Tray Icons with thinks from Mike!
  • Added Server Up/Down timers
  • Added support for ICQ numbers which can be used in Bug Reports/Comments
  • Bug Reports and Comments now use the Interactive Forum interface instead of message boxes
  • Fixed many bugs in stability and general function
  • Version 2.0 Build 28 (04/08/1999)
  • Should finally fix the Error #6 - Overflow errors
  • Added a 'Total Players' for all servers counter
  • Added a Nect Check countdown timer... back by popular demand :)
  • Version 2.0 Build 27 (04/06/1999)
  • Fixes an Event Options bug which prevents event server selections from improperly being saved
  • Should fixe the StatServer Overflow error
  • Addes a new Error Monitoring feature for better error tracking.
  • Version 2.0 Build 26 (04/03/1999)
  • Fixed some crashing bugs
  • Added "Launch EverQuest" to the File menu
  • When you have EverMon set to ask you before launching EverQuest, it will only ask once, instead of sending repeading questions over and over
  • Version 2.0 Build 25 (04/03/1999)
  • Major rewrite of the entire EverMon interface
  • Integration of the new Verant hosted Stat Server
  • Server status is now recieved directly from Veran't network system
  • Number of players in each world, and in chat server have been added
  • Auto-Server updates have been added, so when new worlds are added to the game, you will not have to wait for new EverMon versions to be able to support them.
  • An Equalizer-like interface for displaying zone status information
  • A new built new forum message window for recieving and reviewing broadcast messages.
  • Automatically get past broadcasts that you may have missed.
  • A new scrolling text window for up-to-the-minute game and news information
  • Direct/Stat Server/Update modes have been removed.
  • Cycle and Server delay times have been removed, EverMon will now report status on all servers at the same time, including zones regardleass  if the EverMon Stat Server is up or not.
  • Option menus have been updated to reflect the new features that have been added, and the removal of features that are no longer used.
  • Options have been added to allow the user to select not to access the EverMon Stat Server, as well as not to receive broadcast messages in the Options | Monitor menu
  • Tons of minor errors and stability issues have been fixed.
  • Version 1.7 Build 16 (03/25/1999)
  • Added support for 2 new EverQuest servers (Bristlebane and Innoruuk)
  • Hopefully fixed the EverQuest file selection bug
  • Added an Auto-Sleep mode to client - EverMon will stop monitoring servers when it detects EverQuest running in the foreground, and start monitoring once EverQuest is exited.
  • Added Beta client support to EverMnn
  • Added Enhanced Auto-Update Support
  • Added Multi Stat Server Support, and Load Balancing Support
  • Added "Maximize EverMon" menu option to Tray Icon menu
  • Stat Server will retry up to 3 times on failures
  • Added an 'Interactive' menu with bug and comment reporting
  • Added User Nickname support which is used with the Interactive menu
  • Unselecting servers in the Monitor Options menu will also automatically unselect them from the Event Options menu.
  • Servers will not be selectable in the Event Options menu if they are not already selected in the Monitor Options menu.
  • Version 1.6 Build 13 (03/09/1999)
  • Fixed Server time save bug - Server time will no longer default to 20 seconds at startup
  • Fixed the wave file selection button - Using the button should now properly place the selected wave file into the wave file path box.
  • Fixes the broken auto-update probem - new updates after this version should now properly inform the user tha such an update can automatically be downloaded.
  • Added Chat and Game Server PING Times - This will only work when using Direct mode, and you must go into the Monitor Options window, and select to use Direct Mode Pinging at the bottom.  A 100 ping time is about 1 second round trip.
  • Added new tray icons - The tray icon will now be red if your selected servers are not up, and green if they are up.
  • Added a new File Menu option to switch to Direct Mode from Stat Server Mode.  This should make jumping between the two modes easier.
  • Version 1.5 Build 12 (03/05/1999)
  • Added support for 7 new game servers, and 77 new zone servers
  • Cycle and Server times now properly save
  • Changed min. server-to-server time from 5 to 3 seconds
  • Changed min. cycle time from 30 to 20 seconds
  • only servers which are selected to be monitored will be shown on the main form.
  • Stat server has been updated to only allow client versions 1.4 Build 9 and higher to access it.
  • Version 1.4 Build 10 (02/26/1999)
  • Client / Stat Server now communicate using UDP instead of TCP, which should allow the Stat Server to handle more clients
  • Stat Server now has dynamic Zone Port search ability - this should make zone status reporting much more accurate
  • Cycle and Server times in the Monitor Options menu now properly work
  • Direct/Stat Server mode selection in the Monitor Options menu works
  • File search buttons in the General Options menu are working
  • A new 'Web Sites' menu has been added to the main window, which includes several popular EverQuest news and information sites.
  • Clients now support Stat Server linking, this allows multiple Stat Servers to exist, and allows clients to access the best avalible Stat Server
  • Client should now properly detect that it lost communication with the Stat Server, and correctly jump into Direct monitor mode.
  • Client now has the ability to automatically download new versions using the user's default web-browser.  This feature is only avalible when connected to the Stat Server.
  • Version 1.3 (02/19/1999)
  • Zone monitoring is now in for all zones.  However, since there are over 66 zone servers, zone monitoring is only avalible when working in stat server mode
  • Configuration is now saved in an INI file.  All configuraiton will need to be re-done
  • Configuration windows have changed.  The one main options window has been split into 3 different windows
  • a RESET button has been added to each of the configuration windows
  • An option to Launch EverQuest has been added to the main file menu
  • An option to reconnect to the Stat Server has been added to the file menu, however, this will only work if the stat server with online when you first started EverMon
  • Multiple servers can not be selected as being required before preforming a launch or a sound event
  • A new configuration option to allow the program to Ask before it launches EverQuest has been added.
  • If defined, the EverQuest.exe, and the desired sound file are both checked for existance before they are saved to the configuration file.
  • A new error tracking system as been added.  Errors that occure should now be logged in a text file for easy reporting
  • Version 1.2.1 (02/17/1999)
  • Fixed Stat Server access problems
  • Splash screen can now be skipped by clicking somewhere on splash screen
  • EverQuest can now be launched from right clicking on the EverMon icon in the Icon Tray
  • EverQuest will only be launched once, even if "Close on Launch" box is not checked.
  • Version 1.2 (02/15/1999)
  • Added support for minimizing the Monitoring window, and Icon Tray support
  • Added Status Server support which allows for much faster server monitoring, and updates all servers at once, instead of one at a time.
  • Version 1.1 (02/14/1999)
  • Monitoring of each game server has been added
  • In addition to launching EQ, you can now have it play a sound file instead, or do both.
  • You can now select which server you wish to monitor for launch and sound events
  • Version 1.0.6 (02/13/1999)
  • Fixes a launch close bug where EverMon would mot fully close when it launches EverQuest.exe
  • Fixes a bug that would crash EverMon if the wrong file/path was provided for the EverQuest.exe file.
  • Version 1.0.5 (02/12/1999)
  • Winsock Socket communication routines are now internal to EverMon, which should fix MSWINSCK errors (we no longer need it!)
  • A bug which caused patch files to be downloaded to the wrong directory, when EverQuest is launched on chat server UP has been fixed
  • A bug which prevented the "Close on Launch" check box on the options menu from being saved has been fixed
  • Version 0.98 BETA (02/11/1999)
  • Chat Server Up/Down status will now correctly display the current server status
  • A new update feature has been added to allow clients to know when the latest update is avalible
  • Verion 0.95 BETA (02/11/1999)
  • Timer cycling now works
  • Communication attempts will now time out after 15 seconds if no connection is made
  • The Time cycle count-down display has been added to the main form
  • A Connection Type display has been added, which will be part of a future update.  Currently, this will only show "Direct" meaning that you are monitoring the servers directly.
  • Distribution files are now using a new format
  • An EM icon has been added to replace the stock Visual Basic icon
  • Version 0.9 BETA (02/10/1999)
  • The first publically released version that started it all!
