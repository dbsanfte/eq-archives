 Welcome - March 2, 2001 - {Grim} 
It is our pleasure to welcome you to the opening of Guildtools.Net. This site was designed in mind for Guild Leaders who don't have the time to spend building/learning web design for their Guild.
Currently we offer a Guild Roster Database for the MMORPG EverQuest. We will soon begin work on the Ultima Online and Asheron's Call Roster Database's.
We hope to offer other tools for Guilds in the near future, possibly including Message Boards and a Guild Item Database.
If you have any Questions, Concerns or Ideas, please view the FAQ or visit the Forums and leave a message.

I would like to make a few points here to begin with.

- While it is possible to keep your guilds roster in a rudimentary table on a webpage, we feel that the Roster Database is a better alternative. The Roster Database will allow Admins to create/modify/delete accounts (including multiple admin accounts) and character profiles.

- Every database is set up to allow any guild member to create a custom user name and password to access the database. The user can then create as many profiles as he/she has characters. These profiles can be modified at any time.

- Only the Admins can modify/delete any users accounts. Normal Users without admin permissions can only modify/delete profiles they have created via their user name.

- Guild Masters that currently have websites can import or link the character profiles into the website with just a few lines of code. (More information on this will be able to be found in the Admin Section of the Forums)

- Roster Databases will be released in versions. Over the course of time it will undoubtedly be pointed out that key features would make the database more efficient and useful. As these features are made available in a new version the Admin of the board will have the descision whether or not to upgrade or stick with the current database. (As of now it appears that if you opt to upgrade you will need to rekey all information currently in your database. Of course, if we find a way to port the old database file to the new version we will try to make this available to you.)
