All video is good except switching resolutions or turning off Screen Same Resolution which results in a crash to the Desktop.
 
With WildTangent installed (which is a popular plugin for WinAmp) the EQ video modes try to usurp the WildTangent PID (1244 ) which creates an Access Violation c0000005 at address 004F4163.  After ripping out WildTangent (which requires a reg hack and the incorrect info for it is given at http://wildtangent.com/support/faq.html) the same error occurs but now at PID 788 with the same exception and address.  Explorer.exe was at PID 784.
 
For eqgame.exe to be reaching so low in the PID's as to be +4 from explorer.exe and causing an access violation means that EQ is trying to access the kernel and is approaching the API the wrong way.  When WildTangent is installed it appears it is trying to access the kernel via the WildTangent API rather than the pure Microsoft one.
 
On a side note I cannot use the Glide driver option.  If I do my system hardlocks at the 3DFX splash screen before loading the EQ splash.  I have to select the Primary Display Driver in order to play at all.
