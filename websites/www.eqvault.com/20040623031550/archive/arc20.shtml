Shadows of Luclin System Requirements Change
- Aurum @ 8:09 PM PST
Better graphics for more system power was the exchange announced by Alan VanCouvering on the Developer's Corner today:


    quote:
    -------------------------------------------------------------------

    During final testing of the Shadows of Luclin expansion, It was found that the minimum system requirements needed to be adjusted in order to take full advantage of the new graphics features included.

    For information on this change--including new screenshot comparisons of old and new character models--check out the Shadows of Luclin Minimum Specifications page.

    -------------------------------------------------------------------

    The comparison shots are extraordinary, you should check them out.

    Alan
The new minimum system requires are as follows:
  • Windows® 98/2000/ME/XP
  • Pentium® II 400Mhz or greater
  • 256 MB RAM
  • 16 MB Direct3D compliant video card and hardware T&L (i.e. Nvidia GeForce/GeForce 2)
  • DirectX 8.0 compatible sound card
  • 28.8 K+ Internet connection
  • 4X speed CD-ROM
  • 450 MB+ hard drive space

    Note: Although technically below the minimum specs, players with 128 MB of RAM will still be able to run Luclin by turning off the new character models.
The big change was in the RAM department, here's a blurb on it here:
    The amount of memory required to display 40 all-new models with high quality textures, realistic bone structures, etc., greatly increased polygon counts and as such we have changed our minimum memory requirement to 256MB of RAM, and our recommended to 512MB of RAM.

    Although technically below minimum specifications, EverQuest: The Shadows of Luclin WILL run with 128M of ram, and you will be able to see all the new zones, the new race (Vah Shir) and the new class (Beastlord) without much decrease in performance. However, if you want to see all the new models you will need at least 256MB of RAM. Even at 256MB you may need to modify new graphical options we will be providing. If you want smooth performance and fast loading times, we recommend that you get 512MB of RAM.

Lone Morning Infobit: DX8
- Aurum @ 3:03 PM PST
Mr. VanCouvering returned to the topic of the DirectX 8 upgrade being implemented on the Test Server this morning on the Developer's Corner:

    Greetings from your volunteer editor...

    Unless they're planning on changing it before next week, as of 11/7/01 the file was still called dbg.txt. Even on Test Server (where you never know what they're going to change next... look out for that level 10 fire beetle!)

    - Xelene

    Correct as always… sorry. I'll edit the post to reduce confusion.

    Thanks.


    Wouldn't it make sense to have people delete/rename the dbug.txt file before attempting the test? While bugs may be inconsistent (not readily repeatable), you can't call a bug a fluke. It's still a bug you need info on regardless of how hard it may be to recreate.

    Jzonn
    Ayonae Ro

    Well, yes and no.

    Sure, a bug could only happen once and be very hard to reproduce. We know all about that.

    But if the bug can't be reproduced by some methodology, then we'll have a very, very hard time finding it ourselves.

    And if something only happens once and you can't get it to happen again, then it usually was caused by something on the computer being out of whack (that's the techie term, I'm sure). Perhaps some software that looked like it closed didn't and was hogging memory, or maybe there was some sort of hicup on the PC itself.

    Unfortunately, if you can't repeat the problem then we probably won't be able to do so either, and that means that it will be very hard to fix. And, well, if it doesn't come back… :)

    Yes, I am encouraging people to try to reproduce a problem before sending it in to us. It's infinitely more efficient for us if we have some confirmation that the problem can be reproduced. Otherwise we have to spend a lot of time building a computer to match the one that the customer is using and then spend a few hours trying to reproduce the problem so we can find it and fix it.

    We certainly want all reports of problems, I'm not trying to discourage folks from sending their bug reports in. But we will tackle the ones that are repeatable much more quickly just by their very nature.

    Alan

DirectX 8 on the Test Server
- Aurum @ 8:28 PM PST
The move to DX8 is coming to a Test Server near you. Here are the details from Alan:

    Some time about the middle of next week we plan to put the files for DX8 onto the Test server for you all to download if you'd like to help us test it out.

    What we really need here is for people with all sorts of configurations and hardware and such to try this out and let us know if there are any problems.

    Please try to make sure that you have updated drivers for both your sound and video cards, and that you are indeed running DirectX 8 before you test out the new files. See this post for some suggestions about how to upgrade to DX8.

    If you encounter a problem (assuming you've done all the stuff above), here's what you can do to help us fix the problem:

    1) go into your EverQuest directory and delete your existing dbug.txt (this file can get really big, and having a fresh one helps us a lot).
    2) go back into the game and see if you can recreate the problem. If you can't, then we need to consider it a fluke.
    3) if the problem repeats, then get the following information:
    3a) your dbug.txt file (in your EverQuest directory)
    3b) your dxdiag information (just go to Start, Run and type dxdiag.exe and tell it to Save all information – send us the resulting file)
    3c) a detailed explanation of what happened that caused the problem
    4) send all that information to us at Luclinbugs@station.sony.com. Please use the subject: DX8 Problem.

    We appreciate any help you can give us with this upgrade. We really don't expect there to be too many issues, but we'd like to catch any of them that we can sooner rather than later. We just can't test on enough systems internally to prevent them all.

    Thanks in advance for your help.

    Alan

Guilds on Antonius Bayle
- Aurum @ 5:29 PM PST
Players on the Antonius Bayle server should give this message from Alan a look:

    In our initial enthusiasm to help get things rolling on this new server, we made a mistake with guilds.

    Basically, we started creating guilds before we removed all the guild associations from the characters. It tends to cause all sorts of trouble when we add someone to a guild when they are already associated with a guild (even if that guild doesn't really exist).

    So we had to wipe all the guilds from Anotnius Bayle and start over.

    We will be creating guilds again on Monday the 19th.

    IMPORTANT: If you were in a guild before this wipe, we need you to make certain that you zone at least once before we recreate your guild. This should clear the guild name from above your head if you still have one. And it is a good idea to do it even if you don't have a guild name over your head, just in case.

    We're very sorry about the confusion and the delay.

    Alan
