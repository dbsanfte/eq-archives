Valdanov does not summon, but will gate on low health and will regen to full if you let him do that and don't re-agro immediately. It's possible to kite him right off the balcony and if he's close enough to spawn or out of mana he will not gate.

Remember:

Mob_01 begins to cast the Gate spell.
Mob_01 beings to cast a spell.

...means they are casting gate.

Mob_01 begins to cast the gate spell.
Mob_01 tries to hit YOU, but misses!

...means the script to make them cast gate went off, but they have no mana to actually cast it. If that happens then they will not attempt to gate again and you can continue kiting.

HOW TO TELL THE CORRECT ASHES BEFORE YOU BUY THE WRONG ONES

Type ";"<insert item link> press enter to send the message

It will return an error saying it couldn't find <item tag parse to full text>

Look at the first few numbers on the giberish that the command returns...

if it says "0031751", those are the fake ashes, they are useless.

if it says "0007397", those are the correct ones.

What you are seeing is an error returned by the /tell shortcut ";" because it couldn't find a player by that name. That's because you specified an item link not a player name. The error returns the internal item link code. (Which actually is the reason you can make fake link socials, see The Sekrits: Revealed! (http://www.thesteelwarrior.org/forum/showthread.php?t=5430) thread for more info. Anyway, each item has a unique code, the first number in the item link code is that unique code.

The method above for determining the correct ashes didn't work for me so I did some poking on Alla's and found this:


OK here is how to tell them apart. All of the credit for figuring this out goes to the Paladins of Norrath Message board.

When checking to see if they are the correct ones do a ;t and then past the link of the ashes with no spaces between the t and the link. The HEX code will appear for the item you linked.

Here are the HEX codes for the ashes both bad and good:

t0007397-00001-00001-00001-00001-0000128E219BAVampyre (Good)

t0031751-00001-00001-00001-00001-0001B04FC7F5Vampyre (Bad)

If the HEX code does not match the good one listed above then they are the fake ones. This method has been proven and tested.
