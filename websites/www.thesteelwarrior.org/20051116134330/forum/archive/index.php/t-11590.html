Like they said, you can't do all those thing with one button push.

This is the macro I use for just about every warrior ability (don't use it for things like rend, that can be used in multiple stances). If you are in the correct stance, pressing the button activates the skill. If you aren't in the correct stance, pressing the button changes your stance to the correct one. The next press uses the skill.

/script texture,name,isActive,isCastable = GetShapeshiftFormInfo(3); if isActive then CastSpellByName("Recklessness()"); else CastSpellByName("Berserker Stance()"); end;

I think that's correct, not sure if Recklessness has a rank. If it does, you'd put "Rank 1" (without the quotes) or whatever rank it is in the parantheses after Recklessness.

You can use this macro for any warrior skill. Just change the "GetShapeshiftFormInfo(3)" to the correct number for the stance you want. Battle is 1, defensive is 2, berserker is 3.

The spell name in the parentheses is the skill, with the rank inside the other parentheses. Then the stance you need to use the skill is inside the last set of quotes.

If anyone has covered this before, apologies. Haven't been here long.

CAVEAT - Before the last big patch, you could hit this macro at any time and switch to the correct stance, even before the skill was useable (overpower, execute). Now, the skill has to be useable (button lit) before you can switch stances. Very dumb, you just have to remember which macros have skills that can be used at any time, and use those to switch stances. Or have a dedicated stance hotkey.

Hope this helps.

For abilities that can be used in multiple stances. . .(ie, Execute)


/script texture,name,isActive,isCastable = GetShapeshiftFormInfo(2); if not isActive then CastSpellByName("Execute(Rank 5)"); else CastSpellByName("Berserker Stance"); end;


Instead of checking to see if you're in the right stance, this checks to see if you're not in defense stance.
