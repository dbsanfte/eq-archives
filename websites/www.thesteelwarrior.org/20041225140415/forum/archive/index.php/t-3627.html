Originally posted by Roughbeard
All Independant procs, and if you want to see logs There should be some in the bunker and maybe some in the Overlord Sanctuary. All independant though and have been tested many times.

Close, but not exactly. If a weapon has both a main proc and an augment proc, the augment proc only goes off if the main proc fails to go off. In other words, when you swing, the game checks to see if the main proc goes off first, then if the main proc doesn't it checks to see if the augment proc goes off.

So if a weapon has both a main and an augment proc, the main goes off slightly more than the augment. Because the chance to proc per swing is fairly small, the difference isn't too great.


As a hypothetical example, let's say each proc has a base chance of 1/20 to go off per swing. Then:

- every time you swing, there is a 1/20 chance to get the main proc.

- Every time you swing, IF the main proc fails there is a 1/20 chance for the augment proc to work. Therefore the chance for the augment proc to work would be 19/20 * 1/20, which is just barely less than 1/20.


So mathematically they're not independent. The difference is so slight, though, that for practical purposes they may as well be.

It makes sense. Procs fail constantly the majority of your swings. They roll to see if they go off every swing. That leaves a lot of rolls for the auged proc.

If you swing 20 times and never proc, the original proc rolled it's dice 20 times to see if it went off. The aug proc also rolled it's dice 20 times to see if it went off. If the original proc wins 1 roll out of those 20, that still leaves 19 rolls for the auged proc to try and win. It could very well win 3 of the 19, making it proc 2 more times than the original did during those 20 attacks. Random roll of the dice.

If the original proc goes off 3 times out of the 20 cleaves, that still leaves 17 chances for the auged proc to win it's roll.

Think of them as completely independant, with the added rule that if the original already went off that swing the auged proc won't even throw it's dice. They can both go off any swing if they win their roll, only they both can't occupy the same space at the same time. And the original proc takes precidence over the auged one as to which one gets to go off that swing. It's the default.

I think some of you are confusing swings with rounds.


You can proc multiple times in a round, but not multiple times on a swing.

So you can't proc the 2 procs in the same swing, but you can proc them in the same round.

Example!


Swing Proc y/n
20 n
0 n
14 yes proc (proc 2 cannot proc now in this current swing)
15 n
16 yes proc2


All in one round. That is just a random round, which would be a triple attack + a quad.

So while it looks like you proced 2 times in 1 swing, you just proced twice in that one round.
