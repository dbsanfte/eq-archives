the 1ft KB procs arent really a big problem, even if there were 20 people using them. the probability of just 10 people proccing in the same 2 seconds it takes for the server to update the mobs location before the mob jolts up 5ft is almost a statistic impossibility. lag however, can make it seem like everyone at the raid just landed a KB effect all at once.

the real problem is the long duration stun spells. when a mob is stunned for a longer time frame, a message is sent to the server telling it that the mob should be stationary. but there are those 20-30 people beating away on the mob generating melee push. lag again, making it seem much worse. so the server knowing that the mob should be stationary caculates all the push and KB at once for part, if not the entire duration of the stun. thats why when a mob is stunned you're still hitting it for a second or 2 after it's jolted out of your range.

with GBoS CHoS and the like, the mobs (up to 55) arent even getting stunned. with the 65 stun procs the mobs arent even stunned for a full second, so the server only has to caculate the KB applied to the mob in the 0.0 second time frame when the mob was actually stunned before the mob is no longer regestering as stationary. which makes the KB from those procs go a lot smoother.
