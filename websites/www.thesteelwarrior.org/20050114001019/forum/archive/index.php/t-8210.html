/deny group -- removes the flag

/deny guild -- removes the flag

What you list as the new displayed types for augs will probably be replaced at a later time by a more easily human-parsed display. I'mma go ahead and bet that they're displaying type 6 augs as type:32, 5 as type:16, 4 as type:8, 3 as type:4, 2 as type:2, and 1 as type:1.

What you're seeing is the decimal representation of a binary string of 8. Rather than have a type denoted as a single number, it's instead handled by an array of on/off numbers, convenently expressable as a binary number. So, type 4 is now type 00010000, type 7 is now type 00000010, etc. A type 2/7/8 is now type 01000011. A type all is now type 11111111. For every combination of slots, there's a unique binary array, which we can then convert to a traditional decimal number: Type 2 is type 2, type 7 is 64, type 8 is 128, type 2/7/8 is 194, type all is type 255, etc.

I don't expect to see a lot of 2/7/8s, but I do expect a lot of 2/8, 3/8, 7/8.
