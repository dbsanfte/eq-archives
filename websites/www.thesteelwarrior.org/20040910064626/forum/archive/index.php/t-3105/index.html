Aethor
Ok, I'm not a warrior... a SK... but I was observing warriors for a while and indeed they need some fixing.

Now, the central problem is the fact that all the agro generation that warrior has is not certain to work when you need it; all it is, is a RNG (random number generator).

You may have EB and TB procs, and these procs would indeed get you agro - if they proc. And you cannot control it.

Simple change would be, for all warrior weapons, change the effect from Combat (a proc, which may or may not trigger) to Triggered (right-click when you want it).

Make it 0 cast time, so that you won't be interrupted even with no channeling skill, and that you can use it when needed. Balancing should be done through recast time so that designers can set it for example to 8 seconds, and if it proves too much they can increase it.. if it proves not enough they can easily decrease it.

This would work in a group like this: puller pulls, warrior right clicks EB, gets agro instantly. After tash / slow / whatever, if you need more agro then click your other weapon (dual wielding should be useful), thus adding agro to what you have.

This change should not require much work on developer/designer side, and it's fairly easy to balance by adjusting recast times. It would give instant agro on demand, which would solve 90% percent of warrior problems.
