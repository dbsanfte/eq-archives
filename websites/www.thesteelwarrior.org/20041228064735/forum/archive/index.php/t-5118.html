Actually, before the new mitigation changes went in, I was trying to figure out a better damage model than is currently used.

DMG = DB + DI * RND(20)

If this was completely true, then a MOB could only ever hit for 20 different values, which they don't. I think I've found diparicies that shows you have a small chance to have a "bonus" mitigation in some form which can be anything from 1 damage to about DI/2 and on top of that.


The reason a mob doesn't always hit for 20 different values is because the 20 different possible values are then passed through the mitigation factors of its target. So if you want a model for it....

If:

DMG = DB + DI * RND(20)

then:

ADD = DMG * [MP * MV]

where:
ADD = Actual Damage Dealt
MP = Mitigation Percentage
MV = Mitigation Variance

Or you can simplify it and say....

ADD = [DB + DI * RND(20)] * [MP * MV]

now to further define the variables I have thrown up there.....

MP = whatever mitigation percentage the war has from base + AA's, AC, etc....

MV = The variance (if any exists) in mitigation, aka a check to see if and how well you mitigate etc...

Overall its not that big of a stretch to think something similar to that is going on....
