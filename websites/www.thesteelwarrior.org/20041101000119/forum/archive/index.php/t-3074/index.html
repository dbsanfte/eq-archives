Septikk
I'm sure everyone is looking for hard information regarding the procs/proc rates of LDoN augs and I'd like to start a thread with HARD DATA regarding proc rates etc.

Today I purchased Icicle strike (http://lucy.fnord.net/spell.html?id=4376&source=Live) from Everfrost for my 100 points because I wanted to see how the proc aug's worked, at what rate they procced etc. I wanted to know before spending the big plats so to speak. :)

Fortunately (and what makes this neat) is that since the proc is on the same weapon that means that skill, haste, dex and other information is essentially irrelevant because it's going to be the same for both procs.

I only log my weapon hits (of which I had 2932)

---Force Shock (179DD) (original proc)
........................................65 full damage procs
........................................13 resists
........................................ 3 238Dd criticals

Total procs: 81

---Icicle blast (75DD) (aug proc)
.......................................86 full damage procs
.......................................4 resists
.......................................2 100hp criticals

Total procs: 92



I would like to have folks add further parsing to this with other procs to see if things like the lifetaps and Ykesha procs occur at the same rate.

PS: Calcing my total damage, adding this proc on my mainhand upped my DPS by about 2 percent.

My personal experience is that additional proc doesn't affect the weapon's innate proc rate. A cleric friend of mine did perform a short test. He spent a set ammount of time proccing his cleric hammer on a mob (enough for 100 cleric hammer procs or so) before buying an additional proc. KhalimBloodhelm
I'm guessing that if what the above said is true, the augment procs probably work something like this:

On every swing, the algorithm used to determine whether the weapon procs or not is run on the chance of proccing the inherent proc. IF the inherent proc does NOT proc, THEN the computer runs a second algorithm to determine if the augmented proc will go off. Now, since there will be slightly fewer swings when the augment proc actually has a chance to go off, I'll bet that the formula is set for a slightly greater proc percentage than the normal proc formula. Hence, you get the same or greater number of procs despite the fact that you have fewer swings the augment can proc on.

A few quotes from Absor re proc's:

Quote:
--------------------------------------------------------------------------------
Yes each proc gets its own proc check. Although only 1 proc may go off per swing. So you won't double proc but you can single proc with either proc.
--------------------------------------------------------------------------------

Question:
so, adding say, a dmg proc to a primal weapon, you'd end up with Effect: Avatar and EFFECT: Vampire Kiss (for example) and, depending on the RNG, you'd get either Avatar OR VK as a proc, but never both. Is that correct?

Quote:
--------------------------------------------------------------------------------
Yes thats correct. The primary proc will be the one to go off before the secondary added proc. So if you had a primal / Vampire and primal checked ok it wouldn't even bother checking vampire.
--------------------------------------------------------------------------------

*** My (Wompem's) speculation***

So in theory each proc checks per swing, but there is a hiarchy:
1) Original Proc Check per swing ( if no proc then goto 2 )
2) Augment Proc Check per swing ( if no goto 3 )
3) Buff Proc Check etc etc.

Some people may post about 2 procs from one swing but I would speculate it was a missed double/triple/fericity or what have you. Most of us have had a 2+ procs on the same combat round but this would be attributed to double/triple atk. Further testing to come, but this seems reliable and has a good background to it.

We must remember there is no real magic, but instead a cumputer language, and SOE probably wouldn't want to write something too seriously difficult because every extra bit of code is more to process.
