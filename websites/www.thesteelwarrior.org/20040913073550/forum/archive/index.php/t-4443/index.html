It has been a few years since I took a course on this but basically this is how I understand it:

Basically, there are no truly random numbers generated by general purpose computers. The reason is that there is not an efficient way to cheaply utilize a random effect to create truly random numbers. To actually do that you would need something like a small radioactive peice of material and a geiger counter or some similar setup incorporated on the motherboard. This would be too expensive and not really worth doing for most things. There probably are computers like this at places like the NSA but don't expect to walk into Best Buy and get a random number co-processor because it would probably cost a few million dollars.

Instead, what is usually done for speed and efficiency reasons is a relatively large number (call it the base number) is picked and an algorithm is used that uses the mod function and a seed number to generate psuedo-random numbers in a cyclic fashion (the sequence repeats after a while). In most of the common algorithms, the numbers will fall along a series of parallel straight lines if you plot the Nth number vs the (N+1)th numbers of the sequence in a plane and for this reason, this is called a linear method. This method is fairly good at producing what appear to be random numbers to the human eye but actually are not. The problem is that if you use the same seed value, you get the same sequence of numbers and if you use too small of a number for the base number, the cycle of pseudo random numbers will repeat too soon. Certain tricks are often used to offset the repeated seed problem, such as taking the system clock value for the seed and this works fairly well but the seed must have a sufficient number of possible values to be good enough to use (example--you wouldn't want to use a die with 6 sides to pick seeds because you would just get 6 different pseudo-random sequences even though a die roll is a true random process).
