Well EQ companion can parse diff weapon setups in the same parse as long as they are different skills. Like for example my dual wield setup is two swords, while my two hand setup is a blunt weapon so those are easy to seperate. Other than that the parser doesnt know what weapons you are using.

It is best if you are going to parse a certain combination of weapons vs another to use one set of weapons for an hour or so then say in chat "TEST TEST TEST TEST TEST TEST WEAPON SWITCH" or something, then switch to the other set of weapons. After you log for the night, then you can search your log file for the "TEST TEST TEST...." line and cut the log up into two logs and then parse each one seperately.

The program will distinguish between different damage types (crushing, slashing, piercing, etc...) but it won't know what weapon (or even single or two handed) did the damage.

What I do if I am testing something specific is say something like, "Parse test - hammer main and Blackout off"

Then I do my test.

When I go back to look at my logs, I can use the FIND feature in notepad/Word to search for "Parse Test" and I can rapidly find the correct section of my logs.

I usually then copy the appropriate section out to a new document and parse only the part I am interested in (particular weapon/buff setup).

One of the features I will be adding to EQ Companion soon is the ability to 'tell' it which weapons you are using. The next (unreleased) version already has a new feature where you can assign a 'Command chat channel' and you can send tells to that chat channel to command EQ Companion.

When it is done, you will be able send a tell to your chat channel and say something like '/1 Primary SoD' and '/1 Secondary LSoEE' for example. EQ Companion also have 2 new columns in the fights list for primary and secondary weapons. From then on, each new fight will have 'SoD' in the primary column and 'LSoEE' in the secondary column.

Then when you switch to a 2-hander for example, you can use command like '/1 2-hand WB' for example and then the primary and secondary columns will show 'WB' for all the fights that follow.

This is something I have wanted to do for a long time already and now that I implemented the command chat channel feature, it will be easy to implement. The next 0.792 version will not have this yet, but I have a feeling the version that comes after will.

I actually ended up implementing this last night (it only took about an hour to do) so when I release 0.792, hopefully real soon, this will be available.

In the mean time, you can already prepare for this so you have a log file that shows your weapon info when you get 0.792.

The way this works is you have to join a chat channel and send tells to it to let EQC know your weapon names. The name of the chat channel can be anything, but I would not recommend using something as generic as 'EQC" or 'EQCCommand' for example because other people who start using this feature might join the same chat channel. I would use something like 'SilgurdarEQC' for example if your character's name is Silgurdar (that's my warrior's name).

You can use the /autojoin command so you will automatically join this channel every time you log on to EQ.

Once you have joined the chat channel, you can send tells to it using its channel number. The first channel you join will be number 1, etc. To send a tell to the channel, you would just do a "/1 Hello!".

Your log file will then have a line like:

You tell SilgurdarEQC:1, 'Hello!'

If you have assigned 'SilgurdarEQC' as your command chat channel in EQC, the 'Hello!' will be treated as a command.

The commands are:

PRI weaponname
SEC weaponname
2-H weaponname
ETC etc_information
EXP experience
AA aa_experience

Using command "/1 pri SoD" will tell EQC that your primary weapon is SoD. From then on, every new fight will have 'SoD' in the 'Primary' column of the fights-table.

If you use "/1 2-H WB" command, your secondary weapon will be cleared automatically and your primary weapon will be assigned as '2H: WB'.

The 'ETC' command is for showing buffs, etc. stuff that affect DPS and that you would like to show in the fights-table.

The 'EXP' and 'AA' commands are if you want to track your experience gain. When ever you notice your AA experience for example increases, you can send another '/1 AA 30" command for example to show that now you have 30% AA experience.

All of these PRI, SEC, ETC, EXP and AA are shown in their own columns in the fights table. They are hidden by default so if you never use ETC-command for example, the column will be hidden.
