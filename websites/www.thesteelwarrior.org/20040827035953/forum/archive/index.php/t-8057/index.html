Reposted from the link above's link to GUcomic forums:

UPDATE: I am just starting to get more information on this. Here is what I've heard.

We are currently no longer manufacturing or selling EQMac. However, we are looking into working with other distributors/retailers. The Store will be following up on this later this week.

The reason has nothing to do with cancelling the game itself. Rather, it seems that the availability of the game is low, while the quantity of (theoretically) available copies is high. There should NOT be a shortage of EQMac given the number of units that were made vs. the number of units that were sold. This is also being looked into.

Lastly, this is not a permanant decison, rather one based on simple economics. It seemed like an unnecessary drain on resources to pay to make more when we could simply move existing merchandise around to cover the demand. Should the number of available copies get low, and the demand for new ones increase, then obviously it would be beneficial to crank out more batches. Right now though, there are just so many units in the distribution centers that it would be silly to keep making more. What is already out there should cover demand for a LONG time.

As others have mentioned, this doesn't seem to be the case, SOE is looking into it, and the Store will look into locating a suitable vendor in the interim.

Again, please consider this news to be very "beta." If it changes, I'll let you know. I just understand how eager everyone is for news, so I'm making an effort to let you know what I hear when I hear it. If I was misinformed, I will correct it in a future post.

On a completely different topic, Vance asked a few posts back IF the game shut down, if I would miss our "conversations."

The answer, without sarcasm, is an unequivocable "yes." Please bear in mind I've moderated EVERY SOE community there is: EQ, EQOA, EQMac, SWG, PS, CON, LOE, etc., etc., etc. While EQMac has come with a set of unique challenges, it was also the first time that not one but TWO community members took the time to figure out how to call me at work, just to tell me "thank you for your time" or "have a good weekend."

However, at this time, from what I've been told, it hasn't come to that.

Regards,

Stephen R.

Gyrfalcon
Arathena, I am quite well aware of the WineX project and how far it has come. You still have the issue that it is at the root an emulation of Windows APIs and then translating them into API calls in the Linux environment. Not an easy thing when you are talking about porting DirectX instruction sets and then using a table to translate them into OpenGL instructions. Going through this process, any game running in WineX is lucky to get even 40% performance throughput of what a Windows user would experience on the same hardware, and the Windows user doesn't have to enjoy all those lovely artifact issues that crop up from time to time. While this is usable for most business applications, trust me, for gamers, it really sucks. Also, it must be pure hell for project programmers whenever Microsoft updates it's API sets (such as DirectX).

What I am talking about is a completely different animal. A version of EQ that runs NATIVE to linux (like they did for the Macintosh) and makes use of OpenGL native to the environment.
