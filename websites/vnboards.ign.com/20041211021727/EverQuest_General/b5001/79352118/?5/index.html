Got this description online:


If you get this message, it means that the file being read by your PC or software is corrupted. However, it does not mean all the data is lost forever. When data is transfered, it is usually in small blocks and each block is given a CRC value. If something goes wrong with the data between the time it leaves the source and arrives at its destination, the CRC sent at the source will no longer match the one that is calculated when the data arrives - this is when the cyclic redundancy check error will appear.

The most common times you will see the cyclic redundancy check error message is when trying to read data from a damaged CD or DVD. Just before it appears, your CD/DVD drive will probably grind and whirl away - your PC may also become a little slugglish.

Less frequent causes are the result of system crashes, and buggy software (hello Microsoft), incomplete downloads (often identified by the misleading message

CRC (Cyclical Redundancy Check) errors don't necesarilly mean that the source media or files themselves are corrupt, but the errors do mean the data in the copied/tranfered/converted/whatever versions of the files does not exactly (bit-for-bit or Byte-for-Byte) match the data in original. Anything involved in the chain of the copy process could be responsible for the errors; bad media, faulty drive-controller circuitry, bad RAM are all possible culprits.

Most likely, it is trying to copy files to a spot on the hard drive that went bad.
Run scandisk. Hopefully it will mark that sector and let you write to another.
Another work-around is, instead of deleting the file that is corrupted, rename it, so that section of the disk is still considered taken, then let the reinstall copy it over again somewhere else.

While it is possible that this is caused by a bad driver or errors on the target drive (the harddisc), I find that 9 out of 10 times, this error is caused by a faulty/damaged/dirty CD.

And actually, the fact that this is a new disc that has not been used before, almost guarantees that it *is* infact the CD causing the problem. Contrary to what many people think, professional CD-production is an extremely delicate chemical process with room for lots of error, and high rate of discarded discs. Now add possible error from packaging, transport, and user handling, and you will have no guarantee that a 'new' CD will work.

Suggestions (in order):
1) Scandisc the harddrive. As mentioned, I dont expect this to be the problem, but it can never hurt and easy to do.
2) Wash the CD. Yep, thats right. You can try special CD cleaning fluids, but normal dishwashing liquid and a little water, followed by a soft cloth rubbed in smooth circles around the cd, will do the trick. Just remember that the CD has to be completely dry before play. This trick has made about half the CD's I've done it on, playable again. I've been a proffesional in the industry, and yes..thats what we do.
3) Return it to the store as a faulty product. You could try testing the CD on another computer first if you want to be sure.
