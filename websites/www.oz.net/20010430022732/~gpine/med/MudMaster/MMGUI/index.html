DLLs need to be modified for use with the GUI version of Mud Master. These DLLs here will work with the Console version, but normal console version DLLs will not work correctly with MM2k. Click here for dlls: ChannelLog.dll(15k), MM Misc.dll (10k), Math.dll(6k). Other dlls that are not specifically updated for use with the GUI will NOT work. Will get more DLLs online soon!

A note about fonts -- Most muds use a console type font. Basically it's a fixed point font set, so menus, maps, lists etc are nicely justified. A problem with a GUI telnet client is that people tend to like using fancy fonts. The downfall is many displays (maps, scores..) look like crap.
I edited the Lucinda LC font, which turned out to be a major pain in the butt, but it now works correctly and will print all the "@Chr()" characters just as the console version of MM will. The only difference between MM Console (DOS) and MM2000 is the @Chr(22) character. In the GUI it's an "overslash/line" thinger. This is a totally different version than the "MM2000LC.TTF" I previously posted. When you setup the font in MM2000, make sure you select the SYMBOL style, not Western.
Download MM2000LC.TTF (zipped) 32k

Installing Fonts:
  • Unzip the downloaded font into a temp dir somewhere
  • Double-Click "My Computer"
  • Double-Click "Control Panel"
  • Double-Click "Fonts"
  • Click the "File" menu
  • Click "Install New Fonts"
  • Browse to the directory where you unzipped the MM2000LC.TTF file
  • Select the MM2000LC.TTF file, the little dialog might take a minute to locate the font
  • Once it's selected, hit Ok (you might check mark the "Copy fonts to Fonts Folder" box)
  1. In MM2k Choose the "File" menu"
  2. Choose "Settings"
  3. Choose "Fonts", then select MM2000LC
  4. Make sure you set the font, via the font selection dialog, for SYMBOL style, not Western
  5. Load your session(s) and define the session font (Session->Font menu).
