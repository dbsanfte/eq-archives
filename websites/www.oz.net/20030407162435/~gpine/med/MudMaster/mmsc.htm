MMSC - Mud Master Script Compiler

MMSC allows you to write scripts more like code. You can break down each command, and add notes.

MMSC is "Beta", meaning it is NOT finished and is quite possibly buggy. I could use feedback on it, likes, dislikes, bugs, questions, etc. If you use it please comment so I know how it's working.

Included in the zip file is a sample script. The sample contains a lot of examples and comments, please refer to it for additional help. The sample is can be compiled just how it is, but I suggest you turn off the "Add Notes" option in MMSC.

Basic usage:
Installing: Put it whereever you'd like and just run it. Under the File menu choose "Open..." and open up a script file. The script file you open must be written FOR MMSC to compile. Compiling already created MM scripts won't do much other than create a 0 byte file.

After opening the source file, hit the "compile" button and MMSC will create a new file, in the dir where the source file was. The name of the output (MM ready) script is the same as the source file's name, but with ".MMSC" tagged on the end of it.

Commands in breif:
Adding notes:
/~ note spew
example:
/~ This is a note
/~ Below is a note inside a command
/action {%0 says, 'hi'.}
   /~ doing stuff below to get name of character
   {
   /var Name $0
   }
/*

Ending a command statement
After you've finished with your command you need to let MMSC know, you do that by putting a "/*" on a blank line after the command. Example:

/Alias {xxy}
   {
   say Ooooh phat loot!
   } {stupidaliases}
/*

MMSC will reconstruct the lines, from "/Alias" up to, but excluding, the "/*". The above becomes: /alias {xxy}{say Ooooh phat loot!} {stupidaliases}

You can add comments after the "/*".
/var {MyVar} {100000}
/* MyVar is really important, don't forget to /empty it when you burp.

Careful commenting with /*, /*'s MAIN purpose is as a flag for MMSC to know when to stop building a command string and print it to file. /*'s will add linefeeds to your output file, which is fine, just messy.

Streaming Lots of little things
You can "Stream" commands. Handy for when you have lots of little commands to add to MM. Good example is variables, typically having tons. Using /& tells MMSC that all lines following are indivigual commands and each line should just be added to the output file -- Until a /* is encountered, then stop the process. You can toss notes/whitespace in between commands, but each command must be one continuous line. MMSC won't construct commands when it's streaming. You can add a comment after the /& as well, Example:

/& I'm streaming a bunch of little things...
/var {MyVar1} {1}
/var {MyVar2} {2}

/~ Next var is used by my uberduber script
/var {UberDuber} {0}
/~ Reading in my great script here...
/read MySillyScript.txt
/var {MyVar3} {3}
/var {MyVar4} {4}
/* (end of streaming here)

Why stream? Because if you don't then you need to add /* after every single command, and that includes vars. So Streaming is handy for lots of short commands that you don't need/want to break down into "code" style.

MMSC strips all whitespace from in front of any line. So you can indent to your heart's content. You will want to possibly add trailing spaces though to make the compiled script look nice...
      /~ Hey look a spaced out comment
   /macro {F1}
     {
     shout AHHhhahhhhaha
     } {stupidmacros}
/*

Whitespace inside a command should be fine:
/alias {MyAlias}

   {
   /var Smart @Math(1+1)

   }
  {Smart Group}
/*
