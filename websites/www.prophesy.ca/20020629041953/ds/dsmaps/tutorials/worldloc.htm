 Ever wondered how the peeps over at GPG made it in the multiplayer world so that when you enter an area it says "So-and-so has entered suck-a-place"? Well this will teach you how to do it. First off, create a new text file in "my documents/Dungeon Siege/bits/world/maps/yourmap/info". Now add the following lines to the text file:

[world_locations]
{

}

This is the base for your "world_locations" template. Now we will add our location(s). Add the red lines:

 

[world_locations]
{

[t:location,n:yourlocation]
{
id = 0101;
screen_name = "the town of yourmap";
}

}

You can add as many of the RED entries as you like, replacing "yourlocation" with the location code you want to call on in Siege Editor, and increasing the ID number by one for each successive entry, so that you don't have any conflicting entries. The next entry would look like this (BLUE TEXT):

 

[world_locations]
{

[t:location,n:yourlocation]
{
id = 0101;
screen_name = "the town of yourmap";
}

[t:location,n:yourlocation2]
{
id = 0102;
screen_name = "the forest of yourmap";
}

}

Notice how the ID number has gone up one, and the "t:location,n:" has changed. When you are done, save the file as "world_locations.gas" in the appropriate folder (my documents/Dungeon Siege/bits/world/maps/yourmap/info), and make sure it isn't saved as "world_locations.gas.txt". If it is, rename it and get rid of the TXT. Next, you'll want to call on these locations in Siege Editor.

Find the place where you want the text to pop up, and create a "trigger_generic" located in "game objects/gizmos/triggers", or simply add a new action (set_player_world_location) to your mood trigger, if you have one. Add a new trigger in the "trigger properties" tab of the object properties window of the trigger_generic. You can get to this by clicking on the trigger_generic object in Siege Editor, then right clicking and selecting "object properties", then clicking the "trigger properties" tab. On the top left is the button "add". Click it, then click on the little + beside the world "trigger_generic" in the Triggers sub-window. The tree will expand, showing "Trigger_0". Click on "trigger_0", then on the right near the middle of the main window, beside the "conditions" box, click "add condition". Select "party_member_within_bounding_box" from the drop down menu that appears, and click ok. There should now be a black box surrounding the "trigger_generic". Adjust the Half-diag X Y and Z values in the conditions window until the black box coveres the area that you want the text to pop up at.

Next, Below the condition box, is the action box. Click the button "Add Action" beside that window, it's on the right near the bottom of the main properties window. Select "set_player_world_location" from the drop down menu, and click ok. In the value field to the right of, "World Location", enter the EXACT location that you created in your "world_locations.gas" file. That would be the following RED text, except what you entered:

[t:location,n:yourlocation]

That's all there is to "set_player_world_location"! Now, in-game, you merely have to hit the "backslash \ " key to see where people are located. To set the initial location when people enter your map, just create the same trigger above, with the location you want, and have the black bounding box of the trigger contain all of the start locations. BOOYA!
