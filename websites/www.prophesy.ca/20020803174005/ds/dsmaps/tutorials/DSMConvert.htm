Method 1

This program converts Dungeon Siege v1.1 maps (from the Siege Editor Beta) to the v1.0 format (That the unpatched Dungeon Siege understands)

Usage:
1. Build your map in the Siege Editor.
2. Save it as a .dsmap file somewhere. When saving uncheck the 'Include in build' under lqd options.
3. Use TankViewer to extract the contents of the .dsmap file and then delete it (the .dsmap file that is.)
4. Run DSMConvert.exe, click 'Fix map...' and select the main.gas file residing in the root dir of your (extracted) map.
5. Watch as DSMConvert changes the local guids to the global ones of all the areas in your map.
6. Use TankCreator to build a tank of your map again, place it in /Dungeon Siege/Maps/
7. Start Dungeon Siege and take a stroll through your new map.

Method 2


I found another (much simpler) way of using DSMConvert. This makes it possible to directly use the 'Save Map as .dsmap' option and then start the map in Dungeon
Siege without the hassle of having to uncompress/convert/recompress it.

These are the steps you need to follow (I know it's a bit long, but you only need to do 1-5 once):
1. First you need a map to work with, either if you choose to use a map you've already created (don't forget to back it up! very important!) or decide to create a new one.
Anyway, make sure you have a map with at least one region and a node and make sure its properly saved.

2. Close the Siege Editor and start DSMConvert.

3. Now click the 'Fix Map...' button and browse to your '/My Documents/Dungeon Siege/Bits/world/maps/%mapname%/' folder. (%mapname is, you guessed it, the name of the map you're going to convert). Select the main.gas file and watch as DSMConvert does it's job.

4. Now that the map is converted to a v1.0 map, you need to tell SE this. So open the main.gas file in your favourite text editor (Notepad, Skritpad, Ultraedit or whatever you use) In this file you will find a line that says "b use_node_mesh_index = true;". This is what tells SE to use the mesh index. Change the line to read "b use_node_mesh_index = false;", or remove it completely as it defaults to false. Save.

5. Start the Siege Editor again and make sure the map and region(s) loads correctly. SE will identify the map as a v1.0 map, and will save it in that format from now on.

6. Now, most of us probably tried making a .dsmap of our map even though we were told it wouldn't work (after all, the last thing a human being abandons is hope), and we all found that the map didnt show up in Dungeon Siege. This is because GPG changed the 'Min. Required version' tag of the saved .dsmap file so Dungeon Siege won't load it. But there is a way around this too. When you save your map as a .dsmap file the 'Min Version' box will read: "00.[05].2002.03.0801" but if you open up the saved tank in TankViewer and look, it has been set to "00.[05].4002.03.0801". SE adds 2000 to the third field to make sure the maps do not show up in DS as it doesn't support the new format without the patch. But, we altered our map, didn't we, it's in the v1.0 format and should load nicely. So, when you make a .dsmap out of your map, change the 'Min Version' box to read "00.[05].2.03.0801". SE will add 2000 to the field and it will read 2002, just like any other tank created by TankCreator (and the original .dsres files).

7. Save the .dsmap in the '/My Documents/Dungeon Siege/Maps' or the '/Dungeon Siege/Maps' folder.

8. Start Dungeon Siege, load your map and enjoy.

Remember, next time you want to export your map from SE, just save it as .dsmap, change the 'Min Version' box to "00.[05].2.03.0801" and play!
