 This tutorial will explain how to create a working elevator with 2 switches, one at the bottom and one at the top. I assume you are familiar with placing nodes and objects, and know what a "node door" is. I will say often in this tutorial to "copy the object SCID" and "copy the node GUID". For objects, just click on the object, then right click and select properties. In the window that pops up, at the top on the right will be a field called "Object SCID" followed by a HEX number (ex. 0x0dd4e901). Just highlight that number and right click on it, and select copy, or hit CTRL-C on your keyboard. For getting a node GUID, just select the node, go to the "node" menu up top, select properties, and the GUID will be the HEX number beside the field "Node GUID" in the "Genereal" tab. Just do the same that you would for an object SCID. Now that we have the technical stuff aside, let's make our elevator!

First, create the base of the elevator, or level 1. We will use the node "t_xxx_ledg_12-ele-bottom-a" located in "terrain nodes/generic/ledge". Place the bottom node, then attach floor 2 (t_xxx_ledg_12-ele-top-a) to door 5 of floor 1. Next, attach the wheels (t_xxx_ledg_12-ele-wheels-a) to floor 2 door 11. Lastly, attach the elevator platform (t_xxx_ledg-ele-platform-a) to door 8 of floor 1. Your setup should look like this:

Next, we will want to add all the objects that the elevator requires in order to function. First, place 2 "lever_metal" objects, located in "game objects/gizmos/interactive tools", one on the bottom floor, and one on the top. Next, place a "2-state, 2-connection, 2-node elevator" object, loacted in "game objects/gismoz/elevators", on the elevator platform. Next, create 3 "Use Point" objects, loacted in "game objects/gizmos/special", one on the elevator platform, one on the top floor beside the elevator shaft, and one on the bottom floor in front of the switch. Your setup should now look like this:

Now for the fun part, copying a bunch of object SCIDs, node GUIDs, and door numbers. Firstly, let's set up the "Use Point" objects for use with the switches. These objects move the player to that spot before they can activate the switch. First, copy the SCID of the Use Point by the bottom switch. Now select the bottom switch, right click on it, select properties, and paste the Use Point SCID into the value field beside "use_point_scids" in the template propeties tab. It will be at the very bottom of the main table. Next, copy the SCID of the Use Point on the elevator platform, and paste it into the same field, but seperate it from the first one with a comma and a space. It should look something like this: 0x00100004, 0x00100005

Do the same for the top switch, except use the top Use point and the use point on the elevator platform. If you did it right, there should be orange animated arrows pointing from the levers to their Use Points. Now we'll tell the switches what they are turning on. Get the SCID of the elevator object, and paste it into the value field "On-SCID" in the template properties tab of the properties window for BOTH switches. If you did it right, there will now be green animated arrows pointing from both switches to the elevator object. Now we need to tell the elevator object what nodes it is using. Get the node GUID of the elevator platform, and remember the door numbers of the left and right doors. Should be 1 and 3. Paste the GUID of the platform into the value field "elevator_node" in the template properties tab of the object properties window. Add 1,1,3,3, in that order, to the next 4 fields below. Should look like this (exept for the elevator_node field which will be unique to YOUR elevator node):

elevator_node: 0x9202d007

elevator_door_level1: 1

elevator_door_level2: 1

elevator_additionaldoor_level1: 3

elevator_additionaldoor_level2: 3

Now you need to copy the node GUID of the Floor 1 node, and remember the door numbers to the left and right side of the elevator platform. Should be doors 7 and 9, respectively. Paste the node GUID into the "connect_node_level1" and "additional_node_level1" fields in the template properties window of the object properties window of the elevator object. Put the door number for the left side (7) into the field "connect_door_level1" and the door number for the right side (9) into the field "additional_door_level1". Should look like this (except for the node GUID, which will be unique to your level 1 node):

connect_node_level1: 0x51DAE419

connect_door_level1: 7

additional_node_level1: 0x51DAE419

additional_door_level1: 9

Do the same thing for all the level 2 doors and the level 2 node, into the corresponding fields of the "level 2" values in the elevator object properties window. In the end, it all looks something like this:

One last thing you will need to do, is adjust the elevator's "Duration", so it doesn't seem like it's "teleporting" you to the top. To do this, right click on the elevator object, select properties. In the teplate properties tab there will be a value field called "duration". This is how long, in seconds, it takes for your lift to go from the bottom to the top, and vice-versa. Put in something like 4 or 6.

Congradulations! Save and export to .dsmap! If you did it right, it should work when you test it in-game! If it didn't you missed something, double check everything!
