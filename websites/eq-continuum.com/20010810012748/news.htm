December 22, 2000
  • I fired off a message to someone at Verant regarding the EverQuest Status Server protocol. When I get something back, I'll let everyone know.  Due to the holidays, I don't expect anything soon. Hopefully though, when I hear back from them, I will be able to use the information to build a server status script. The script used by EQ Vault is proprietary, and I'm guessing that all others are as well. I found a public domain one, but it is written in Perl and requires a scheduler to run. We don't own this web server, and don't have access to the cron, so we can't schedule anything. Besides, I think that I might be able to write an in-line script in VBScript or JScript that will be dynamic and refresh with the page. I would think that would be better for Verant anyway since it will only querry the server when someone hits the page, and not every few seconds as most other status scripts would. My only other option is to use EQ ServerScan , pull the information from the logs into a database, and script the page to pull from this database. To do that, though, I would have to employ the use of my server, and we all know about the problems that occur when my IP address changes.

--Adammair
