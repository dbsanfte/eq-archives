Ack! Where's my comment??

A good start...
Submitted by: NiteRaven on Friday, January 5th at 12:05am EST

Keyword indexing the entire database in combination with a new search script seemed to help out a lot during peak hours. Although the speed increase is nice, I'm not completely happy with the search results it's spitting back. For example, if I search for a "Runed Totem Staff", the number one item returned is a "Blued Two-Handed Hammer". This is because the word "staff" appears 7 times in the description of the item.

At this point I'm looking at two solutions, allowing + and - to be used in front search words to indicate this word must or must not appear in the item. The second solution is giving a score bonus to items that have all search words appearing in them. I'm leaning more towards the first solution since it puts more power into the hands of the user.

Both of these solutions require some pretty complex SQL selects, requiring me to do a bit of research. I've been reading over some chapters in the MySQL book by New Riders, a excellent book for anyone still reading this post and actually interested in what I'm saying. If anyone out there has experience in coding a search engine I would love to hear your thoughts and suggestions. I'll continue to make adjustments till I feel the search engine is in good shape then move onto other parts of the site.


Poor Site Performance
Submitted by: NiteRaven on Wednesday, January 3rd at 02:26am EST

I apologize for the poor site performance yesterday. Since the release of Velious this site and most other EQ information and spoiler sites have seen traffic numbers almost double. This is due in part to the large influx of new players to the game and the already huge player base scrapping for every bit of Velious information they can get their hands on.

I am currently in the process of optimizing code throughout the website to reduce server load, thus increasing overall speed. I've started by rewriting the number one killer of most websites, the search engine. One can generally do this by keyword indexing the entire database, instead of a brute force attack on it every time a search is performed.

The basic functionality of the search engine is in place now, so I can get some sleep, but will be finished and polished tomorrow. I will continue to make code optimizations throughout the next few days, which will hopefully bring EQ Maps back to its old glory. Finding the best information quickly is what keeps everyone coming back here, and I want to keep it that way.
