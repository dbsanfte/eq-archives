2008/1/25

CT1のクロススキルで遊んでたらカマエル専用武器も実装されてることに気づいた。（GMshopにないから未実装と思ってたらスキルで変換可能でした）
新スキルと新武器がそろってればCT1対応したいなあということで外部鯖をCT1にバージョンアップしてみました。
CT1対応しておけば今後の細かいバージョンアップも楽そうだし。

鯖アップデートの注意点としては

(1)dbデータ移行
(2)gameserver\config移行
(3)l2.ini書き換え＆system差分作成（CT1蔵を私的鯖に接続先変更を適用する）
(4)レイドボス石化対応

が対応できれば移行できそうです。

以下l2j:830 l2jdp:3320（インタールード）→l2j:981 l2jdp3957（ファーストスローン）に移行したときのメモ。

(1)一番のハードルはdb移行。

accounts
character_macroses
character_quests
character_recipebook
character_shortcuts
character_skills
characters
clan_data
clan_privs
items

をデータ移行対象としました。SQLyogでcsvエクスポート→db変更内容を確認してインポート　でとりあえずうまく行った模様。

charactersが大幅に変わってたので編集した位で、あとはそのまま流し込みでOKでした。
SQLyogでデータのインポート／エクスポートやる場合はデフォで50rowsしか表示してないので、そのまま吐出すと50個分のデータしか出力されないので注意。

(2)configは追加項目があったのでdfで比較しながら数値を合わせて完了。

(3)l2.iniはCT1対応のl2encdecでl2.iniの接続先を書き換えて、patcherあてて動作OK。起動はl2.exeからloaderCT1.exeに変更となります。
カマエル蔵インスト→最新パッチ当て→dec対応差分当て→loaderCT1.exeから起動でOKです。

(4)以前と同様にjava/net/sf/l2j/gameserver/model/L2Character.javaの
　＞// Check Raidboss attack
　＞if (target.isRaid() && getLevel() > target.getLevel() + 8)
　＞if (player.isRaid() && getLevel() > player.getLevel() + 8)
を8→80辺りに変更してリビルドでOK。

javaエラーで鯖が動かない場合は、リビルドに使用したバージョンのjavaをインストールしましょう。

ドロップ倍率は
(1)レイドボスからのアイテム入手が多すぎてゲームバランス破壊
(2)オートルートONでdbまで破壊？
というわけで変更しました。

大量アイテムドロップ→鯖停止→再ログインでクライアントクラッシュ　と大変なことになりました。
dbバックアップを復帰させたらクライアントクラッシュが治まり、一部プレイヤーしか影響ない感じなのでインベントリへの追加処理がおっつかなくてdbぶっ壊れるのかなと予想。
ドロップ数減らしても大量アイテムをオートルートした瞬間鯖動作がおかしくなるので、レイドボス時はオートルートoffにしておくのが安全です。

長期出張から帰還したので9レベル差以下レイドボスの石化対策をしました。

/java/net/sf/l2j/gameserver/model/L2Character.javaのCheck Raidboss attackコメント付近、

－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－
// Check Raidboss attack
// Character will be petrified if attacking a raid that's more
// than 8 levels lower
＞コメントアウト //if (level > target.getLevel() + 8)
＞追加 if (level > target.getLevel() + 80)

// Check Raidboss attack
＞コメントアウト //if (player.isRaid() && getLevel() > player.getLevel() + 8)
＞追加 if (player.isRaid() && getLevel() > player.getLevel() + 80)
－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－

と8レベル以上→80レベル以上と変更してリビルドするだけです。
リビルド後の/build/dist/gameserver/l2jserver.jarと/build/dist/login/l2jserver.jarをそれぞれ上書きして実行すればOK。

②接続先DNS名の変更

④再暗号化l2.iniを扱えるようにl2.exeにパッチ当て
－－－－－－－－－－－－－－－－－－－

dec-l2.iniの項目
Port=7777　は鯖側設定と共に変更してみたけど、設定無視して？7777固定で通信してるっぽかったのでsoftether側標準ポートを変更しました。（後述）
ServerAddr=auth.lineage2.jp　この部分を自前DNS名に変更したところ、自前鯖に接続先を変更できました。

l2.ini書き換えにより、クラ側は毎度接続先設定を変えることなく繋ぐことができます。

－－－－－－－－－－－－－－－－－－－
①softetherを使っている場合はsoftether標準ポートを7777以外に変更しておく

・ビルド済L2J-serverフォルダ
・hostsとEasyPHP\wwwに置くGameGuardファイル
