How to flex inet records

README for generating flex file with INET records

 

Procedure:

 

1)      Download make_inetrec.exe and make_hs.pl  

2)      Create an input file with your records, follow the procedure descried bellow

a.       Each record MUST have 5 fields (Even though you don’t need all five fields), each fields must be in the different line.

b.      Each record should be in this order

                                                               i.      APPID

                                                             ii.      LOC_ID

                                                            iii.      Rel_position (less than 254)

                                                           iv.      Prompt (less than 25 char)

                                                             v.      Url (less than 129 char)

c.       Start the new record immediately following the previous record

d.      End the file with ^ (shift + 6) char.

e.       See the Sample_input_file input file

f.        To find out APPID value and LOC ID value please see APP_ID and LOC_ID

g.      Note: Please note that you CAN NOT use enum value (e.g. DL_DB_INET_J2ME_GAMES_ID) in your input file, instead you need to use its corresponding value (e.g. 0, 1,2, etc…)

h.       If you want to leave prompt as NULL or Url as NULL, simply hit enter at that location (see my Sample_input_file ).

   

3)      Once you create this input_file do the following

$make_inetrec <input_file>

This will put output into DL_DB_INET_RECORDS file.

      

4)      Once you have DL_DB_INET_RECORDS file use the make_hs.pl to make the .hs file

$make_hs.pl –i 1 DL_DB_INET_RECORDS

 

This will create fsac.hs file you can flex simulator or phone with this file.
