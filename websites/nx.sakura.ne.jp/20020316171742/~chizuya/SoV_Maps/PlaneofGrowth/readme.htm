〇 はじめに…

このＣＧＩはフリーウェアですが著作権は放棄してません。
だから、スクリプト規定はちゃんと守ってくださいです。

〇 My Diary とは

ツリー形式の掲示板スクリプトです。
高機能かつ使いやすいように設計されています。

〇 設置するには？

初めて設置する方や、良くわからない方は CGI-StaTion の "初めてのＣＧＩ設置講座" をご覧ください。
ここでは、簡単にしか説明してないです(^-^;;

□ ファイルの簡単な説明

flashbbs.lzh を解凍すると次のファイルが作成されるはずなので、確かめてください。 

readme.htm   ... このファイルです
flashbbs.htm ... HTML（利用方法）
flashbbs.cgi ... CGIスクリプト
fbeditor.cgi ... CGIスクリプト
flashbbs.log ... 記録ファイル
flashbbs.cnt ... 記録ファイル(カウンター)
all_news.gif ... 画像ファイル
new_news.gif ... 画像ファイル
news.gif     ... 画像ファイル
space.gif    ... 画像ファイル
jcode.pl     ... 日本語変換ライブラリ(Kazumasa Utashiro)

diary.log は log フォルダの中にあります。

Mac(マッキントッシュ）の一部の解凍ソフトで、記録ファイル等が作成されない場合があるみたいです(T_T)
その場合は、お手数ですが他の解凍ソフトでやってみてくださいです。
(たいていの場合、作成されないのは空のファイルなので、自分で作成できる方は作成してもらっても構わないと思います。）

□ テスト設置

まずはテスト設置します（テスト設置は必要です）。
flashbbs.cgi,fbeditor.cgi をエディタで開きましょう！
先頭行、#!/usr/local/bin/perl を書き換えます。 


1|#!/usr/local/bin/perl
2|
3|#| XXXXX Version x.xx
4|#| This script is free.

ここは一般的な Perl のパスを記述してあるので、あなたのプロバイダで違う場合は書き換えてください。
間違っても適当に書かないように(^-^;

□ ファイルをFTPで転送する

普通は全てのファイルを同じディレクトリ（フォルダ）内に置けばオーケーです。
(プロバイダによって指定されている場合は、オプション設定で変更の必要あり）。
jcode.pl がすでに存在している場合は、上書きしてください。
なお、パーミッション（アクセス権）は次のように設定しましょう(プロバイダが指定している場合は、プロバイダの指示に従ってください。) 

readme.htm   ... 転送不要です。転送しないほうが良いでしょう。
flashbbs.htm ... パーミッションは[644] に設定しましょう。
flashbbs.cgi ... パーミッションは[755] に設定しましょう。
fbeditor.cgi ... パーミッションは[755] に設定しましょう。
flashbbs.log ... パーミッションは[666] に設定しましょう。
flashbbs.cnt ... パーミッションは[666] に設定しましょう。
xxxxx.gif(画像ファイル)    ... パーミッションは[644] に設定しましょう。
jcode.pl     ... パーミッションは[755] に設定しましょう。
一般的に解凍時に作成されたままで、ＦＴＰ送信すればオーケーです。

□ ブラウザで確認する

ＦＴＰで転送したらうまく設置できているか確認しましょう。
ブラウザを立ち上げ、直接 flashbbs.cgi を呼び出しましょう。
ちなみに、リンクをはるときも、flashbbs.cgi にはってください。
うまく表示されたでしょうか？
エラーが出たら、慌てずにトラブルシューティングを読みましょう（ちなみに、初めての方は８割ぐらい失敗するみたいです。僕も良く失敗するし〜♪）

〇 トラブルシューティング

どうしてもうまくいかない、エラーが頻発する、文字化けがある等、不都合がある場合はトラブルシューティングをご利用ください。
CGI-StaTion にはトラブルシューティングのページがあります。
トップページのクイックリンクでトラブルシューティングを選択して下さい。
「トラブルシューティングでもあかんわぁ」って方は、サポート掲示板をご利用下さい。

掲示板の管理者（あなた）の名前を書き込んで下さい。
〜 の部分のみを書き換えてください。

あなたのメールアドレス:$admin_email = '〜';

掲示板の管理者（あなた）のメールアドレスを書き込んで下さい。
〜 の部分のみを書き換えてください。

タイトル:$title = '〜';

掲示板のタイトルです。
タグは無意味です。使わないで下さい。
〜 の部分のみを書き換えて下さい。

<BODY>タグの設定

<BODY>タグの設定です。
基本的な色や、背景画像等を設定できます。
それぞれの '〜' の間のみを書き換えて下さい。

帰りのページの URL:$back_url = '〜';

掲示板から戻る時のリンクURL を記述します。
〜 の部分のみを書き換えて下さい。

時間の書式:$date = '〜'; $date_type = '〜';

タイムスタンプの設定です。
この CGI に投稿された記事の、時間の記録書式を指定します。
$date = '〜'; の 〜 を書き換えます。
(yearは年、monは月、dayは日、hourは時、minは分、secは秒をあらわします。）

例） $date = "year/mon/day hour:min:sec"; なら、
２０００年１月１５日、３時１１分２０秒は 00/1/15 3:11:20

$date_type = n; の n を書き換えることで、日付の修正方法を設定できます。
例）１月１１日、１０時４分７秒 のそれぞれについて
0:修正しないときは、1/11 10:4:7 となる。
1:修正する（時分秒）ときは、1/11 10:04:07 となる。 （時分秒が二桁になる）
2:修正する（全て）ときは、01/11 10:04:07 となる。（全てが二桁になる）

色の設定:$xxxxxx_color = '〜';

掲示板の色の設定ができます。
$em_color はタイトルの色で、初期で ■[#FF0000]になってます。
$kiji_title_color はタイトル欄の背景色で、初期で ■[#EEEEFF]になってます。
$kiji_title_bgcolor はボーダー色で、初期で ■[#261B10]になってます。
$form_bgcolor はメッセージ欄の背景色で、初期で ■[#CED1B5]になってます。
$res_color は日付などの色で、初期で ■[#444499]になってます。

＞ FlasH BBS Pro は 2000年問題に対処されてますか？
＞ もし、対処されてないようでしたら、どのような問題があるんでしょう？

v1.41 から対処されています。
それ以前のバージョンでは、日付部分がおかしくなる可能性があります（100/12/11 等と記録される。100 は 2000年と同値らしいです。）
また、クッキー処理がまったく機能しなくなると思われます。
クッキー処理とは、一度掲示板に書き込んだら名前等を記入しなくても自動的に記録される処理ですが、これが機能しなくなります。
ログが消えたり、壊れるといったことはありません。

いずれも、〜 の部分のみを書き換えて下さい。

画像の設定:$gif_xxxxx = '〜';

記事を選ぶ際にわかりやすくするための画像アイコンを設定します。
それぞれの画像ファイル名を用意した画像名に書換えることで変更できます。
また、プロバイダ等の設定で、画像ファイルが正しく表示されない場合は http:// から書いてみてください。

また、$gif_width = n; と、$gif_height = n; はそれぞれ画像の横幅、縦幅を指定します。
このままで問題ありませんが、画像を換えた場合には調整してください。

いずれも、〜 の部分のみを書き換えて下さい。

jcode.pl のある場所:$jcode = '〜';

jcode.pl（日本語変換ライブラリ）のある場所（パス）を指定します。
プロバイダによって変えるように指示がある場合にのみ、プロバイダの指示に従って書き換えて下さい。
通常は書き換える必要はありません。
〜 の部分のみを書き換えて下さい。

記録用ファイル:$logfile = '〜';

記録用ファイルの名前を指定します。
通常は書きかえる必要はありません。
〜 の部分のみを書き換えて下さい。

鍵ファイル:$lock1,$lock2 = '〜';

鍵ファイルを作成させる場所を指定します。
鍵ファイルは自動的に作成・削除されるため、通常は書き換える必要はありません。
しかし、プロバイダから注意がある場合は書き換えてください。
〜 の部分のみを書き換えて下さい。

鍵ファイルの使用:$lock_flag = n;

鍵ファイルの使用の有無は決めます。
プロバイダのサーバーが WinNT の場合、使わない方が良いです。
通常は変更する必要はありません（使って下さい）
n の部分のみを 0 か 1 に設定して下さい。

パスワード:$password = "〜";

管理用エディタを使うためのパスワードです。
半角英数 8 文字以内で必ず書き換えて下さい。
このパスワードを利用すると、投稿の削除、編集、投稿者に返事ができます。
〜 の部分のみを書き換えて下さい。

〇 管理用エディタ(FlasH BBS Pro Editor の使い方)

FlasH BBS Pro に関しては flashbbs.htm をご覧ください。
ここではエディタの使い方を説明します。

Q.ページを移動するには？
A.ページを選択し、[編集開始]で移動できます。

Q.記事を削除するには？
A.削除したい記事の 削 をチェック（複数可）した後、[編集開始]でできます。
ただし、ツリー掲示板の特性上、完全に削除されない場合があります。
完全に消す場合はツリー削除をしてください。

Q.記事を編集するには？
A.編集したい記事のタイトルをクリックして下さい。
その記事の情報が現れます。
内容を書き換えて、内容更新で書き換えられます。

Q.ツリーごと削除するには？
A.ツリーごと削除したいツリーの [ツリー削除] をチェック（複数可）した後、[編集開始]でできます。

Q.ダウンロードって？
A.ダウンロードボタンをクリックすると、記事のデータをダウンロードできます。
ダウンロードしておくと、万が一、ログが壊れた場合に復帰させることが可能です。
ダウンロードしたファイル名は fbeditor.cgi になってしまうので、flashbbs.log に名前を変えておくといいでしょう。

Q.最適化処理って？
A.通常は絶対に使わないで下さい。
これはある問題が発生した場合の非常用の処理です。
もし、このボタンを押すと、最新一括機能等がしばらく壊れます。
このボタンは FlasH BBS Pro が最適化をしてくださいという警告があった場合にのみ利用してください。
この警告は使いはじめて約9999件の投稿があると、発生すると思います（多少、変わると思います）
