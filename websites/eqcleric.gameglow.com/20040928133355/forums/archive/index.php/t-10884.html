Yes, Lord Yelinak is very killable currently. If you are having problems with lots of Linkdeads, you MUST MAKE SURE PEOPLE SET THEIR DATARATE.TXT file.

It defaults to 5.0, even with a rock solid DSL, zero PL, you WILL go LD in a 50 man raid.

If you use a modem, set it to 7.0 or 9.0

If you have a DSL, CABLE, or T1+ set it to at least 12.0

Basically the datarate.txt is the value at which the server will arbitrarly disconnect you if you pull more data than is set in that file. On raids I regularly spike at over 9000 transfer rate, which is why I have mine set at 15.0 and almost never go LD (I have a 384kbit dsl connection).

DO NOT set your datarate.txt file just according to your connection speed! What this file does is tell the system how much data to try to give you in a small amount of time. If you run on an old system (< 500mhz) and have the data rate set to anything greater then 7-8 you'll probably have issues.

I have a 400 mhz system and a very strong cable modem line and if my datarate is set to 7 I go LD. Since I changed it from 9 to 6 though I have NOT gone link dead on any raid, unless the zone has come down.

Verant also said to not set the number abouve 9.0. Doing so might help if you have a very strong system but I have freinds with cable and great systems and do large raids with everything on and there datarate is only set to 9. Anything above that is over kill.

Here is an old post I had on another board with some more suggestions to avoid going LD.
Most of this is common since.

Make 2 eqcleint.ini files, one that is normal day to day and one for raids.


1. In the raid eqcleint.ini file setup the following lines like this
LoadArmor17=FALSE
LoadArmor18=FALSE
LoadArmor19=FALSE
LoadArmor20=FALSE
LoadArmor21=FALSE
LoadArmor22=FALSE
LoadArmor23=FALSE

2. Stuff you know
ShowSpellEffects=0
ServerFilter=1(Don't do this as a cleric, get a better system you just can't wait 6 seconds to see health bars move.)

3. Video Settings
UseD3DTextureCompression=TRUE
VideoModeBitsPerPixel=16

4. In the Display settings in the game
Sky = 3d Sky off
Particle Density off
PC Names = off
Level of Detail = off
MIP Map = on (If you video card does this. It will dramatically slow down your zone load times but helps lag once in the zone.)

5. BIOS
Check your BIOS, and fiddle with the setting called “AGP Aperture size”. Make sure it’s higher than 16MB. I found that the best setting for me was 256MB (usually it is good at around half of your total Ram, but it varies from computer to computer).
There are a lot of other bios settings that help things but my bios is so old I couldn't do anything else and thus I can't give you more suggestions.
6. Mess with datarate.txt. With my cable modem I had it on 9, but this was sending more data then my CPU could handle. Don't be afraid to turn this down.


Doing these changes I went from going LD once or twice a night to NEVER going LD, and didn't change any system components.
