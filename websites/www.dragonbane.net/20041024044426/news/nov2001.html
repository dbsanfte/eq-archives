Looking for Information/Help | by Sandman
I'm looking for information from anyone that is knowledgable about using SCSI drives in a Win2k Server environment. I am still have trouble getting the TA server up and running on Win2k with the new SCSI Drive. The O/S will install, then upon the first boot after the installation (and every boot thereafter) the O/S will hang after the "log in" and "personalizing settings" screens. There are no error messages displayed on the all blue screen, and the mouse continues to work, so it is not frozen. I am sure that the cause is a SCSI issue, because I borrowed an IDE drive, removed the SCSI card, and performed the exact same install, and the O/S installed and performed perfectly. The SCSI hardware works
fine under a Win98 O/S........ but not under Win2k Server. The symptoms are not caused by a bad CD, because both CD's (Win2k Server and Win2k Advanced Server) seem to work great. Here is a list of the hardware installed:

- Abit BP6 dual processor board with the most recent BP6_RU Bios installed.
- Adaptec 29160LP SCSI Controller with the most recent 3.10 Bios installed, and using the most recent Win2k Server driver from the Adaptec website (w2k_100b_pci_drvs.zip)
- IBM Ultrastar Ultra160 DDYS SCSI Drive plugged into the end of the SCSI cable, as the operators manual dictates.
- 2 port LVD Ultra 160 SCSI cable......... terminated at the end of the cable, immdiately after the drive, as the operators manual dictates.
- a powered 80-pin (drive) to 68-pin (LVD Cable) adapter, located at the end of the drive.
- a dual drive bay cooler........... :-)
