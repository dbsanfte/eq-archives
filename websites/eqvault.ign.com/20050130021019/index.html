Wed 26 Jan 2005

  Collsion Update
Print News | Mail News
Grumbuk posted a new thread over on EQ Live concerning the state of collsion updates and zone status.

Here's a clip:
The first bug that was fixed was related to subtle differences in the way that the zone servers and the clients load geometry. Turns out that objects (but not the terrain) on the zone server had their "triangle normals" pointed in the opposite direction for zones created using the new DX9 data format (EQG files). This was causing problems for NPC pathing and any other server code that depending on being able to "find the ground" when standing on top of one of these objects (as opposed to the standard terrain geometry). Many Omens of War zones had a number of places where entire sections of the zone were using this type of geometry, such as the tunnels in the bloodfields, and this would cause all kinds of problems with pathing, summoning pets, and things of that nature. After this problem was finally identified thanks to some debugging features we've recently added, it was easy to fix and seems to have helped a lot. This fix went in with today’s update.

Interesting article with a ton of answers. Check it out!

Fri 21 Jan 2005

  Brenlo posts on class re-envisioning
Print News | Mail News
Brenlo posted over on EQ Live boards about the status on upcoming class changes. Here's a clip:
Does SOE consider every role to be of equal value to a group?

As the game has changed and evolved, the roles have shifted in their value. For example, the advent of longer buff timers means that a buffer might not be as needed in a pick up group as a DPS class is because the buffs can be acquired by players in a safe zone long in advance of actually reaching the content that requires them. This is another tricky part of this puzzle that we are sorting out. We are looking at the group roles and intend to make sure that each role is necessary.

Interesting stuff, and you can read the whole post here.
