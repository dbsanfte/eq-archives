Tables

Html tables work fine in our wiki version, the only code that will not function in your tables is the <tbody> element, simply delete the tbody tags and your table will function properly. For wiki codes see below.

How to create a very basic table: 

{| class="wikitable"
|-
! header 1
! header 2
! header 3
|-
| row 1, cell 1
| row 1, cell 2
| row 1, cell 3
|-
| row 2, cell 1
| row 2, cell 2
| row 2, cell 3
|}

Turns out like:

header 1 header 2 header 3
row 1, cell 1 row 1, cell 2 row 1, cell 3
row 2, cell 1 row 2, cell 2 row 2, cell 3
Breaking the code down. 
{|            tells the wiki that a table code is starting

class=        tells the wiki what class it is, you do not need to use this code  
              and there are many other codes you can use in this spot to 
              customize your table

!             Tells the wiki to bevel that section, can be used for both headers
              and normal cells

|-            Start/End Section code

|             Beginning of new cell (Can also be used for headers if you do not 
              wish for them to be beveled)

|}            Table Finished

Table Of Contents Tips

[edit]

Repositioning the TOC

Using __TOC__ will place the table of contents on the left hand side of the line wherever you place that code instead of its default position over the first header.
Note those are double underscores on either side

[edit]

TOC aligned to right

<div style="float:right; margin: 1em 0 1em 1.5em;">__TOC__</div> forces the table of contents on to the right side.
Note those are double underscores on either side

[edit]

Removing the Table of Contents

__NOTOC__ Forces the system to hide the table of contents for all users viewing the page.
Note those are double underscores on either side

[edit]

Hiding subsections in TOCs

When an article or project page has a very large number of subsections, it may be appropriate to hide lower-level subsections from the TOC. You can specify a limit for the lowest-level section that should be displayed using {{TOClimit|limit=n}}, where n is the number of = signs that are used on each side of the lowest-level section header that should be displayed (e.g. 3 to show ===sections=== but hide ====sections====). Note: I have not had to use this and can not be certain it works with our wiki version -Ocura

[edit]

Headers

You need 4 or more of any heading size to create a table of contents, once you have 4 headings of any size the TOC will create itself. The = signs on either end tell the wiki what size of header it is, one = on each side being the largest and four ==== on each side being the smallest. Note that if you use a larger heading and later use a smaller heading, the smaller will show up in the table of contents as a sub-header to the larger one.

HTML elements

[edit]

<a>, <span>, <div> Elements

An important thing, while the version of media wiki which we use will accept insane amounts of html it does NOT accept <a> elements (such as <a href) so switch to <span> or <div> elements which function properly.


[edit]

Table <tbody> Elements

<tbody> and </tbody> elements Do Not work, simply remove them if transferring a web page to wiki and your table will function properly.

Category Tips:=

[[Category:Test]] Creates a category for easy page tracking, shows up on the bottoms of the pages, clicking on the category at the bottom of the page will take you to a 'slightly' alphabetical index of pages in that category. Note I say slightly alphabetical as it appears to only take the first letter of the page titles into consideration.



Putting a : before the Category link will enable you to link to a category page without including the current article as part of the category. Example [[:Category:Help]] links you there whereas [[Category:Help]] simply tells the wiki to include this article in the specified category.

Stop All Html and Wiki Formatting

The <pre></pre> elements or putting a space before the first word in each paragraph will stop all wiki and html formatting.

Without the Pre Elements: This sentence is an example of my attempt to mess up the formatting and using the pre elements to stop

it from realigning the text!


With the Pre Elements: 
This sentence is an example    of    my attempt to mess up the formatting and using the pre elements to stop

it from realigning the text!


Using a space before the first word on the line but no other formating codes also functions as a Pre command: 

This sentence is an example    of    my attempt to mess up the formatting and using the pre elements to stop

it from realigning the text!

More On External Images

This is an issue that had me stumped, Kaltemar has posted a work around for posting thumbs from our Vault servers without having to manually download and re-upload them to the wiki:

Kaltemar posted:
Wanting to display an image from your screenshots DB? Just add "&picture.png" to the end of your screenshot URL (so http://wowvault.ign.com/screenshots/thumb.php?ss=1 becomes http://wowvault.ign.com/screenshots/thumb.php?ss=1&picture.png) - this confuses MediaWiki, which would otherwise just assume you're pasting a link - it then believes you're linking to a file called picture.png not thumb.php!

Positioning Images

[edit]

Uploaded Wiki Images

[[Image:Tabula.jpg|250px|thumb|mouse over text]] shows up like this Image:Tabula.jpg Laymans Terms, breaking down the code:

  • [[ and ]] on the ends declares it an internal image.
  • Image: Tells the wiki what type of file to display
  • Pipes | Tells the wiki a new but related command follows
  • 250px is the overall size the image will display, adjust it accordingly
  • thumb tells the wiki to display the image to the right hand side in a clean thumb box
  • mouse over text This is what viewers will see when they hover their cursor over the image




[edit]

Positioning External Images

[[Image:imagename.jpg|right|alternate text]] The 'Image:' tag tells the wiki to display the item as an image not a link. '|right' tells the wiki to display the image in the far right column, '|left' would display the image on the left side, text will wrap around the image. '|alternate text' isn't required but obviously can be added if you want the mouse over to read differently.


Div Tags and Tables work wonders for positioning external images, also the use of div tags allows text wrapping around the image.

With <div> Tags

Example: <div style="display:inline; width:220px; float:right;"> http://vnmedia.ign.com/tabularasavault.ign.com/images/articleclips/thumbs/striderthumb.PNG </div>

With Wiki Table Codes

Example:

Pages all admins should know of:

In your own wiki just add the Special:address like this on the address bar site.ign.com/wiki/index.php/Special:page or drop it onto a private screen (such as this one which is not linked off my main page) with the [[Special:SpecialPages]] command and have an instant direct link to the premade pages.


Special:Specialpages -Displays All 'special' pages for running the wiki- an extremely handy list to have at your fingertips (examples below) Special:Specialpages

Special:Allpages -Displays all pages on your wiki to made indexing easier! Special:Allpages

Special:Allmessages -Don't play with them :p If you get a strange error message this is a good spot to look up what it meant Special:Allmessages

Special:Recentchanges -Shows all recent changes on the Wiki, great for tracking visitors edits etc. Special:Recentchanges

Special:Statistics -Your Wiki stats, Special:Statistics

Special:Newpages -Just what it says ;) Displays pages in order they went up Special:Newpages

Special:Version - Shows basic info on version, help links, and extensions list. Special:Version



[edit]

Easy Navigation Tip

If you get lost and can't find the special pages you need (or for ease of navigation around behind the scenes pages at any time) go to your preferences in the wiki and switch yourself to the 'Nostalgia' skin for a quick drop down bar with easy links. Remember what you see in the Nostalgia skin is not what users see by default! Remember to switch back to ensure the pages look normal after editing them in the Nostalgia skin.
