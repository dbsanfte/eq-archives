Network speed can not possibly be the sole issue. The vast majority of us
have extremely similar connection speed between 28800 and 50000 - I would
guess most have V.90 running just under 50000. While of course upstream is
31200 for V.90 - it's safe to assume the data transfer on "zoning" would be
primarily downstream. I have heard that people running on fat pipes get
good zone times - so clearly this IS an important issue.

However - those of us without those fat pipes (again the vast majority of
us) notice HUGE discrepancies in zoning times based on speed of the machine.
Ever have a party member running on a 166 machine? So clearly - there is
also something processor intensive going on here in addition to the
data-transfer.

I'm guessing that there is some sort of code building complex data
structures from the data it is getting from the net. This would lead me to
believe that with enough memory - there could be something done in code to
store multiple zones in memory...

But of course - only the people with access to the code could answer that
question.

To be honest - I'd be thrilled with a very simple change Verant could put in
with a few lines of code: get rid of that bloody license agreement that I
have to click accept on EVERY single time I load the game... Put a switch
in an INI file or something!
