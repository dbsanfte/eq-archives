From a programming point of view this would be very simple to implement.
Each item already has to be kept track of in the game as a unique item.
Adding the name of the player that created it would be one more data
field for each item. Now the actual problem is how fast can the database
retrieve the information for the item with unique identifiers on each
one. Currently a given item held by any person in the game (e.g. banded
gorget) goes to a generic stats file to retrieve its data. The hit from
adding names of crafters is the extra time when it has to retrieve the
crafters' name. Each item has a unique name in the game so adding the
data field there might not be too bad since it is keeping track of the
item on the character with that unique name. If they were stored
together it might make it faster. Oh well, just the musing of a
programmer.
