From time to time in my gaming career, I have had to fight battles with
joystick drivers. Sometimes, you could load the equipment and program it
with the OEM software but the OEM drivers would not work with Windows. So,
you had to set the joystick/gameport to a generic one (two-axis/two button)
to get things to work or to partially work.

Sometimes you needed to have the resources set to a very specific port
setting in device manager - resources (201-201 for a certain joystick I once
had would work where 200-209 would fail.) And of course if you have
conflicting interuppts or memory addresses, you need to resolve those.

Mine hooks up inline with my keyboard cable so it can send in keystrokes.
If the Gravis tries to do it through redirecting the calls to the keystroke
interupts and EQ is polling that area of ram directly - there could be
trouble. Are there any 'switches' or jumpers on the Gravis you can set?
Call Gravis Tech Supp? Do they know if it will work with EQ?
