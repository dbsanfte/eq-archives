I get the impression that many of Verant's developers are like one I have
here at work. He writes good code for the most part and recently completely
rewrote a subprogram from scratch to clean it up and improved performance
threefold. He changed the input parameters to remove obsolete ones as well.
Unfortunately he forgot to change the actual calls to the subprogram so
nothing worked when he put it through the pipe and it got to me for review.
I had to actually go into the code myself and physically show him where the
mistake was before he'd even admit the possibility of an error. Here it was
a case where he'd look at his code, see no errors and reject all claims to
the contrary. It never occurred to him that he actually forgot to write
some extremely simple code in another program altogether. This guy
continually makes these sorts of errors and these simple errors have
obscured his generally sound coding to the point where he is likely to lose
his job if it continues much longer. Bottom line is, it does no good to
write brilliant code if that code can't ever be executed properly due to
lack of ability to pay attention to details.
