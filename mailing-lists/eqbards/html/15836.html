I'm pretty sure the effect icons are entirely client-side, and
have no basis in reality (what the server thinks). You'll
notice that in those long lag bubbles, your icons will drop.
But if the effect is server-side (e.g. Hymn), the effect is
still there (hp still increment).

I think the way it works is your client sends a "starting this
song" message to the server. The server sends back an "ok,
starting this song" message (aka "your feet move faster"),
which causes the client to put up the icon. The icon stays
for two heartbeats (somehow synchronized with the server, but
timed locally). So if you yank the network cable, the icon
disappears on its own at the appropriate time. If there's any
server-side lag, this can cause the client/server to get out
of synch, causing icons to disappear locally before the server
ends the effect.
